# 컴퓨트 노드

**컴퓨트 노드** 메뉴에서는 Cloud:iA에 등록된 컴퓨트 노드의 상태와 리소스 정보를 확인하고, 오버커밋 비율을 관리할 수 있습니다.
컴퓨트 노드 상세 화면에서는 리소스, 시스템 정보, 인스턴스 정보를 탭으로 구분하여 제공합니다.

---

## 컴퓨트 노드 목록

컴퓨트 노드 목록 화면에서는 등록된 컴퓨트 노드의 상태와 기본 운영 정보를 확인할 수 있습니다.

### 주요 작업

- **`편집`**: 노드를 한 개 선택했을 때 활성화됩니다. 클릭 시 **컴퓨트 노드 편집** 화면으로 이동합니다.
- **이름 클릭**: 선택한 컴퓨트 노드의 **상세 화면**으로 이동합니다.

> 💡 **참고:** 컴퓨트 노드 메뉴에서는 생성/삭제 기능을 제공하지 않으며, 편집 기능만 제공합니다.

### 테이블 컬럼

| 컬럼 | 설명 |
| --- | --- |
| **이름**          | 컴퓨트 노드 이름입니다. 클릭 시 상세 화면으로 이동합니다. |
| **상태**          | 현재 컴퓨트 노드 상태입니다. |
| **인스턴스 수**      | 해당 노드에 배치된 인스턴스 수입니다. |
| **CPU 오버커밋 비율** | CPU 오버커밋 비율입니다. |
| **메모리 오버커밋 비율** | 메모리 오버커밋 비율입니다. |
| **생성 일시**       | 컴퓨트 노드가 등록된 일시입니다. |

---

## 컴퓨트 노드 편집

컴퓨트 노드 편집 화면에서는 노드 이름은 조회만 가능하며, CPU/메모리 오버커밋 비율만 수정할 수 있습니다.

### 입력 항목

| 항목 | 설명 |
| --- | --- |
| **이름** | 읽기 전용 항목입니다. |
| **CPU 오버커밋** | 슬라이더로 비율을 조정합니다. |
| **메모리 오버커밋** | 슬라이더로 비율을 조정합니다. |

### 오버커밋 비율 규칙

- 설정 범위: **1.00 ~ 2.00**
- 권장 기준: **1.5 이하**
- 기존 값과 동일하면 **저장 버튼이 비활성화**됩니다.

### 저장/취소

- **`저장`**: 변경한 오버커밋 비율을 반영하고 목록 화면으로 이동합니다.
- **`취소`**: 변경 사항을 저장하지 않고 목록 화면으로 이동합니다.

---

## 컴퓨트 노드 상세

컴퓨트 노드 상세 화면은 상단 **기본 정보**와 하단 **탭 정보**로 구성됩니다.

### 기본 정보

| 항목 | 설명 |
| --- | --- |
| **이름** | 컴퓨트 노드 이름입니다. |
| **상태** | 컴퓨트 노드 운영 상태입니다. |
| **VDSM 연결 상태** | VDSM 연결 상태를 표시합니다. |
| **Agent 연결 상태** | Host Agent 연결 상태를 표시합니다. |
| **네트워크 상태** | 네트워크 점검 결과를 표시합니다. |
| **인스턴스 개수** | 노드에 배치된 인스턴스 수입니다. |
| **CPU 오버커밋** | CPU 오버커밋 비율입니다. |
| **메모리 오버커밋** | 메모리 오버커밋 비율입니다. |
| **생성 일시** | 컴퓨트 노드 생성 일시입니다. |

### 탭 구성

#### 리소스

| 항목 | 설명 |
| --- | --- |
| **전체 논리 CPU 수** | 노드의 전체 논리 CPU 수입니다. |
| **할당된 논리 CPU 수** | 현재 할당된 논리 CPU 수입니다. |
| **전체 메모리** | 전체 메모리 용량입니다. |
| **할당된 메모리** | 현재 할당된 메모리 용량입니다. |
| **수정 일시** | 리소스 정보 마지막 갱신 시각입니다. |

#### 시스템 정보

| 항목 | 설명 |
| --- | --- |
| **커널 버전** | 운영체제 커널 버전입니다. |
| **QEMU-KVM 버전** | 하이퍼바이저(QEMU-KVM) 버전입니다. |
| **Libvirt 버전** | Libvirt 버전입니다. |
| **VDSM 버전** | VDSM 버전입니다. |
| **하이퍼스레딩** | 하이퍼스레딩 사용 여부입니다. |

#### 인스턴스

인스턴스 탭에서는 현재 컴퓨트 노드에 배치된 인스턴스 목록을 제공합니다.

| 컬럼        | 설명                   |
|-----------|----------------------|
| **이름**    | 인스턴스 이름입니다.          |
| **ID**    | 인스턴스 ID입니다.          |
| **상태**    | 인스턴스 상태입니다.          |
| **보안 유형** | 인스턴스 보안 타입입니다.       |
| **vCPU**  | 인스턴스에 할당된 vCPU 수입니다. |
| **메모리**   | 인스턴스 메모리 용량입니다.      |
| **IP 주소** | 인스턴스 IP 주소입니다.       |
| **생성 일시** | 인스턴스 생성 일시입니다.       |

### 인스턴스 마이그레이션

인스턴스 탭에서 인스턴스를 한 개 선택하면 **`마이그레이션`** 버튼이 활성화됩니다.

#### 수행 절차

- 마이그레이션 모달에서 상단 **`라이브 마이그레이션` 토글**로 방식 선택이 가능합니다.
  - 토글 `OFF`: **콜드 마이그레이션**
  - 토글 `ON`: **라이브 마이그레이션**
- 대상 노드를 선택하지 않으면 `실행` 버튼이 비활성화되며, 대상 노드 선택 시 `실행` 버튼이 활성화됩니다.
