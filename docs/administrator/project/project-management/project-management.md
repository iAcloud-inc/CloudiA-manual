# 프로젝트 관리

### 주요 작업

- `생성`: 새로운 프로젝트를 등록합니다. (항상 활성화)
- `편집`: 선택한 프로젝트를 편집합니다. **1개** 선택 시 활성화됩니다.
- `삭제`: 선택한 프로젝트들을 삭제합니다. **1개 이상** 선택 시 활성화됩니다.

> 💡 **공통 조건:** 목록에서 좌측 **체크박스**를 통해 프로젝트를 선택합니다. 다중 선택이 가능합니다.
> 헤더 체크박스를 통해 전체 선택/해제를 할 수 있습니다.

### 검색

- **검색 조건**: 드롭다운에서 `이름`을 선택합니다.
- **검색어 입력**: 검색어를 입력합니다. (플레이스홀더: `검색어를 입력해주세요.`)
- **검색 버튼**: 검색 버튼을 클릭하여 검색을 실행합니다.

### 테이블 컬럼

프로젝트 관리는 다음과 같은 컬럼으로 구성된 표 형태로 표시됩니다:

| 컬럼          | 설명                                                      |
| ------------- | --------------------------------------------------------- |
| **(선택)**    | 체크박스를 통해 프로젝트를 선택합니다. 개별/전체 선택이 가능합니다. |
| **이름**      | 프로젝트의 표시명입니다. **클릭 시 상세 화면으로 이동합니다.** |
| **ID**        | 프로젝트의 고유 식별자(ID)입니다.                         |
| **vCPU**      | 사용 중인 vCPU 코어 수와 할당된 vCPU 총량을 표시합니다. (형식: `사용량 / 할당량`) |
| **메모리**    | 사용 중인 메모리 용량과 할당된 메모리 총량을 표시합니다. (형식: `사용량 / 할당량`, 단위 포함) |
| **디스크**    | 사용 중인 스토리지 용량과 할당된 스토리지 총량을 표시합니다. (형식: `사용량 / 할당량`, 단위 포함) |
| **외부 IP**   | 사용 중인 외부 IP 개수와 할당된 외부 IP 총 개수를 표시합니다. (형식: `사용량 / 할당량개`) |
| **설명**      | 프로젝트에 대한 설명 글입니다.                            |
| **생성 일시** | 프로젝트가 생성된 날짜와 시간을 표시합니다.               |

## 프로젝트 상세

### 상단 요약

- **이름**: 프로젝트의 표시명입니다. 
- **ID**: 프로젝트의 고유 식별자(ID)입니다.
- **설명**: 프로젝트에 대한 설명 글입니다.
- **vCPU**: 사용 중인 vCPU 코어 수와 할당된 vCPU 총량을 표시합니다.
- **메모리**: 사용 중인 메모리 용량과 할당된 메모리 총량을 표시합니다.
- **디스크**: 사용 중인 스토리지 용량과 할당된 스토리지 총량을 표시합니다.
- **외부 IP**: 사용 중인 외부 IP 개수와 할당된 외부 IP 총 개수를 표시합니다.
- **생성 일시**: 프로젝트가 생성된 날짜와 시간을 표시합니다. 
- **수정 일시**: 프로젝트가 마지막으로 수정된 시점을 표시합니다.

### 하단 탭  
- **멤버**: 멤버 이름, 멤버 ID, 역할 정보를 표로 제공합니다. 
  - **`편집`** 버튼을 클릭하여 멤버 편집 화면에서 해당 프로젝트의 멤버 역할 등록 및 삭제가 가능합니다.

## 프로젝트 생성

프로젝트의 이름과 설명, 프로젝트가 사용할 수 있는 리소스의 최대치(할당량)를 지정합니다. **메모리**와 **디스크**는 입력란 오른쪽의 드롭다운에서 단위를 `MB`, `GB`, `TB` 중 선택할 수 있습니다.

### 1. 기본 정보

- **이름**: 프로젝트 이름을 입력합니다.
  - 입력 힌트: `영문 대/소문자, 숫자, -, _ 만 허용하며 최대 20자까지 입력 가능합니다.`
- **설명**: 프로젝트에 대한 부가 설명을 입력합니다. (선택)
  - 글자 수 카운트: `0/200`

### 2. 프로젝트 리소스 할당량

- **vCPU**: 프로젝트에서 사용 가능한 가상 CPU 코어의 총 개수를 설정합니다.
  - 입력 힌트: `0 이상의 정수를 입력하세요.`
- **메모리**: 프로젝트에서 사용 가능한 메모리 용량을 설정합니다.
  - 입력 힌트: `0 이상의 정수 또는 소수를 입력하세요.`
  - 단위 드롭다운: `MB / GB / TB`
- **디스크**: 프로젝트에서 사용 가능한 스토리지 용량을 설정합니다.
  - 입력 힌트: `0 이상의 정수 또는 소수를 입력하세요.`
  - 단위 드롭다운: `MB / GB / TB`
- **외부 IP**: 프로젝트에서 사용할 수 있는 외부 IP 주소의 개수를 설정합니다.
  - 입력 힌트: `1 이상의 정수를 입력하세요.`

#### 입력 필드 유효성

| 필드           | 허용 값                  | 검증 규칙                                    | 비고 |
| -------------- | ------------------------ |------------------------------------------| ---- |
| **프로젝트 이름** | 최대 20자  | 영문 대/소문자, 숫자, 하이픈(-), 밑줄(_)만 사용 가능; 동일 이름 중복 불가 | 필수 |
| **설명**      | 최대 200자 | 입력 제한 없음 (모든 문자 허용)                      | 선택 |
| **vCPU**    | 정수 (0 이상)            | -                                        | 필수 |
| **메모리** | 숫자 (0 이상), 단위 선택 | -                                        | 필수 |
| **디스크** | 숫자 (0 이상), 단위 선택 | -                                        | 필수 |
| **외부 IP** | 정수 (1 이상)            | -                                        | 필수 |

## 프로젝트 편집

> ⚠️ **주의:** 편집 시에는 기존 사용량 이상으로만 수정할 수 있습니다.

프로젝트의 이름과 설명, 프로젝트가 사용할 수 있는 리소스의 최대치(할당량)를 지정합니다. **메모리**와 **디스크**는 입력란 오른쪽의 드롭다운에서 단위를 `MB`, `GB`, `TB` 중 선택할 수 있습니다.

> 💡 **참고:** **편집**에서는 기존 값이 미리 채워져 표시됩니다.

- **이름**: 프로젝트 이름을 입력합니다.
- **설명**: 프로젝트에 대한 부가 설명을 입력합니다. (선택)
- **vCPU**: 프로젝트에서 사용 가능한 가상 CPU 코어의 총 개수를 설정합니다.
- **메모리**: 프로젝트에서 사용 가능한 메모리 용량을 설정합니다. (예: GB 단위)
- **디스크**: 프로젝트에서 사용 가능한 스토리지 용량을 설정합니다. (예: GB 단위)
- **외부 IP**: 프로젝트에서 사용할 수 있는 외부 IP 주소의 개수를 설정합니다.

#### 입력 필드 유효성

| 필드           | 허용 값                  | 검증 규칙                                          | 비고 |
| -------------- | ------------------------ |------------------------------------------------| ---- |
| **프로젝트 이름** | 최대 20자  | 영문 대/소문자, 숫자, 하이픈(-), 밑줄(_)만 사용 가능; 동일 이름 중복 불가       | 필수 |
| **설명**      | 최대 200자 | 입력 제한 없음 (모든 문자 허용)                            | 선택 |
| **vCPU**    | 정수 (0 이상)            | *(편집 시)* 현재 사용량 이상으로 입력 가능                     | 필수 |
| **메모리** | 숫자 (0 이상), 단위 선택 | *(편집 시)* 현재 사용량 이상으로 입력 가능                     | 필수 |
| **디스크** | 숫자 (0 이상), 단위 선택 | *(편집 시)* 현재 사용량 이상으로 입력 가능                     | 필수 |
| **외부 IP** | 정수 (1 이상)            | *(편집 시)* 현재 사용량 이상으로 입력 가능                     | 필수 |

> ⚠️ **주의:** **유효성 검사 실패 시**
> - 현재 사용량보다 작은 값을 입력하면 입력 필드가 오류 스타일(빨간색)로 표시됩니다.
> - 필드 하단에 에러 메시지가 노출됩니다.
> - `편집` 버튼이 비활성화됩니다.

> 💡 **참고:** **에러 메시지 예시**
> - vCPU: `4 이상의 정수를 입력하세요.` (현재 사용량이 4인 경우)
> - 메모리: `8 이상의 정수 또는 소수를 입력하세요.` (현재 사용량이 8GB인 경우)
> - 디스크: `111 이상의 정수 또는 소수를 입력하세요.` (현재 사용량이 111GB인 경우)
> - 외부 IP: `1 이상의 정수를 입력하세요.` (현재 사용량이 1개인 경우)

> 💡 **참고(공통 조건):** 편집 모드에서는 각 할당량 값을 **현재 사용 중인 자원보다 적게** 입력할 수 없습니다. 예: 현재 vCPU 사용량이 4개인 경우, vCPU는 최소 4 이상으로만 수정 가능합니다.

## 프로젝트 삭제

1. **프로젝트 선택:** 목록 화면에서 삭제하려는 프로젝트들을 체크박스로 선택한 뒤, 상단 도구 모음의 **`삭제`** 버튼을 클릭합니다.
2. **삭제 확인:** 프로젝트 **삭제 확인** 모달이 나타납니다. 모달은 선택한 프로젝트의 삭제 가능 여부에 따라 다르게 표시됩니다:

   **2-1. 삭제 가능한 항목이 있는 경우:**
   - **모달 제목**: `프로젝트 삭제`
   - **섹션 구분**:
     - `삭제 가능한 프로젝트`: 리소스가 없어 삭제 가능한 프로젝트 목록
     - `삭제 불가능한 프로젝트`: 사용 중인 리소스가 있는 항목이 있을 때만 표시
   - **하단 안내 문구** (강조 색상):
     - `사용 중인 리소스가 없는 경우에만 삭제가 가능합니다. 이에 해당하는 N개의 프로젝트를 삭제하시겠습니까?`
   - **하단 버튼**: `취소`, `삭제`

   **2-2. 삭제 가능한 항목이 없는 경우:**
   - **모달 제목**: `프로젝트 삭제`
   - **모달 내용**: `삭제 가능한 항목이 없습니다.`
   - **하단 버튼**: `취소`, `삭제`(`삭제` 비활성)

3. **삭제 실행:** 확인 모달에서 삭제 대상을 다시 확인한 뒤 **`삭제`** 버튼을 눌러 진행합니다. 삭제 가능한 프로젝트만 실제로 삭제되며, 완료 시 상단에 알림이 표시되고 목록에서 제거됩니다.

> ⚠️ **주의:** 프로젝트 내에 **하나라도** 사용 중인 리소스(예: VM 인스턴스, 스토리지 볼륨, 외부 IP, 프로젝트에 속한 이미지 등)가 존재하면 해당 프로젝트는 **삭제할 수 없습니다**. 이 경우 해당 프로젝트는 확인 모달에 *삭제 불가능한 프로젝트* 섹션으로 표시되고 삭제되지 않습니다.

> 💡 **참고:** 한 번에 **여러 프로젝트를 선택하여 일괄 삭제**할 수 있습니다. 단, 앞서 언급한 조건을 만족하는 **삭제 가능한 프로젝트**들만 실제로 삭제 처리됩니다. 선택한 프로젝트가 모두 삭제 불가능한 경우 `삭제 가능한 항목이 없습니다.` 메시지가 표시됩니다.


## 멤버 편집

프로젝트에 할당된 멤버와 각 멤버의 역할을 편집하는 기능입니다. 

1. **멤버 편집:** 프로젝트 상세 화면의 **멤버** 탭에서 **`편집`** 버튼을 클릭하여 멤버 편집 화면으로 이동합니다.
2. **멤버 편집 방법:**
     - **멤버**: 드롭다운 (검색 가능)
       - 멤버 목록에서 선택
       - 드롭다운 내부 검색 입력 제공
     - **역할**: 드롭다운 (검색 가능)
       - 역할 목록에서 선택
       - 드롭다운 내부 검색 입력 제공
     - **삭제** 버튼: 해당 행 삭제
   - **멤버 추가 버튼**: 우측 하단에 위치하며, 클릭 시 새로운 행 추가
   - **하단 버튼**: `취소`, `저장` (유효성 통과 전 비활성화 가능)
3. **멤버 역할 추가:** **`멤버 추가`** 버튼을 클릭하여 새로운 행을 추가하고, 멤버와 역할을 선택합니다.
4. **멤버 역할 삭제:** 삭제하려는 행의 **`삭제`** 버튼을 클릭하여 해당 멤버 역할을 제거합니다.
5. **변경사항 저장:** 모든 변경이 완료되면 **`저장`** 버튼을 클릭하여 변경사항을 적용합니다.

> 💡 **참고:** 멤버 추가 버튼은 최대 멤버 수까지만 활성화됩니다.
