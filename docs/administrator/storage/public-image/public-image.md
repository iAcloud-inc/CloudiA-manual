# 공용 이미지 관리

`공용 이미지 관리`는 모든 프로젝트에서 공통으로 사용할 OS 이미지를 등록하고 관리하는 화면입니다. 관리자는 공용 이미지를 생성, 복제, 삭제해 프로젝트 사용자에게 표준 이미지를 제공합니다.

## 화면 개요
- `목록 화면`: 공용 이미지 현황을 조회하고 생성, 복제, 삭제 작업을 시작합니다.
- `생성 화면`: 스토리지 도메인과 이미지 파일을 지정해 새 공용 이미지를 등록합니다.
- `복제 모달`: 선택한 이미지를 다른 스토리지 도메인으로 복제합니다.

## 공용 이미지 목록
### 주요 작업
- `생성`: 새로운 공용 이미지를 생성합니다.
- `복제`: 선택한 1개 공용 이미지를 다른 스토리지 도메인으로 복제합니다.
- `삭제`: 선택한 1개 공용 이미지를 삭제합니다.

> 💡 **참고:** 목록은 단일 선택 방식으로 동작합니다.

### 검색
- **검색 조건**: `이름`
- **검색어 입력**: 검색어를 입력합니다.
- **검색 버튼**: 조건에 맞는 이미지를 조회합니다.

### 테이블 컬럼
| 컬럼 | 설명|
| --- |---|
| **이름** | 공용 이미지 이름입니다. 생성 시 사용한 이미지 파일 이름과 동일합니다. |
| **ID** | 공용 이미지 고유 ID입니다.|
| **OS 타입** | 공용 이미지의 OS 타입입니다.|
| **크기** | 공용 이미지 파일 크기입니다.|
| **스토리지 도메인** | 이미지가 저장된 스토리지 도메인입니다.|
| **생성 일시** | 공용 이미지를 등록한 시각입니다.|

## 공용 이미지 생성
### 기본 정보
- **스토리지 도메인**: 이미지가 저장될 스토리지 도메인을 선택합니다.
- **이미지 파일**: `.qcow2`, `.img` 형식의 이미지 파일을 업로드합니다.

### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
| --- | --- | --- | --- |
| **스토리지 도메인** | 현재 선택 가능한 스토리지 도메인 | 필수입니다. 1개를 선택합니다. | 저장 위치입니다. |
| **이미지 파일** | `.qcow2`, `.img` 파일 | 필수입니다. 파일을 선택해야 합니다. | 업로드 대상입니다. |

### 생성 시 참고 사항
- 스토리지 도메인과 이미지 파일을 모두 선택하면 `생성` 버튼이 활성화됩니다.
- 생성된 공용 이미지는 모든 프로젝트의 인스턴스 생성과 시작 템플릿 생성에 사용할 수 있습니다.

## 공용 이미지 복제
1. 목록에서 복제할 공용 이미지를 선택합니다.
2. `복제`를 클릭합니다.
3. 대상 스토리지 도메인을 선택합니다.
4. 복제를 실행합니다.

> ⚠️ **주의:** 원본 이미지와 동일한 스토리지 도메인은 복제 대상으로 선택할 수 없습니다.

## 공용 이미지 삭제
- 삭제는 한 번에 1개만 수행합니다.
- 삭제 전에 프로젝트에서 사용 중인 배포 표준인지 확인해야 합니다.
