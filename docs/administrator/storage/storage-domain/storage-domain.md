# 스토리지 도메인

**스토리지 도메인**은 이미지/블록/파일시스템 등 스토리지 리소스가 실제로 저장될 저장소를 정의하는 기능입니다.

## 스토리지 도메인 목록
### 주요 작업
- **`생성`**: 새 스토리지 도메인을 생성합니다.
- **`삭제`**: 선택한 1개 스토리지 도메인을 삭제합니다.
  - 연관 리소스가 있으면 삭제할 수 없습니다.

### 테이블 컬럼
| 컬럼 | 설명 |
|---|---|
| **스토리지 도메인 이름** | 스토리지 도메인 이름입니다. |
| **ID** | 스토리지 도메인 고유 ID입니다. |
| **스토리지 도메인 타입** | `LOCAL`, `GFS2`, `NFS`, `CEPH` |
| **루트 디렉토리** | 마운트 경로(루트 디렉토리)입니다. |
| **생성 일시** | 생성된 날짜와 시간입니다. |

## 스토리지 도메인 생성
### 입력 항목
- **이름**(필수)
- **스토리지 도메인 타입**(필수): `LOCAL`, `GFS2`, `NFS`, `CEPH`
- **물리 머신**(LOCAL 타입일 때 필수): 1개 선택
- **루트 디렉토리**(필수)

### 입력 필드 유효성
| 필드 | 규칙 |
|---|---|
| **이름** | 영문/숫자/`-`/`_`, 최대 80자 |
| **루트 디렉토리** | `/`로 시작, `/` 단독 불가, `//` 불가, 허용 문자: 영문/숫자/`/`/`.`/`-`/`_` |

## 스토리지 도메인 삭제
- 삭제 시 확인 모달에서 도메인 정보를 확인한 뒤 진행합니다.
- `삭제 가능(isDeletable)` 상태인 경우에만 삭제 버튼이 활성화됩니다.
- 일반적으로 연관 리소스(이미지, 볼륨 등)가 없는 경우에만 삭제할 수 있습니다.
