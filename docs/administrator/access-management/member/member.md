# 멤버

## 멤버 목록

### 검색
- 이름, 이메일, 휴대폰 번호로 멤버를 검색할 수 있습니다.
- 검색 조건을 변경하면 기존 선택은 해제됩니다.

### 주요 작업
- `생성`: 새 멤버를 등록합니다.
- `편집`: 선택한 멤버의 정보를 수정합니다. 한 명을 선택했을 때만 사용할 수 있습니다.
- `삭제`: 선택한 멤버를 삭제합니다. 한 명을 선택했을 때만 사용할 수 있으며, 슈퍼 관리자 `admin`은 삭제할 수 없습니다.
- `전체 임시 잠금 해제`: 현재 임시 잠금 상태인 멤버를 한 번에 해제합니다. 실행 전에 대상 목록과 확인 문구가 표시됩니다.
- `더보기 > 비밀번호 변경`: 선택한 멤버의 비밀번호를 변경합니다.
- `더보기 > 임시 잠금 해제`: 선택한 멤버의 임시 잠금을 해제합니다. 이미 정상 상태인 멤버를 선택한 경우에는 해제할 대상이 없다는 안내가 표시됩니다.
- `더보기 > OTP 재등록`: 선택한 멤버의 OTP를 초기화합니다.

### 테이블 컬럼
| 컬럼 | 설명 |
|---|---|
| 이름 | 클릭하면 멤버 상세 화면으로 이동합니다. |
| 타입 | 관리자 멤버인지, 프로젝트 멤버인지 표시합니다. |
| 이메일 | 멤버 이메일 주소를 표시합니다. |
| 휴대폰 번호 | 등록된 휴대폰 번호를 표시합니다. |
| 임시 잠금 | `임시 잠금` 또는 `정상` 상태를 표시합니다. |
| 생성 일시 | 멤버가 생성된 시각을 표시합니다. |

### 사용 시 참고
- 목록은 한 번에 한 명만 선택할 수 있습니다.
- `더보기` 메뉴는 멤버를 선택해야 활성화됩니다.

## 멤버 생성/편집

### 입력 항목
- 생성 화면: 이름, 비밀번호, 비밀번호 확인, 이메일, 관리자 여부를 입력합니다.
- 편집 화면: 이름, 이메일, 휴대폰 번호를 수정할 수 있습니다.
- `관리자 여부`를 켜면 관리자 멤버로, 끄면 프로젝트 멤버로 등록됩니다.
- 슈퍼 관리자 `admin`은 편집 화면에서 이름을 변경할 수 없습니다.

### 입력 규칙
- 이름: 영문 소문자, 숫자, 점(`.`), 언더스코어(`_`), 하이픈(`-`)만 사용할 수 있으며 2자 이상 20자 이하로 입력합니다.
- 비밀번호: 현재 비밀번호 정책에 맞게 입력해야 합니다.
- 비밀번호 확인: 비밀번호와 동일해야 합니다.
- 이메일: 올바른 이메일 형식으로 입력해야 합니다.
- 휴대폰 번호: 10자리 또는 11자리 숫자로 입력합니다.

### 저장이 되지 않는 경우
- 이미 사용 중인 이름이나 이메일을 입력한 경우
- 비밀번호가 현재 정책을 만족하지 않는 경우
- 이메일 또는 휴대폰 번호 형식이 올바르지 않은 경우

### 사용 시 참고
- 존재하지 않거나 접근할 수 없는 멤버는 편집 화면으로 이동할 수 없습니다.

## 멤버 상세

### 기본 정보
- 이름, ID, 타입, 휴대폰 번호, 이메일, 상태, 생성 일시를 확인할 수 있습니다.
- 프로젝트 멤버는 기본 정보에서 연결된 프로젝트 수를 함께 확인할 수 있습니다.
- 관리자 멤버는 기본 정보에서 현재 관리자 역할을 확인할 수 있으며, 여기서 역할 변경도 진행합니다.

### 역할 확인 방식
- 프로젝트 멤버: 하단 영역에 프로젝트별 역할 목록이 표시됩니다. `편집` 버튼을 눌러 프로젝트별 역할을 수정할 수 있습니다.
- 관리자 멤버: 하단 역할 목록 대신 기본 정보 영역에서 관리자 역할을 확인하고 변경할 수 있습니다.

### 사용 시 참고
- 슈퍼 관리자 `admin`은 관리자 역할을 변경할 수 없습니다.

## 멤버 역할 수정

### 화면 구성
- 프로젝트와 역할을 한 줄씩 짝지어 관리합니다.
- `추가` 버튼을 누르면 새 행이 추가됩니다.
- 각 행의 `삭제` 버튼으로 해당 프로젝트-역할 연결을 제거할 수 있습니다.
- 같은 프로젝트는 중복해서 선택할 수 없습니다.

### 저장 조건
- 모든 행에서 프로젝트와 역할을 모두 선택해야 합니다.
- 기존 값에서 변경된 내용이 있을 때만 `저장` 버튼이 활성화됩니다.

### 사용 시 참고
- 한 프로젝트에는 멤버 역할을 하나만 지정할 수 있습니다.
