# 멤버

## 멤버 목록

### 테이블 컬럼
| 컬럼 | 키 | 설명 |
|---|---|---|
| 이름 | `userName` | 클릭 시 멤버 상세 화면으로 이동합니다. |
| 유형 | `type` | 인증 유형에 맞게 포맷팅되어 표시됩니다. |
| 이메일 | `email` | 멤버 이메일을 표시합니다. |
| 연락처 | `phone` | 멤버 연락처를 표시합니다. |
| 임시잠금 | `disabled` | `임시잠금` 또는 `정상` 상태를 표시합니다. |
| 생성일 | `createdAt` | 로컬 시간 기준으로 변환해 표시합니다. |

### 검색
- 검색 필드: `이름(userName)`, `이메일(email)`, `연락처(phone)`
- 검색 조건이 바뀌면 기존 선택 상태는 초기화됩니다.
- 검색 실행 시 페이지는 1페이지로 이동합니다.

### 상단 액션 및 활성 조건
| 액션 | 동작 | 활성 조건 |
|---|---|---|
| 생성 | 생성 화면으로 이동 | 항상 활성 |
| 편집 | 선택 멤버 편집 화면 이동 | 1건 선택 |
| 삭제 | 선택 멤버 삭제 모달 오픈 | 1건 선택 + `admin` 제외 |
| 전체 임시 잠금 해제 | 잠금 사용자 일괄 해제 모달 오픈 | 항상 활성 |
| 더보기 > 비밀번호 변경 | 비밀번호 변경 모달 오픈 | 1건 선택 |
| 더보기 > 임시 잠금 해제 | 잠금 해제 모달 오픈 | 1건 선택 |
| 더보기 > OTP 재등록 | OTP 초기화 모달 오픈 | 1건 선택 |

### 모달 동작
- 비밀번호 변경: 관리자 비밀번호 변경 폼을 표시합니다.
- 임시 잠금 해제: 잠금 사용자 상세와 확인 문구를 표시합니다.
- 임시 잠금 해제(예외): 이미 잠금이 아닌 경우 안내 문구를 표시합니다.
- 전체 임시 잠금 해제: 잠금 사용자 목록을 스크롤 영역으로 표시합니다.
- 전체 임시 잠금 해제(빈 상태): 대상이 없으면 빈 안내 문구를 표시합니다.
- OTP 재등록: 대상 사용자 OTP 재설정 확인 문구를 표시합니다.
- 멤버 삭제: 대상 사용자명을 포함한 삭제 확인 문구를 표시합니다.

### 운영 제약
- 슈퍼 관리자 계정(`admin`)은 삭제할 수 없습니다.
- 목록은 단일 선택 방식으로 동작합니다.
- 더보기 액션은 선택 멤버가 있을 때만 활성화됩니다.

## 멤버 생성/편집

### 경로 및 권한
- 생성: `/administrator/access-management/member/create` (`member-edit`)
- 편집: `/administrator/access-management/member/update?id={memberId}` (`member-edit`)

### 입력 항목
| 구분 | 항목 |
|---|---|
| 공통 | 이름*(`username`), 이메일*(`email`) |
| 생성 모드 | 비밀번호*, 비밀번호 확인*, 관리자 여부 토글(`ADMIN/PROJECT`) |
| 편집 모드 | 연락처(`phone`) |

- 편집 화면에서는 비밀번호/비밀번호 확인/관리자 여부 입력을 제공하지 않습니다.
- 슈퍼 관리자(`admin`) 편집 시 이름 필드는 비활성 처리됩니다.

### 유효성 및 서버 오류
- 이름: 멤버명 규칙 검증
- 비밀번호: 현재 비밀번호 정책 기준 검증
- 비밀번호 확인: 비밀번호 일치 검증
- 이메일: 형식 검증
- 연락처: 형식 검증

서버 오류는 다음 케이스를 사용자 메시지로 매핑해 표시합니다.
- 이름 중복
- 이메일 중복 또는 이메일 형식 오류
- 비밀번호 정책 위반(이력, 패턴, 연속값 등)

### 예외 처리
- 편집 모드에서 상세 조회 실패 시 접근 오류 토스트 후 목록으로 이동합니다.

## 멤버 상세
- URL: `/administrator/access-management/member/detail?id={memberId}`
- 라우트 권한: `member-view`

### 기본 정보 영역
- 이름, ID, 유형, 연락처, 이메일, 상태, 생성일
- 프로젝트 멤버인 경우 프로젝트 개수(`pjtCnt`)를 표시합니다.
- 관리자 멤버인 경우 관리자 역할 정보를 표시하며, 역할 변경은 상세 화면 내 모달로 처리합니다.

### 하단 영역 분기
- `type !== ADMIN`: 프로젝트 역할 목록을 표시하고 `편집` 버튼으로 역할 수정 화면으로 이동합니다.
- `type === ADMIN`: 하단 역할 목록은 노출하지 않고 기본 정보 영역에서 관리자 역할 변경 액션을 제공합니다.

### 상세 화면 제약
- 슈퍼 관리자(`admin`)는 관리자 역할 변경 버튼이 비활성화됩니다.

## 멤버 역할 수정
- URL: `/administrator/access-management/member/role-update?id={memberId}`
- 라우트 권한: `member-edit`

### 화면 구성
- 컬럼: 프로젝트 / 역할 / 삭제
- 행 단위 드롭다운으로 프로젝트-역할 매핑을 편집합니다.
- `추가` 버튼으로 행을 늘릴 수 있습니다.
- 동일 프로젝트 중복 선택은 허용되지 않습니다.
- 프로젝트 드롭다운 데이터는 전체 프로젝트 목록을 사용합니다.

### 저장 활성 조건
- 모든 행에서 프로젝트/역할이 모두 유효해야 합니다.
- 기존 데이터 대비 변경사항이 있어야 저장 버튼이 활성화됩니다.
