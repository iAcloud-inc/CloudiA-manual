# 정책 설정

## 비밀번호 정책

### 주요 항목
- 커스텀 비밀번호 정책(`customPwdPolicy`) 토글: 조회 전용, UI 비활성
- 패턴 정책(`patternPolicy`) 토글: 조회 전용, UI 비활성
- 최소 길이
- 최대 길이(`maximumLength`): UI 비활성
- 대문자 최소 개수
- 소문자 최소 개수
- 숫자 최소 개수
- 특수문자 최소 개수
- 비밀번호 이력 개수

### 유효성
- 최소 길이: `9` 이상
- 최대 길이: `20` 이하
- 대문자/소문자/숫자/특수문자/이력: `1~5`

### 업데이트 버튼 활성 조건
- 조회 성공 상태
- 모든 필드 검증 통과
- 기존 값 대비 변경사항 존재
- 업데이트 API 요청 중 아님

## 계정 잠금 정책

### 주요 항목
- 잠금 모드(`bruteForceMode`)
  - 현재 UI 값: `lockoutTemporarily(temporary)` 1개
  - 모드 드롭다운은 비활성
- 최대 로그인 실패 횟수
- 최대 대기 시간 증가치(`wait increment`)
- 최대 대기 시간(`max wait`)
- 실패 횟수 초기화 시간(`failure reset`)
- 시간 단위 드롭다운: `seconds` 고정 표시(비활성)

### 유효성
- 최대 로그인 실패 횟수: 최대 `5`
- wait increment: `300~1200`
- max wait: `300~1200`
- failure reset: `maxWait` 이상

### 업데이트 조건
- 변경사항이 있어야 저장 가능합니다.

## OTP 설정

### 주요 항목
- 활성화 상태 토글
- 해시 알고리즘
  - SHA1(`HmacSHA1`)
  - SHA256(`HmacSHA256`)
  - SHA512(`HmacSHA512`)
- OTP 자리수: `6` 또는 `8`
- 지원 애플리케이션 목록(읽기 전용)

### 동작 규칙
- 비활성 상태에서는 알고리즘/자리수/지원 앱 섹션을 숨깁니다.
- 활성 상태에서 변경사항이 있을 때만 업데이트 가능합니다.
- 상단 안내: TOTP(시간 기반 OTP)만 지원합니다.

## 세션 설정

### 주요 항목
- 자동 로그아웃 시간(숫자 입력)
- 단위 선택(`seconds` 또는 `minutes`)
- API 저장 시 `idle-time(ms)` 값으로 변환해 전송

### 유효성
- 단위가 초(`seconds`)일 때 최소 `60`
- 단위가 분(`minutes`)일 때 최소 `1`
- 변경사항 + 검증 통과 시 업데이트 가능
- 경고 툴팁 임계치: `5분`
