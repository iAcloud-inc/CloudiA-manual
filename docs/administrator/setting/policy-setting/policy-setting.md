# 정책 설정

## 비밀번호 정책

### 주요 항목
- `커스텀 비밀번호 정책(customPwdPolicy)` 토글은 조회 전용이며 UI에서 비활성화되어 있습니다.
- `패턴 정책(patternPolicy)` 토글은 조회 전용이며 UI에서 비활성화되어 있습니다.
- 최소 길이
- 최대 길이: `maximumLength`, UI 비활성
- 대문자 최소 개수
- 소문자 최소 개수
- 숫자 최소 개수
- 특수문자 최소 개수
- 비밀번호 이력 개수

### 입력 규칙
- 최소 길이는 `9` 이상이어야 합니다.
- 최대 길이는 `20` 이하여야 합니다.
- 대문자, 소문자, 숫자, 특수문자, 비밀번호 이력 개수는 각각 `1`부터 `5` 사이여야 합니다.

### 업데이트 조건
- 정책 조회가 성공한 상태여야 합니다.
- 모든 필드가 검증을 통과해야 합니다.
- 기존 값 대비 변경사항이 있어야 합니다.
- 업데이트 API 요청 중이 아니어야 합니다.

## 계정 잠금 정책

### 주요 항목
- 잠금 모드: `bruteForceMode`
- 최대 로그인 실패 횟수
- 최대 임시 잠금 횟수: 현재 UI에서는 사용하지 않습니다.
- 최대 대기 시간 증가치: `wait increment`
- 최대 대기 시간: `max wait`
- 실패 횟수 초기화 시간: `failure reset`
- 시간 단위 드롭다운: `seconds` 고정, UI 비활성

### 입력 규칙
- 잠금 모드는 현재 `lockoutTemporarily(temporary)` 한 가지 값만 표시됩니다.
- 최대 로그인 실패 횟수는 최대 `5`까지 입력할 수 있습니다.
- `wait increment`는 `300`부터 `1200` 사이여야 합니다.
- `max wait`는 `300`부터 `1200` 사이여야 합니다.
- `failure reset`은 `max wait` 이상이어야 합니다.

### 업데이트 조건
- 기존 값 대비 변경사항이 있어야 저장할 수 있습니다.

### 사용 시 참고
- 잠금 모드 드롭다운은 비활성화되어 있어 변경할 수 없습니다.
- 현재 UI에서는 임시 잠금 정책만 편집할 수 있습니다.

## OTP 설정

### 주요 항목
- 활성화 상태 토글
- 해시 알고리즘: `HmacSHA1`, `HmacSHA256`, `HmacSHA512`
- OTP 자리수: `6`, `8`
- 지원 애플리케이션 목록: 읽기 전용

### 동작 규칙
- 비활성 상태에서는 알고리즘, 자리수, 지원 애플리케이션 섹션이 숨겨집니다.
- 활성 상태에서는 변경사항이 있을 때만 업데이트할 수 있습니다.

### 사용 시 참고
- 상단 안내 문구로 TOTP 기반 OTP만 지원한다는 설명이 표시됩니다.
- 지원 애플리케이션 목록은 조회 전용 라벨로만 표시됩니다.

## 세션 설정

### 주요 항목
- 자동 로그아웃 시간
- 시간 단위 선택: `seconds`, `minutes`
- API 저장 시 `idle-time(ms)` 값으로 변환하여 전송합니다.

### 입력 규칙
- 단위가 `seconds`일 때는 최소 `60` 이상이어야 합니다.
- 단위가 `minutes`일 때는 최소 `1` 이상이어야 합니다.

### 업데이트 조건
- 기존 값 대비 변경사항이 있어야 합니다.
- 모든 입력값이 검증을 통과해야 합니다.

### 사용 시 참고
- 경고 툴팁은 `5분`을 기준으로 표시됩니다.
- 기존 저장값이 `60초` 이상이면 화면에는 분 단위로 환산해 표시합니다.
