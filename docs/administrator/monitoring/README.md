# 모니터링

관리자 `모니터링` 메뉴는 Cloud:iA 전체 환경에서 발생한 이벤트, 감사 로그, 이벤트 트리거를 통합해 확인하는 영역입니다. 장애 대응과 운영 추적, 알림 자동화에 사용합니다.

## 구성 메뉴
- **이벤트**: 프로젝트 전반에서 발생한 이벤트를 조회합니다.
- **감사 로그**: 사용자와 시스템의 주요 요청 이력을 추적합니다.
- **이벤트 트리거**: 특정 조건의 이벤트를 감지해 알림 채널로 전달하는 규칙을 설정합니다.

## 권장 작업 
1. 이상 징후가 발생하면 `이벤트`에서 최근 이벤트를 확인합니다.
2. 누가 어떤 작업을 수행했는지 확인해야 하면 `감사 로그`를 확인합니다.
3. 반복적으로 감시해야 하는 조건은 `이벤트 트리거`로 자동화합니다.

## 참고 사항
- `이벤트`와 `감사 로그`는 모두 조회 전용입니다.
- 이벤트 트리거를 사용하려면 `알림 설정` 메뉴의 SMTP 또는 채널 구성이 선행되어야 합니다.
