# 클러스터

**클러스터**는 쿠버네티스 컨트롤 플레인, 워커 노드, 네트워크, 스토리지, 리소스를 통합 관리하는 기능입니다.

## 클러스터 목록
클러스터 목록을 확인합니다.

### 주요 작업
- `생성`: 클러스터 생성 페이지로 이동합니다.
- `삭제`: 선택한 클러스터를 삭제합니다.

> 💡 **공통 조건:** 목록은 **단일 선택** 방식입니다. 상태에 따라 버튼이 자동으로 비활성화될 수 있습니다.

### 테이블 컬럼
| 컬럼 | 설명 |
|------|------|
| **이름** | 클러스터 이름입니다. 클릭하면 상세 화면으로 이동합니다. |
| **ID** | 클러스터의 고유 ID입니다. |
| **버전** | 클러스터의 쿠버네티스 버전입니다. |
| **엔드포인트 IP** | API 서버 접근에 사용하는 엔드포인트 IP 주소입니다. |
| **상태** | 예: `UP`, `DOWN`, `ERROR_DOWN`, `CREATING`, `DELETING`, `POWERING_UP`, `PENDING`, `CA_ROTATING` |
| **컨트롤 플레인** | 정상(`READY`) 컨트롤 플레인 수/전체 수를 `N/N` 형식으로 표시합니다. |
| **워커 노드** | 정상(`READY`) 워커 수/전체 수를 `N/N` 형식으로 표시합니다. |
| **생성 일시** | 생성 일시를 표시합니다. |

> ⚠️ **주의:** 기본 정렬은 `createdAt` 기준 내림차순입니다. `삭제 중(DELETING)` 상태의 클러스터는 삭제 확인창에서 다시 삭제할 수 없습니다.

## 클러스터 상세
선택한 클러스터의 기본 정보와 하위 탭을 확인합니다.

### 상단 요약
- **이름**: 클러스터 이름입니다.
- **ID**: 클러스터의 고유 ID입니다.
- **버전**: 쿠버네티스 버전입니다.
- **VPC**: `VPC 이름 (ID: VPC ID)` 형식으로 표시됩니다.
- **IP**: 엔드포인트 IP 주소입니다.
- **상태**: 클러스터 상태입니다.
- **생성 일시**: 생성 일시입니다.

### 주요 작업
- `Config 추출`: 서비스 계정(Service Account) 이름과 토큰(Token)을 입력해 kubeconfig 파일을 다운로드합니다.
- `CA 파일 재설정`: 새 `ca.crt`와 `ca.key`를 업로드해 CA를 재설정합니다.

> ⚠️ **주의:** CA 파일 재설정은 백엔드 기준으로 **모든 노드가 `READY` 상태일 때만** 수행할 수 있습니다.

> ⚠️ **주의:** CA를 재설정하면 기존 kubeconfig 기반 연결을 다시 내려받아야 할 수 있습니다.

#### Config 추출
- 입력 항목은 **서비스 계정(Service Account)**, **토큰(Token)** 2개입니다.
- 두 값이 모두 있어야 `추출` 버튼이 활성화됩니다.
- 다운로드 파일명은 `config`입니다.
- 토큰 입력란은 최대 `1000`자까지 입력할 수 있습니다.

#### CA 파일 재설정
- `ca.crt`, `ca.key`가 모두 선택되어야 저장할 수 있습니다.
- 재설정 요청 후 상태가 `CA_ROTATING`으로 표시될 수 있으며, 완료되면 다시 `UP`으로 복귀합니다.

### 하단 탭
상세 화면 하단에서는 `모니터링`, `컴퓨트`, `워커 오토 스케일링 그룹`, `네트워크`, `스토리지`, `리소스` 탭을 확인할 수 있습니다.

> ⚠️ **주의:** `CREATING` 상태에서는 `모니터링`, `스토리지`, `리소스` 탭이 비활성화됩니다.

#### 모니터링 탭
클러스터 상태와 사용량을 모니터링합니다.

##### 주요 작업
- `새로고침`: 즉시 최신 데이터를 다시 조회합니다.
- `자동 새로고침`: 주기를 설정해 반복 조회할 수 있습니다.

##### 주요 표시 영역
- **컨트롤 플레인 평균 부하**: CPU, Memory, Disk 사용률
- **파드 상태**: `Running`, `Pending`, `Succeed`, `Failed`, `Unknown`
- **워커 노드 사용량**: CPU, Memory, `Network rx`, `Network tx`
- **노드 상태**: 컨트롤 플레인/워커의 정상 수와 전체 수

> 💡 **참고:** 워커 메트릭에 `CPU Throttling` 또는 `OOM Risk`가 감지되면 경고 문구가 함께 표시될 수 있습니다.

#### 컴퓨트 탭
컨트롤 플레인과 워커 노드 목록을 확인하고, 워커 노드 관련 작업을 수행합니다.

##### 주요 작업
- `실행`: 선택한 노드가 `NOT_READY` 또는 `MACHINE_DOWN`일 때 활성화됩니다.
- `워커 노드 추가`: 클러스터 상태가 `UP`일 때만 활성화됩니다.
- `워커 노드 삭제`: 워커 노드 선택 시 사용할 수 있습니다.

##### 테이블 컬럼
| 컬럼 | 설명 |
|------|------|
| **역할** | `컨트롤 플레인` 또는 `워커 노드`입니다. |
| **노드 이름** | 클릭 시 VM 인스턴스 상세로 이동합니다. |
| **ID** | 노드의 고유 ID입니다. |
| **버전** | 쿠버네티스 버전입니다. |
| **기본 인터페이스 IP 주소** | 노드 IP 주소입니다. |
| **인스턴스 타입** | 연결된 인스턴스 타입입니다. |
| **상태** | 예: `READY`, `NOT_READY`, `UNKNOWN`, `PENDING`, `CREATING`, `DELETING`, `ERROR_DOWN`, `CA_ROTATING` |
| **생성 일시** | 생성 일시입니다. |

> ⚠️ **주의:** 컨트롤 플레인 노드는 워커 삭제 기능으로 제거할 수 없습니다. `PENDING`, `CREATING`, `DELETING` 상태의 워커 노드는 삭제할 수 없습니다.

> 💡 **참고:** 노드 이름 클릭은 현재 상태가 `CREATING`, `DELETING`, `ERROR_DOWN`이 아닌 경우에만 허용됩니다.

#### 워커 오토 스케일링 그룹 탭
워커 오토 스케일링 그룹 목록을 확인하고 관리합니다.

##### 주요 작업
- `생성`: 워커 오토 스케일링 그룹 생성 페이지로 이동합니다.
- `삭제`: 목록에서 선택한 그룹 1개를 삭제합니다.

##### 테이블 컬럼
| 컬럼 | 설명 |
|------|------|
| **이름** | 그룹 이름입니다. 클릭하면 상세 화면으로 이동합니다. |
| **ID** | 그룹의 고유 ID입니다. |
| **인스턴스 상태** | `READY/전체` 형식으로 표시합니다. |
| **상태** | 예: `UP`, `CREATING`, `DELETING`, `SCALING_OUT`, `SCALING_IN`, `UPDATING`, `SUSPENDED`, `DOWN` |
| **생성 일시** | 생성 일시입니다. |

> 💡 **참고:** 목록은 단일 선택 방식이며, 페이지당 `5개`씩 표시됩니다. URL 쿼리스트링에 `page`, `sort`가 반영됩니다.

#### 네트워크 탭
클러스터에 연결된 네트워크 리소스를 조회합니다.

##### 테이블 컬럼
| 컬럼 | 설명 |
|------|------|
| **서브넷** | `서브넷 이름 (ID: 서브넷 ID)` 형식으로 표시되며, 클릭하면 서브넷 상세로 이동합니다. |
| **보안그룹** | 연결된 보안그룹 목록입니다. 각 항목을 클릭하면 보안그룹 상세로 이동합니다. |
| **로드밸런서** | `로드밸런서 이름 (ID: 로드밸런서 ID)` 형식으로 표시되며, 클릭하면 로드밸런서 상세로 이동합니다. |

> 💡 **참고:** 현재 UI 기준으로 VPC와 엔드포인트 IP는 네트워크 탭이 아니라 상세 상단 요약에 표시됩니다.

> 💡 **참고:** 클러스터 생성 시 기본 보안그룹은 자동 생성되며, 추가 보안그룹을 선택해 생성한 경우 네트워크 탭의 보안그룹 목록에 함께 표시됩니다.

#### 스토리지 탭
스토리지 클래스와 PVC를 전환하여 조회하고 생성합니다.

##### 주요 작업
- `스토리지 클래스 생성`
- `PVC 생성`

> 💡 **참고:** 현재 상세 화면은 조회/생성 중심이며, 프론트엔드에는 스토리지 클래스/PVC 삭제 버튼이 노출되지 않습니다.

##### 스토리지 클래스 목록 컬럼
| 컬럼 | 설명 |
|------|------|
| **이름** | 스토리지 클래스 이름입니다. |
| **경과 시간** | 생성 후 경과 시간(`age`)입니다. |
| **볼륨 확장 허용** | `allowVolumeExpansion` 여부입니다. |
| **서버** | `parameters.server` 값입니다. |
| **공유 경로** | `parameters.share` 값입니다. |
| **프로비저너** | `provisioner` 값입니다. |
| **반환 정책** | `Delete` 또는 `Retain`입니다. |
| **볼륨 바인딩 모드** | `volumeBindingMode` 값입니다. |
| **기본** | 기본 스토리지 클래스 여부입니다. |

##### PVC 목록 컬럼
| 컬럼 | 설명 |
|------|------|
| **이름** | PVC 이름입니다. |
| **경과 시간** | 생성 후 경과 시간(`age`)입니다. |
| **상태** | 예: `Pending`, `Bound` |
| **스토리지 클래스** | 연결된 스토리지 클래스입니다. |
| **볼륨** | 바인딩된 볼륨입니다. |
| **설정 용량** | 요청 용량입니다. |
| **접근 모드** | 접근 모드입니다. |

##### 생성 동작
- **스토리지 클래스 생성**: 이름, 파일 시스템, 반환 정책을 입력합니다.
- **PVC 생성**: 이름, 용량, 스토리지 클래스, 네임스페이스, 접근 모드를 입력합니다.

> 💡 **참고:** 스토리지 클래스 생성 시 파일 시스템은 선택한 클러스터와 동일 VPC에서 조회 가능한 항목이어야 합니다.

> 💡 **참고:** PVC 생성 시 용량은 최소 `1 MiB` 이상이어야 하며, 테스트 케이스 기준으로 네임스페이스 드롭다운에는 `default` 타입 네임스페이스만 노출됩니다.

#### 리소스 탭
클러스터 리소스를 드롭다운으로 전환해 조회합니다.

##### 조회 대상
- `시스템 파드`
- `디플로이먼트`
- `서비스`
- `데몬셋`
- `이벤트`

##### 표시 항목
- **시스템 파드**: 이름, 네임스페이스, Phase, 노드 이름, Pod IP, Host IP, 재시작 횟수, 경과 시간, Ready, 상태, QoS Class
- **디플로이먼트**: 이름, 네임스페이스, Replicas(`readyReplicas/replicas`), 경과 시간, Ready, 상태, Strategy
- **서비스**: 이름, 네임스페이스, Type, Cluster IP, External IP, Ports, 경과 시간, 상태
- **데몬셋**: 이름, 네임스페이스, Desired Nodes, Available Nodes, 경과 시간, Ready Pods, 상태, Update Strategy
- **이벤트**: 경과 시간, Type, 네임스페이스, Reason, Message

> ⚠️ **주의:** 리소스 탭은 조회 전용입니다.

## 클러스터 생성
클러스터 생성 화면에서 네트워크, 레지스트리, 컨트롤 플레인, 인증 정보를 입력해 새 클러스터를 생성합니다.

> 💡 **참고:** 생성 버튼은 필수 항목이 모두 유효하고, 선택한 VPC에 정상(`healthy`) 상태의 쿠버네티스 부트스트랩 레지스트리가 있을 때 활성화됩니다.

### 1. 기본 정보
- **클러스터 이름**을 입력합니다.
- 이름은 소문자, 숫자, 하이픈(`-`)을 사용할 수 있습니다.
- 최대 길이는 `40자`입니다.
- 숫자로 시작할 수 없습니다.

### 2. VPC 및 서브넷 설정
- **VPC**를 먼저 선택합니다.
- 선택한 VPC를 기준으로 **서브넷**과 **추가 보안그룹** 목록이 갱신됩니다.
- **추가 보안그룹**은 선택 사항입니다.
- 추가 보안그룹은 최대 `4개`까지 선택할 수 있습니다.
- 기본 보안그룹은 클러스터 생성 시 자동 생성됩니다.

> 💡 **참고:** 현재 프론트엔드 코드는 서브넷 드롭다운을 `PUBLIC_SUBNET` 기준으로 조회합니다. 테스트 케이스에는 프라이빗 서브넷 생성 시나리오도 있으므로, 실제 배포 정책과 화면 동작을 함께 확인하십시오.

### 3. 레지스트리 확인
- 선택한 VPC에 정상(`healthy`) 상태의 쿠버네티스 부트스트랩 레지스트리가 있어야 합니다.
- 레지스트리가 없으면 `해당 VPC에 레지스트리가 없으니 생성하세요.` 안내가 표시됩니다.
- 이 경우 `레지스트리 생성` 버튼으로 별도 생성 모달을 열 수 있습니다.
- 레지스트리 생성이 완료되어 `해당 VPC에 레지스트리가 존재합니다.` 상태가 되어야 클러스터 생성이 가능합니다.

### 4. 컨트롤 플레인 설정
- **인스턴스 구성**을 선택합니다.
- **버전**을 선택합니다.
- 실제 프로비저닝은 선택한 버전에 매핑된 최신 활성 이미지 기준으로 수행됩니다.
- **컨트롤 플레인 개수**는 현재 UI 기준으로 `1` 또는 `3`만 선택할 수 있습니다.
- 선택 가능한 개수는 서브넷의 사용 가능 IP 수에 따라 비활성화될 수 있습니다.
- **부팅 디스크 크기**는 버전 선택 폼에서 함께 관리됩니다.

### 5. 사용자 계정 및 인증 방식
- **사용자 이름**을 입력합니다.
- 인증 방식은 비밀번호 또는 SSH 키를 사용할 수 있습니다.
- 비밀번호 사용 시 확인 값까지 일치해야 합니다.

### 6. CA 파일 설정
- **CA 인증서(`ca.crt`)**
- **개인 키(`ca.key`)**

두 파일이 모두 있어야 생성할 수 있습니다.

#### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
|------|------|------|------|
| **클러스터 이름** | 소문자, 숫자, `-` | 최대 `40자`, 숫자로 시작할 수 없음 | **필수** |
| **VPC** | 프로젝트 내 VPC 1개 | 선택 필요 | **필수** |
| **서브넷** | 선택한 VPC의 서브넷 | 현재 프론트엔드 기준 `PUBLIC_SUBNET` 목록 조회 | **필수** |
| **추가 보안그룹** | 선택한 VPC의 보안그룹 | 최대 `4개` 선택 가능 | 선택 |
| **레지스트리 상태** | `healthy` 레지스트리 | 정상 레지스트리 없으면 생성 버튼 비활성화 | **필수** |
| **버전** | 활성 쿠버네티스 버전 | 선택한 버전에 매핑된 최신 활성 이미지 사용 | **필수** |
| **컨트롤 플레인 개수** | `1` 또는 `3` | 사용 가능 IP 수에 따라 비활성화 가능 | **필수** |
| **CA 인증서/개인 키** | 파일 업로드 | 두 파일 모두 필요 | **필수** |
| **사용자 계정/인증 방식** | 사용자명 + 비밀번호 또는 SSH 키 | 유효성 검사 통과 필요 | **필수** |

### 생성 결과
- 생성 요청 직후 목록에 새 클러스터가 `CREATING` 상태로 표시됩니다.
- 완료되면 `UP` 상태로 전환됩니다.
- 테스트 케이스 기준으로 다음 리소스가 함께 생성됩니다.
  - `{클러스터 이름}-cp-xxxxxxxx` 형식의 컨트롤 플레인 인스턴스
  - `{클러스터 이름}-kubernetes` 형식의 타겟 그룹
  - `{클러스터 이름}-kubernetes` 형식의 로드밸런서

> ⚠️ **주의:** 서버 측에서는 프로젝트 쿼터(`vCPU`, 메모리, 디스크), 서브넷 사용 가능 IP 수, 이름 형식, 버전/인스턴스/파일 입력값을 함께 검증합니다.

## 워커 노드 추가
컴퓨트 탭에서 워커 노드를 추가합니다.

> 💡 **참고:** 클러스터 상태가 `UP`일 때만 `워커 노드 추가` 버튼이 활성화됩니다.

### 1. 기본 정보
- 인스턴스 구성을 선택합니다.
- 워커 노드 개수를 입력합니다.
- 이미지 디스크 크기를 입력합니다.

### 2. 사용자 계정
- 사용자 계정과 인증 방식을 입력합니다.
- 비밀번호 또는 SSH 키를 사용할 수 있습니다.

#### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
|------|------|------|------|
| **워커 노드 개수** | 정수 | 최소 `1`, 최대 `min(5, 사용 가능 IP 수)` | **필수** |
| **인스턴스 구성** | 인스턴스 타입 또는 사양 | 유효한 값 선택 필요 | **필수** |
| **이미지 디스크 크기** | 정수 | 시스템 정책 범위 내 입력 | **필수** |
| **사용자 계정/인증 방식** | 사용자명 + 비밀번호 또는 SSH 키 | 유효성 검사 통과 필요 | **필수** |

> 💡 **참고:** 워커 노드는 클러스터와 같은 쿠버네티스 버전을 따라 생성되며, 실제 프로비저닝은 현재 클러스터 버전에 매핑된 최신 활성 이미지 기준으로 수행됩니다.

> 💡 **참고:** 테스트 케이스 기준으로 생성 후 상태는 `생성 중` 계열 상태를 거쳐 `READY`로 전환됩니다.

## 워커 노드 삭제
컴퓨트 탭에서 워커 노드를 선택한 뒤 삭제할 수 있습니다.

- 컨트롤 플레인 노드는 삭제 대상이 아닙니다.
- `PENDING`, `CREATING`, `DELETING` 상태의 워커 노드는 삭제할 수 없습니다.

## 워커 오토 스케일링 그룹 상세
워커 오토 스케일링 그룹의 상세 정보와 하위 노드 목록을 확인합니다.

### 상단 요약
- **이름**: 그룹 이름입니다.
- **클러스터 ID**: 소속 클러스터의 고유 ID입니다.
- **상태**: 그룹 상태입니다.
- **인스턴스 상태**: `READY/전체` 형식으로 표시됩니다.
- **시작 템플릿**: 시작 템플릿 이름과 ID입니다.
- **Cloud-init**: 그룹에 적용된 Cloud-init 정보입니다.
- **타겟 그룹**: 연결된 타겟 그룹입니다.
- **그룹 정보**: `scaleMin`, `scaleMax`, `scaleDelaySec` 정보입니다.
- **생성 일시**: 생성 일시입니다.
- **설명**: 그룹 설명입니다.

### 하단 노드 목록
하단에는 해당 그룹에 속한 워커 노드 목록이 표시됩니다. 컬럼 구성은 컴퓨트 탭과 동일한 계열입니다.

## 워커 오토 스케일링 그룹 생성
워커 오토 스케일링 그룹 생성 화면에서 시작 템플릿, 타겟 그룹, 그룹 정보, 조정 정책을 입력합니다.

### 1. 기본 정보
- **이름**
- **설명**(선택)

### 2. 시작 템플릿
- 인스턴스 구성
- 부팅 디스크 크기
- 사용자 계정 및 인증 방식

### 3. 타겟 그룹 설정
- 타겟 그룹 생성 여부
- 생성 시 프로토콜, 포트 등 타겟 그룹 정보

### 4. 그룹 정보
- 초기 노드 수(`initialSize`)
- 최소 노드 수(`scaleMin`)
- 최대 노드 수(`scaleMax`)
- 스케일 간격(`scaleDelaySec`)

### 5. 조정 정책
- 지표(예: CPU)
- 스케일 인 대상값
- 스케일 아웃 대상값

#### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
|------|------|------|------|
| **이름** | 문자열 | 이름 규칙을 만족해야 함 | **필수** |
| **인스턴스 구성** | 인스턴스 타입 또는 사양 | 유효한 값 선택 필요 | **필수** |
| **부팅 디스크 크기** | 정수 | 시스템 정책 범위 내 입력 | **필수** |
| **사용자 계정/인증 방식** | 사용자명 + 비밀번호 또는 SSH 키 | 유효성 검사 통과 필요 | **필수** |
| **타겟 그룹 설정** | 생성 여부 + 설정값 | 생성 사용 시 프로토콜/포트 등 입력 필요 | 조건부 필수 |
| **그룹 정보** | 초기/최소/최대 노드 수, 스케일 간격 | 일반적으로 `scaleMin <= initialSize <= scaleMax` | **필수** |
| **조정 정책** | 지표, 스케일 인/아웃 대상값 | 스케일 아웃 값이 스케일 인 값보다 커야 함 | **필수** |

> 💡 **참고:** 워커 오토 스케일링 그룹은 별도 쿠버네티스 버전이나 보안그룹을 직접 입력하지 않습니다. 클러스터의 VPC, 서브넷, 보안그룹 구성을 상속합니다.

> 💡 **참고:** 워커 오토 스케일링 그룹 노드도 클러스터 버전에 매핑된 최신 활성 이미지 기준으로 생성되며, 현재 구현은 생성 시 `isAutoScaling=true`로 저장됩니다.
