# SSH 키

## 화면 개요

SSH 키 화면에서는 프로젝트에 등록된 SSH 키를 조회하고 등록, 편집, 삭제와 연결 상태 확인을 수행합니다.

## SSH 키 목록
SSH 키 목록을 확인합니다.

### 주요 작업
- `등록`: 새 SSH 키를 추가합니다. (항상 활성화)
- `편집`: 선택한 **1개** SSH 키를 수정합니다. 선택 시 활성화됩니다.
- `삭제`: 선택한 **1개** SSH 키를 삭제합니다. 선택 시 활성화되며, 실제 삭제 가능 여부는 연결된 인스턴스의 키 구성 조건을 따릅니다.

> 💡 **공통 조건:** 목록은 라디오 버튼 기반 **단일 선택**만 허용합니다.

### 검색
- **검색 조건**: 드롭다운에서 `이름`을 선택합니다.
- **검색어 입력**: 검색어를 입력합니다. (플레이스홀더: `검색어를 입력해주세요.`)
- **검색 버튼**: 검색 버튼을 클릭하여 검색을 실행합니다.

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **이름** | SSH 키 이름입니다. 클릭 시 상세 화면으로 이동합니다. |
| **ID** | SSH 키 고유 ID입니다. |
| **지문** | 공개키 Fingerprint입니다. |
| **연결된 인스턴스 수** | 해당 SSH 키가 적용된 인스턴스 수입니다. |
| **생성 일시** | SSH 키를 등록한 일시입니다. |

> ⚠️ **주의:** `연결된 인스턴스 수`가 1 이상이어도 항상 삭제 불가인 것은 아닙니다. 실제 삭제 가능 여부는 각 인스턴스에 남는 SSH 키 개수 조건으로 판단됩니다.

## SSH 키 등록·편집
> 💡 **참고:** 등록과 편집은 동일한 양식을 사용하며, **편집 모드**에서는 공개키 수정이 불가능합니다.

### 기본 정보
- **이름***: SSH 키 식별 이름을 입력합니다.
  - 입력 힌트: `영문 대/소문자, 숫자, -, _ 만 허용하며 최대 80자까지 입력 가능합니다.`
- **공개키***: OpenSSH 형식의 공개키 문자열을 입력합니다.
  - 글자 수 카운트: `0/10240` (편집 시 기존 길이 기준으로 표시)
  - *(편집)* 공개키는 읽기 전용으로 표시됩니다.

### 하단 버튼
- 등록 화면: `취소`, `등록` (유효성 통과 전 `등록` 비활성화)
- 편집 화면: `취소`, `편집` (유효성 통과 전 `편집` 비활성화)

### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
| --- | --- | --- | --- |
| **이름** | 영문 대/소문자, 숫자, `-`, `_` | 최대 80자, 공백 미허용 | **필수** |
| **공개키** | `<type> <base64> [comment]` 단일 행 | Type: `ssh-ed25519`, `ssh-rsa`<br/>Base64: 영문자/숫자/`+`/`/`/`=` 허용<br/>길이: 최대 10,240자 | 등록 시 입력, 편집 시 읽기 전용 |

## SSH 키 상세

### 상세 정보
- **ID**: SSH 키 고유 ID입니다.
- **이름**: SSH 키 이름입니다.
- **지문(Fingerprint)**: 공개키 Fingerprint입니다.
- **공개키**: `보기` 버튼을 통해 전체 공개키를 모달(`공개키 상세`)에서 확인합니다.
- **연결된 인스턴스 수**: 현재 키가 적용된 인스턴스 수를 확인합니다.
- **생성 일시**: 등록된 날짜/시간입니다.
- **수정 일시**: 마지막 수정된 날짜/시간입니다.

### 연결된 인스턴스
- 하단 `연결된 인스턴스` 영역에서 연결된 인스턴스 목록을 확인합니다.
- 테이블 컬럼:
  - 이름
  - ID
  - 생성 방식
  - IP 주소
  - 상태
  - vCPU
  - 메모리
  - 생성 일시
- 데이터가 없으면 `리스트 데이터가 존재하지 않습니다.`가 표시됩니다.

> 💡 **참고:** 인스턴스에 SSH 키 연결은 `인스턴스 생성/편집`을 통해 제공합니다.

## SSH 키 삭제
1. **SSH 키 선택:** 목록에서 삭제하려는 SSH 키를 선택한 뒤 **`삭제`** 버튼을 클릭합니다.
2. **삭제 가능 여부 확인:** 삭제 모달에서 삭제 가능 여부에 따라 다음 중 하나로 표시됩니다.
   - **삭제 가능:** `<키 이름> 삭제하시겠습니까?` + `취소`, `삭제` 버튼(삭제 활성)
   - **삭제 불가능:** `삭제 불가능한 SSH 키` + `취소`, `삭제` 버튼(삭제 비활성)
3. **삭제 실행:** 삭제 가능한 경우 `삭제` 버튼을 눌러 진행합니다.

### 삭제 가능/불가능 기준
- **삭제 가능:** 키가 연결된 인스턴스가 있더라도, 해당 인스턴스마다 **다른 SSH 키가 1개 이상 더 연결**되어 있는 경우
- **삭제 불가능:** 연결된 인스턴스 중 하나라도 해당 SSH 키가 **유일한 1개 키**인 경우

> ⚠️ **주의:** `연결된 인스턴스 수`는 참고 값이며, 최종 삭제 가능 여부는 인스턴스별 SSH 키 구성 조건에 따라 결정됩니다.

## 인스턴스에 SSH 키 적용
- 인스턴스 **생성/편집** 화면 ▸ `사용자 계정 ▸ SSH 키`에서 사전 등록된 SSH 키를 다중 선택할 수 있습니다.
- 드롭다운에서 최대 **5개** 키까지만 선택 가능합니다.
- 선택된 SSH 키는 인스턴스 Cloud-init 설정에 포함되어, 부팅 시 지정한 계정의 `~/.ssh/authorized_keys`에 반영됩니다.

> 💡 **참고:** Cloud-init 사용자 비밀번호와 SSH 키는 `인증 방식` 내에서 병행 설정할 수 있으며, SSH 키를 사용하려면 해당 프로젝트에 먼저 키를 등록해야 합니다.
