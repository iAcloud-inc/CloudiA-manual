# 인스턴스

가상머신 인스턴스를 생성하고, 실행 상태를 관리하고, 필요한 리소스를 연결해 사용합니다. CloudiA의 인스턴스 메뉴는 단순히 가상머신을 만드는 화면이 아니라, 전원 관리, 스냅샷, 디스크 구성, 네트워크 연결, 성능 확장까지 이어지는 운영의 중심 화면입니다.

## 화면 개요
- `목록 화면`: 인스턴스 현황을 조회하고 생성, 편집, 전원 제어, 삭제, 스냅샷 생성 같은 작업을 시작합니다.
- `생성 화면`: 이름, 스펙, 설치 유형, 네트워크, 사용자 계정, 고급 설정, 사용자 스크립트를 입력합니다.
- `편집 화면`: 생성 화면과 같은 구성을 사용하지만, 생성 후 고정되어야 하는 항목은 잠겨 있으며 실행 중에는 일부 항목만 수정할 수 있습니다.
- `상세 화면`: 인스턴스 상태와 연결된 리소스를 확인하고, 디스크 부팅 순서 변경, 스냅샷 관리, AI 가속기 확인, 가상 CPU 피닝 확인 같은 후속 작업을 수행합니다.
- `후속 작업`: 스냅샷 생성은 목록 화면의 `더보기`, 부팅 순서 변경과 가상 CPU 피닝 편집은 상세 화면의 해당 탭에서 진행합니다.

## 인스턴스 목록

인스턴스 목록 화면은 프로젝트에 생성된 인스턴스를 확인하고, 전원 관리나 편집, 삭제 같은 주요 작업을 시작하는 화면입니다. 특히 `생성 방식`, `상태`, `IP 주소`를 함께 보면 지금 어떤 인스턴스를 어떤 메뉴에서 관리해야 하는지 빠르게 판단할 수 있습니다.

### 주요 작업
- `생성`: 새 인스턴스를 만듭니다. 생성 버튼은 항상 표시됩니다.
- `편집`: 선택한 **1개** 인스턴스를 수정합니다. `종료됨`, `정지됨`, `실행 중` 상태에서 사용할 수 있으며, `ISO 파일` 설치 유형의 인스턴스는 `실행 중`일 때 편집할 수 없습니다.
- `삭제`: 선택한 **1개** 인스턴스에 대해 삭제 확인 창을 엽니다.
  - 목록 버튼은 `생성 중`, `실행 중`, `삭제 중`, `부팅 중`, `재부팅 중`, `종료 중` 상태에서는 비활성화됩니다.
  - 삭제 확인 창이 열리더라도, 실제 삭제는 **수동 생성 인스턴스**이고 **연결된 인스턴스 스냅샷이 0개**일 때만 실행할 수 있습니다.
- `실행`: 전원이 꺼진 인스턴스를 켭니다. `종료됨`, `정지됨` 상태이면서 **수동 생성** 인스턴스일 때 활성화됩니다.
- `종료`: 실행 중인 인스턴스를 정상 종료합니다. `실행 중`, `일시 중지` 상태이면서 **수동 생성** 인스턴스일 때 활성화됩니다.
- `재부팅`: 실행 중인 인스턴스를 다시 시작합니다. `실행 중` 상태이면서 **수동 생성** 인스턴스일 때 활성화됩니다. `SEV-ES` 보안 유형 인스턴스는 재부팅할 수 없습니다.
- `더보기 > 스냅샷 생성`: 인스턴스 상태를 스냅샷으로 저장합니다. `실행 중` 또는 `종료됨` 상태에서 사용할 수 있으며, 진행 중인 작업이 있는 경우에는 비활성화될 수 있습니다.

> 참고:
> 목록에서는 좌측 **라디오 버튼**으로 인스턴스를 선택합니다.
> 한 번에 **1개 인스턴스만 선택**할 수 있으며, 상단 작업도 모두 단일 선택 기준으로 동작합니다.

### 테이블 컬럼

인스턴스 목록은 다음 정보를 표 형태로 보여줍니다.

| 컬럼 | 설명 |
|---|---|
| **이름** | 인스턴스 표시 이름입니다. 클릭하면 상세 화면으로 이동합니다. 다만 `생성 중`, `삭제 중`, `에러` 상태에서는 상세 화면으로 이동할 수 없습니다. |
| **ID** | 인스턴스 고유 ID입니다. |
| **생성 방식** | 인스턴스가 만들어진 경로입니다. `수동 생성`, `오토 스케일링 그룹`, `쿠버네티스`, `파일 시스템`, `Gen:iA` 등으로 표시됩니다. |
| **IP 주소** | NIC에 할당된 IPv4 주소를 표시합니다. NIC가 여러 개면 쉼표로 구분해 함께 표시합니다. |
| **상태** | `실행 중`, `종료됨`, `생성 중` 등 현재 상태를 색상 점과 텍스트로 보여줍니다. |
| **vCPU** | 인스턴스에 할당된 vCPU 개수입니다. |
| **메모리** | UI에서 자동 변환된 메모리 크기를 표시합니다. |
| **생성 일시** | 인스턴스가 생성된 날짜와 시간입니다. |

### 검색과 목록 동작

- **검색 조건**: 드롭다운에서 `이름`을 선택합니다.
- **검색어 입력**: 찾고 싶은 인스턴스 이름을 입력합니다.
- **검색 버튼**: 검색을 실행합니다.
- **정렬**: 대부분의 컬럼은 헤더에서 정렬할 수 있습니다. 다만 **IP 주소 컬럼은 정렬되지 않습니다.**
- **기본 정렬**: 생성 일시 기준으로 정렬됩니다.
- **페이지 크기**: 한 페이지에 10개 항목이 표시됩니다.

### 목록 화면에서 확인할 사항
- `생성 방식`을 보면 현재 인스턴스를 이 화면에서 직접 관리해야 하는지, 아니면 오토 스케일링 그룹이나 쿠버네티스 같은 상위 기능에서 함께 관리해야 하는지 빠르게 판단할 수 있습니다.
- `IP 주소`를 보면 단일 NIC 인스턴스인지, 여러 NIC를 가진 인스턴스인지 바로 구분할 수 있습니다. IP가 여러 개면 다중 NIC 운영 여부를 함께 확인하는 것이 좋습니다.
- `상태`에 따라 가능한 버튼이 달라집니다. 같은 인스턴스라도 `실행 중`인지 `종료됨`인지에 따라 편집, 전원 제어, 삭제 가능 여부가 달라집니다.

### 운영 시 주의사항
- 전원 제어와 삭제는 일반적으로 **수동 생성 인스턴스**를 기준으로 동작합니다. 시스템이 자동으로 생성한 인스턴스는 해당 상위 서비스 화면에서 관리하는 것이 안전합니다.
- 스냅샷이 남아 있으면 삭제 확인 창이 열려도 실제 삭제를 진행할 수 없습니다.
- 이름 클릭이 되지 않는 상태는 단순 UI 오류가 아니라 상세 화면 진입이 제한된 상태일 수 있습니다. 생성이나 삭제 같은 작업이 끝난 뒤 다시 확인합니다.

### 원하는 인스턴스가 보이지 않을 때 확인할 사항
- 검색 조건이 `이름`으로 되어 있는지 확인합니다.
- 검색어가 인스턴스 표시 이름과 일치하는지 확인합니다.
- 목록은 생성 일시 기준으로 정렬되므로, 오래된 인스턴스는 뒤 페이지에 있을 수 있습니다.
- 한 페이지에 10개씩 표시되므로 다음 페이지도 함께 확인합니다.
- `생성 중`, `삭제 중` 상태에서는 목록에 보이더라도 상세 진입이나 일부 작업이 바로 되지 않을 수 있습니다.

## 인스턴스 생성·편집

인스턴스 생성과 편집은 같은 화면 구성을 사용합니다. 다만 **편집 모드에서는 설치 유형, 이미지, VPC처럼 바꾸면 안 되는 항목이 잠기고**, 인스턴스가 `실행 중`이면 일부 항목만 수정할 수 있습니다. 따라서 생성 화면에서는 앞으로 사용할 네트워크와 접속 방식, 스토리지, 확장 계획까지 함께 생각하고 입력하는 것이 좋습니다.

### 인스턴스 생성 전 준비하면 좋은 것
- 사용할 `VPC`와 `서브넷`을 먼저 정합니다.
- 접속 방식을 `비밀번호`, `SSH 키`, 또는 둘 다 사용할지 정합니다.
- 추가 블록 디스크, 파일 시스템, AI 가속기(NPU)가 필요한지 미리 결정합니다.
- 설치 유형을 `OS 이미지`와 `ISO 파일` 중 어떤 방식으로 가져갈지 결정합니다.

> 주의: 실행 중 편집(라이브 업데이트)
>
> 인스턴스가 `실행 중`일 때 편집 화면 상단에는 경고 배너가 표시됩니다.
>
> 실행 중에도 가능한 작업:
> - vCPU 변경
> - 메모리 증설
> - 네트워크 인터페이스 추가/편집/삭제 (기본 NIC 제외)
> - 블록 디바이스 추가/삭제
>
> 실행 중에는 잠기는 작업:
> - 인스턴스 이름
> - 인스턴스 타입 선택/직접 입력 전환
> - 하이퍼스레딩 변경
> - 설치 유형, OS 이미지, ISO 파일, 부팅 디스크 크기
> - VPC 변경
> - 기본 네트워크 인터페이스 편집/삭제
> - 사용자 계정 전체
> - 보안 유형, 확장 한도
> - 파일 시스템
> - AI 가속기
> - 가상 CPU 피닝 추가
> - 사용자 스크립트 등록
>
> 게스트 OS 내부 조치가 필요한 작업:
> - 실행 중 네트워크 인터페이스를 추가하면 서브넷에서는 IP가 할당되지만, **게스트 OS 내부에는 자동으로 반영되지 않으므로 직접 설정**해야 합니다.

### 1. 기본 정보

기본 정보 섹션에서는 인스턴스 이름, 인스턴스 구성, 설치 유형을 설정합니다. 생성 시에는 인스턴스의 전체 방향을 정하는 영역이고, 편집 시에는 현재 인스턴스의 구성을 다시 확인하는 기준 화면으로 사용합니다.

#### 1.1 인스턴스 이름

- **인스턴스 이름**: 인스턴스를 식별할 이름을 입력합니다.
- 이름은 목록과 상세 화면에서 가장 먼저 보이는 식별 정보이므로, 서비스 목적이나 역할이 드러나도록 작성하는 것이 좋습니다.

##### 입력 필드 유효성

| 필드 | 허용 값 | 검증 규칙 | 비고 |
|---|---|---|---|
| **인스턴스 이름** | 영문 대/소문자, 숫자, `-`, `_` | 최대 80자, 첫 글자 숫자 불가, 공백과 기타 특수문자 불가 | **필수** |

#### 1.2 인스턴스 구성

인스턴스 스펙은 **인스턴스 타입 선택** 또는 **직접 입력** 두 방식 중 하나로 설정합니다. 빠르게 표준 스펙을 고를지, 세부 조정이 필요한지에 따라 방식을 선택합니다.

##### 선택 방식 가이드
- **인스턴스 타입 선택**: 사전에 정의된 스펙 중에서 빠르게 선택하고 싶을 때 적합합니다.
- **직접 입력**: vCPU, 메모리, 하이퍼스레딩 등 세부 값을 직접 조정해야 할 때 적합합니다.
- 인스턴스 타입 모달의 `GPU` 컬럼은 타입 정보를 이해하기 위한 참고 정보입니다. 현재 인스턴스 생성 화면에서는 별도의 GPU 설정 항목을 직접 다루는 구조가 아닙니다.

##### 모드 1: 인스턴스 타입 선택
- **인스턴스 타입 선택** 라디오를 선택합니다.
- 우측 **선택** 버튼을 누르면 **인스턴스 타입 선택** 모달이 열립니다.
- 모달에서는 다음 정보를 확인할 수 있습니다.
  - `이름`
  - `vCPU`
  - `메모리`
  - `GPU`
  - `연산 성능`
- 타입을 선택하면 해당 타입에 맞는 vCPU와 메모리가 자동으로 적용됩니다.
- 또한 선택한 타입의 **최대 네트워크 인터페이스 개수** 제한도 함께 적용됩니다.

##### 모드 2: 직접 입력
- **직접 입력** 라디오를 선택합니다.
- **vCPU**와 **메모리**를 직접 입력합니다.
- 직접 입력 모드에서는 인스턴스 타입 제한 대신 **기본 최대 NIC 개수 100개 기준**으로 동작합니다.
- 편집 시 인스턴스가 `실행 중`이면, 현재 값보다 작게 줄일 수 없고 **최대 vCPU / 최대 메모리** 범위 안에서만 늘릴 수 있습니다.

##### vCPU 상세 설정
`상세 설정`을 펼치면 vCPU의 내부 구성을 더 자세히 조정할 수 있습니다.

- **기본 vCPU 사용**: 현재 선택한 vCPU 값을 기준으로 기본 구성을 사용합니다.
- **옵션 지정**: 하이퍼스레딩과 코어 수를 직접 조정합니다.
- **적용된 vCPU**: 현재 설정으로 실제 적용될 vCPU 개수를 보여줍니다.

###### 하이퍼스레딩
- 토글 스위치로 설정합니다.
- `사용 안 함`: 코어당 1 스레드
- `사용함`: 코어당 2 스레드
- 실행 중 편집에서는 하이퍼스레딩을 바꿀 수 없습니다.

###### 옵션 지정 시 추가 항목
- **CPU 코어당 스레드 수**
- **코어 수**
- **계산 규칙**
  - `적용된 vCPU = CPU 코어당 스레드 수 × 코어 수`

##### 입력 필드 유효성

| 필드 | 허용 값 | 검증 규칙 | 비고 |
|---|---|---|---|
| **vCPU** | 1 이상의 정수 | 생성 시 1 이상, 실행 중 편집 시 현재 값 이상이면서 최대 vCPU 이하 | 라이브 업데이트에서 `SEV-ES`는 비활성 |
| **메모리** | 크기 입력 + 단위 선택 | 생성 시 1 이상, 실행 중 편집 시 현재 값 이상이면서 최대 메모리 이하 | 실행 중 증설 시 최소 1GiB, 512MiB 단위 |

> 참고:
> 인스턴스 타입을 선택한 경우 사용자는 주로 스펙을 빠르게 고르는 데 집중하면 됩니다.
> 직접 입력은 세부 조정이 필요할 때 적합합니다.

#### 1.3 설치 유형

- **설치 유형**은 `OS 이미지` 또는 `ISO 파일` 중 하나를 선택합니다.
- 생성 화면에서만 선택할 수 있고, 편집 화면에서는 변경할 수 없습니다.

##### 설치 유형 선택 가이드
- **OS 이미지**: 빠르게 인스턴스를 만들고 Cloud-init 기반 사용자 계정과 초기 설정을 함께 적용하고 싶을 때 적합합니다.
- **ISO 파일**: 운영체제 설치 과정을 직접 진행하거나, 표준 이미지 대신 별도 ISO 기반 구성을 사용해야 할 때 적합합니다.
- ISO 방식은 설치 자유도가 높지만, 사용자 계정, 고급 설정, 실행 중 편집이 제한되므로 이 점을 감안해 선택합니다.

> 주의:
> `ISO 파일` 설치 유형에서는 **사용자 계정**과 **고급 설정**이 비활성화되며, 실행 중 편집도 지원하지 않습니다.

##### OS 이미지
- **OS 이미지**를 선택해 부팅용 이미지를 지정합니다.
- 우측 **선택** 버튼을 누르면 **이미지 선택** 모달이 열립니다.
- 모달에서는 `이름`, `ID`, `분류`, `OS 타입`, `크기`, `스토리지 도메인`, `생성 일시`를 확인할 수 있습니다.
- 편집 화면에서는 이미지를 바꿀 수 없습니다.

##### ISO 파일
- **ISO 파일**을 선택해 설치용 ISO를 지정합니다.
- 우측 **선택** 버튼을 누르면 **ISO 파일 선택** 모달이 열립니다.
- 모달에서는 `이름`, `ID`, `OS 타입`, `크기`, `스토리지 도메인`, `생성 일시`를 확인할 수 있습니다.
- 편집 화면에서는 ISO를 바꿀 수 없습니다.

##### 디스크 크기 설정

`디스크 크기 설정`을 펼치면 부팅 디스크 크기를 지정할 수 있습니다.

- `OS 이미지` 설치 유형
  - 선택한 이미지의 크기가 기본값으로 채워집니다.
  - 이미지 기본 크기보다 작게 설정할 수 없습니다.
- `ISO 파일` 설치 유형
  - OS를 설치할 빈 부팅 디스크 크기를 입력할 수 있습니다.
  - 값을 입력하지 않으면 시스템 기본값이 적용됩니다.
- 편집 화면에서는 부팅 디스크 크기를 변경할 수 없습니다.

##### 입력 필드 유효성

| 필드 | 허용 값 | 검증 규칙 | 비고 |
|---|---|---|---|
| **OS 이미지 디스크 크기** | 이미지 기본 크기 이상 | 소수점 불가 | 기본값은 선택한 이미지 크기 |
| **ISO 부팅 디스크 크기** | 시스템 기본 크기 이상 | 소수점 불가 | 미입력 시 시스템 기본값 사용 |

### 2. 네트워크 인터페이스 구성

인스턴스는 최소 1개의 네트워크 인터페이스가 필요합니다. 이 섹션에서는 VPC와 서브넷을 선택하고, NIC를 추가하거나 편집합니다. 실제 서비스 통신이 이 영역에서 결정되므로, 생성 전에 어떤 네트워크에 연결할지 먼저 정해 두는 것이 좋습니다.

#### 인터페이스를 추가하기 전에 확인할 사항
- 어떤 `VPC`와 `서브넷`에 연결할지 먼저 확정합니다.
- 외부 통신이 필요한지, 내부 전용 통신인지에 따라 연결할 서브넷이 달라질 수 있습니다.
- 보안그룹을 어떤 기준으로 묶을지, 기본 NIC 외에 추가 NIC가 필요한지 먼저 생각하면 이후 편집이 수월합니다.
- 생성 화면에서 인터페이스를 추가한 뒤에는 VPC를 바꾸기 어렵기 때문에, 처음 선택이 중요합니다.

- **VPC 선택**
  - 드롭다운에서 선택합니다.
  - 검색 입력을 통해 빠르게 찾을 수 있습니다.
  - 편집 모드에서는 VPC를 바꿀 수 없습니다.
  - 생성 화면에서도 이미 인터페이스를 추가한 뒤에는 VPC를 바꾸기 어렵기 때문에, 처음에 사용할 VPC를 먼저 확정하는 것이 좋습니다.

#### 인터페이스 목록
- 인터페이스 목록은 표로 표시됩니다.
- 목록이 비어 있으면 `"리스트 데이터가 존재하지 않습니다."` 메시지가 표시됩니다.
- 상단 버튼으로 `추가`, `편집`, `삭제`를 수행합니다.
- 화면에는 **현재 인터페이스 개수**가 표시됩니다.
  - 인스턴스 타입 선택 시: `현재 인터페이스 개수: 1 / 최대 개수`
  - 직접 입력 시: 기본 100개 기준
- 최대 개수를 초과하면 개수 표시가 경고 색상으로 바뀝니다.

#### 인터페이스 추가
**추가** 버튼을 클릭하면 **인터페이스 추가** 모달이 열립니다.

- **서브넷**: 드롭다운에서 선택합니다.
- **IP 주소**:
  - `자동 할당`: 서브넷 안에서 자동으로 할당합니다.
  - `수동 할당`: 직접 IPv4 주소를 입력합니다.
  - 수동 할당을 하려면 먼저 서브넷을 선택해야 합니다.
- **보안그룹**:
  - 다중 선택할 수 있습니다.
  - 최대 5개까지 선택할 수 있습니다.

첫 번째로 추가한 인터페이스는 자동으로 **기본 네트워크 인터페이스**가 됩니다.

#### 인터페이스 편집
**편집** 버튼을 클릭하면 **인터페이스 편집** 모달이 열립니다.

- 현재 UI 기준으로 다음 항목을 수정할 수 있습니다.
  - **서브넷**
  - **IP 주소**
  - **보안그룹**
- 실행 중 편집에서는 **기본 네트워크 인터페이스**를 편집할 수 없습니다.

#### 다중 인터페이스를 쓰는 경우 적합한 상황
- 서비스 트래픽과 관리 트래픽을 분리하고 싶을 때
- 서로 다른 서브넷 또는 네트워크 구간을 동시에 사용해야 할 때
- 특정 애플리케이션에 별도 고정 IP를 붙여 운영하고 싶을 때

#### 다중 인터페이스 사용 시 참고
- 첫 번째 네트워크 인터페이스는 기본 인터페이스로 동작합니다.
- 실행 중에는 **기본 네트워크 인터페이스를 삭제할 수 없습니다.**
- 실행 중에 추가한 NIC는 CloudiA에서 IP가 할당되더라도, **게스트 OS 내부에 자동 설정되지 않습니다.**
- NIC를 2개 이상 사용하면 운영체제 라우팅 설정에 따라 **비대칭 라우팅**이 발생할 수 있습니다.

> 참고:
> 다중 NIC를 사용할 때 통신이 되지 않으면, 게스트 OS에서 `ip rule` 기반 정책 라우팅(PBR)이나 `rp_filter` 설정을 확인하는 것이 좋습니다.

#### 입력/검증 규칙

| 항목 | 설명 | 비고 |
|---|---|---|
| **서브넷** | 선택한 VPC 안의 서브넷 중 지정 | 최소 1개 인터페이스 필요 |
| **IPv4 주소** | `A.B.C.D` 형식과 서브넷 범위를 함께 검사 | 미입력 시 자동 할당 가능 |
| **보안그룹** | 다중 선택 | 최대 5개 |

### 3. 사용자 계정

`OS 이미지` 설치 유형에서는 Cloud-init을 통해 사용자 계정과 인증 방식을 함께 설정할 수 있습니다. 초기 접속을 어떤 방식으로 가져갈지 이 섹션에서 결정하며, 이후 운영 편의성에도 직접 영향을 줍니다.

> 주의:
> 설치 유형이 `ISO 파일`이면 이 섹션은 비활성화됩니다. 이 경우 게스트 OS 설치 후 운영체제 안에서 직접 계정을 만들어야 합니다.

- **사용자 이름**: 게스트 OS에 생성할 로그인 계정 이름입니다.

#### 접속 방식 선택 가이드
- **비밀번호만 사용**: 빠르게 콘솔이나 임시 접속을 해야 하는 환경에서 간단하게 사용할 수 있습니다.
- **SSH 키 함께 사용**: 운영 환경에서는 SSH 키를 함께 구성하는 것이 일반적으로 더 안전하고 편리합니다.
- ISO 설치 유형을 선택했다면 이 화면에서 계정을 넣을 수 없으므로, OS 설치 이후 운영체제 내부에서 계정과 인증 방식을 직접 구성해야 합니다.

#### 인증 방식

Cloud-init 인증 방식은 **비밀번호**와 **SSH 키**를 함께 또는 선택적으로 설정할 수 있습니다. 단, **둘 중 하나는 반드시 지정해야** 생성이나 편집을 완료할 수 있습니다.

##### 비밀번호 등록
- 초기에는 비밀번호 값이 비어 있으며, 우측에 **등록** 버튼이 표시됩니다.
- `등록`을 누르면 인라인 패널이 열리고 다음 항목을 입력합니다.
  - `새 비밀번호`
  - `비밀번호 확인`
- 등록 후에는 비밀번호가 `********` 형태로 표시되고, `삭제`, `변경` 버튼이 활성화됩니다.
- `변경`을 누르면 같은 방식으로 새 비밀번호를 다시 입력합니다.
- `삭제`를 누르면 비밀번호 인증을 제거합니다.

##### SSH 키
- 드롭다운에서 프로젝트에 등록된 SSH 키를 선택합니다.
- 내부 검색을 지원합니다.
- 체크박스 기반 다중 선택이 가능하며, 예시는 `key-name (ID: 2)` 형태로 표시됩니다.
- 최대 5개까지 선택할 수 있습니다.
- 편집 화면에서는 기존에 연결된 SSH 키가 미리 선택된 상태로 표시됩니다.

#### 입력 필드 유효성

| 필드 | 허용 값 | 비고 |
|---|---|---|
| **사용자 이름** | 4~20자, 첫 글자 영문 소문자, 소문자/숫자/`_`/`-` | 예약어는 사용할 수 없음, `OS 이미지` 설치 유형에서 **필수** |
| **비밀번호** | 4~20자, 허용된 영문/숫자/특수문자 | `OS 이미지` 설치 유형에서 선택 입력, SSH 키가 없으면 필요 |
| **SSH 키** | 프로젝트에 등록된 SSH 키 목록 | 최대 5개, 비밀번호가 없으면 1개 이상 필요 |

### 4. 고급 설정 (선택)

`고급 설정`을 펼치면 인스턴스의 보안/확장 한도, 스토리지, AI 가속기, CPU 피닝을 설정할 수 있습니다. 성능 최적화, 보안 강화, 운영 편의성 확보가 필요한 경우 이 영역을 함께 사용합니다.

> 주의:
> 설치 유형이 `ISO 파일`이면 고급 설정 전체가 비활성화됩니다.

#### 4.1 보안 및 확장 설정

보안 요구 사항이 있는 워크로드이거나, 실행 중 증설을 염두에 두고 인스턴스를 만들 때 사용하는 영역입니다.

##### 보안 유형
- `사용 안 함`
- `SEV`
- `SEV-ES`
- `SEV-SNP`

현재 환경에서 지원하는 항목만 활성화되어 표시됩니다.

##### 확장 한도
- **최대 vCPU**: 실행 중 편집에서 vCPU를 어디까지 늘릴 수 있는지 설정합니다.
- **최대 메모리**: 실행 중 편집에서 메모리를 어디까지 늘릴 수 있는지 설정합니다.
- 값을 입력하지 않으면 현재 설정한 vCPU, 메모리 값이 그대로 상한으로 사용됩니다.

##### 입력 필드 유효성

| 필드 | 허용 값 | 검증 규칙 | 비고 |
|---|---|---|---|
| **최대 vCPU** | 1 이상의 정수 | 현재 vCPU 이상 | 실행 중 증설 상한 |
| **최대 메모리** | 크기 입력 + 단위 선택 | 현재 메모리 이상 | 실행 중 증설 상한 |

> 참고:
> 실행 중 메모리 증설은 현재 값보다 **최소 1GiB 이상**, **512MiB 단위**로만 늘릴 수 있습니다.

#### 4.2 스토리지 추가

운영 데이터와 OS 디스크를 분리하거나, 공유 저장소를 연결해야 할 때 사용하는 영역입니다.

##### 블록
- 프로젝트에 있는 블록 자원을 선택해 인스턴스에 연결합니다.
- 잘못 선택한 항목은 목록에서 선택한 뒤 `삭제`로 제거할 수 있습니다.
- `추가`를 누르면 **블록 선택** 모달이 열리며, 여러 개를 동시에 선택할 수 있습니다.

##### 파일 시스템
- 파일 시스템(NFS, virtiofs) 자원을 선택해 인스턴스에 연결합니다.
- `추가`를 누르면 **파일 시스템 선택** 모달이 열립니다.
- 파일 시스템을 선택하고 **마운트 포인트**를 입력합니다.

> 주의:
> 실행 중 편집에서는 **부팅 디스크를 제외한 추가 블록 디스크만** 추가/삭제할 수 있습니다.
>
> 파일 시스템은 실행 중 편집에서 변경할 수 없습니다.
>
> NFS 파일 시스템이 연결된 경우, 인스턴스를 종료할 때 일반적인 경우보다 시간이 더 걸릴 수 있습니다.

#### 4.3 AI 가속기

AI 추론이나 모델 처리처럼 NPU 자원이 필요한 워크로드에서 사용하는 영역입니다. 현재 인스턴스 화면에서 AI 가속기는 **NPU 기준**으로 제공됩니다.

##### NPU
- **장치**:
  - `선택` 버튼을 눌러 NPU 장치 선택 모달을 엽니다.
  - 모달에서 `공급업체`, `제품명`, `전체 카드 수`, `사용 가능한 카드 수`를 확인한 뒤 1개 장치를 선택합니다.
- **카드 수**:
  - 선택한 장치 기준으로 요청할 카드 수를 입력합니다.
- 편집 화면에서는 기존에 연결된 NPU가 있으면 장치와 카드 수가 자동으로 채워집니다.

##### 입력/검증 규칙

| 필드 | 허용 값 | 검증 규칙 | 비고 |
|---|---|---|---|
| **장치** | 등록된 NPU 장치 1개 | 미선택 시 저장 불가 | 필수 |
| **카드 수** | 정수 1 이상 | 선택한 장치의 전체 카드 수 이하 | 필수 |

> 주의:
> `실행 중 편집(라이브 업데이트)`과 `ISO 설치` 유형에서는 AI 가속기 입력이 비활성화됩니다.

#### 4.4 가상 CPU 피닝 설정

일정한 성능 특성이 중요한 워크로드에서 vCPU를 특정 논리 CPU 범위에 고정해 배치하고 싶을 때 사용하는 기능입니다.

- 가상 CPU 피닝은 인스턴스의 vCPU를 호스트의 논리 CPU 범위에 고정하는 설정입니다.
- `추가`를 눌러 규칙을 입력하고, 필요 없으면 목록에서 선택한 뒤 `삭제`로 제거합니다.
- 각 규칙에는 다음 정보가 포함됩니다.
  - **vCPU ID**
  - **논리 CPU 범위(시작, 끝)**

##### 입력/검증 규칙

| 필드 | 허용 값 | 검증 규칙 | 비고 |
|---|---|---|---|
| **vCPU ID** | 0 이상의 정수 | 현재 인스턴스 vCPU 개수 미만 | 현재 vCPU 기준으로 검사 |
| **논리 CPU 범위(시작)** | 0 이상의 정수 | 허용 범위 이내 | 팝업 상단 안내 참고 |
| **논리 CPU 범위(끝)** | 0 이상의 정수 | 시작 값 이상, 중복 범위 불가 | 동일 vCPU에 중복 설정 불가 |

> 참고:
> 현재 설정한 vCPU 범위를 넘는 CPU 피닝 값은 저장할 수 없습니다.

### 5. 사용자 스크립트 (선택)

사용자 스크립트는 인스턴스 부팅 시 Cloud-init을 통해 자동 실행되는 작업입니다. 초기 패키지 설치, 설정 파일 생성, 서비스 시작 같은 반복 작업을 자동화할 때 유용합니다.

- 섹션 상단에는 `등록된 사용자 스크립트: N개`가 표시됩니다.
- 우측 **등록** 버튼을 누르면 **사용자 스크립트** 모달이 열립니다.

#### 사용자 스크립트 등록

- **실행 순서**: 드래그 핸들로 순서를 바꿀 수 있습니다.
- **파일 이름**: 스크립트 파일 이름을 입력합니다.
- **단계**:
  - `PER_BOOT`: 인스턴스가 부팅할 때마다 실행됩니다.
  - `PER_INSTANCE`: 해당 인스턴스 기준으로 실행되며, 수정 후 다시 적용할 수 있습니다.
  - `PER_ONCE`: **이미지 기준 1회성** 동작입니다. 이미 같은 이미지에서 1회 실행된 상태라면 내용을 바꿔도 수정분이 반영되지 않을 수 있습니다.
- **내용**: 텍스트 영역에 스크립트 내용을 입력합니다.
- **규칙 추가**: 여러 개의 스크립트를 순서대로 등록할 수 있습니다.

#### 활용 예시
- 패키지 설치: 웹 서버, 에이전트, 운영 도구를 부팅 시 자동 설치할 때
- 설정 파일 생성: 애플리케이션 초기 설정 파일이나 환경 변수 파일을 만들 때
- 서비스 자동 시작: 부팅 후 서비스가 자동으로 올라오도록 구성할 때
- `PER_BOOT`: 부팅할 때마다 실행되어야 하는 초기화 작업에 적합합니다.
- `PER_INSTANCE`: 특정 인스턴스에만 반복 반영하고 싶은 설정 작업에 적합합니다.
- `PER_ONCE`: 같은 이미지에서 한 번만 적용돼야 하는 초기 1회 작업에 적합합니다.

> 참고:
> `PER_BOOT`와 `PER_INSTANCE`는 내용을 수정하면 수정된 내용으로 다시 반영됩니다.

### 편집이 안 될 때 확인할 사항
- 인스턴스가 `실행 중`이면 라이브 업데이트 허용 항목 외에는 대부분 잠깁니다.
- 설치 유형이 `ISO 파일`이면 사용자 계정, 고급 설정, 실행 중 편집이 제한됩니다.
- 기본 네트워크 인터페이스는 실행 중 편집에서 수정하거나 삭제할 수 없습니다.
- 파일 시스템, AI 가속기, 사용자 스크립트 등록은 실행 중 편집에서 변경할 수 없습니다.

## 인스턴스 상세

인스턴스 상세 화면에서는 현재 인스턴스 상태와 연결된 리소스를 한눈에 확인하고, 스냅샷이나 부팅 순서 변경 같은 후속 작업을 진행할 수 있습니다. 이 화면에서는 지금 인스턴스가 어떤 상태인지, 어떤 리소스와 연결되어 있는지, 다음에 어떤 작업을 해야 하는지를 빠르게 판단할 수 있습니다.

### 상단 정보

상세 화면 상단에는 다음 3개의 정보 박스가 표시됩니다.

- **기본 정보**: 이름, 상태, ID, UUID, VPC, 물리 머신, 생성 일시
- **인스턴스 구성 및 보안**: 인스턴스 타입(사용한 경우), vCPU, 메모리, 최대 vCPU, 최대 메모리, 보안 유형
- **프로비저닝 정보**: 생성 방식, 설치 유형(`OS 이미지` / `ISO 파일`), OS 이미지 / ISO 파일, 시작 템플릿, 사용자 계정 정보

> 참고:
> `콘솔` 버튼은 `실행 중` 상태에서만 쓰는 것이 아니라, 현재 UI 기준으로 아래 상태에서 활성화될 수 있습니다.
> - `실행 중`
> - `부팅 중`
> - `재부팅 중`
> - `종료 중`
> - `마이그레이션 중`
> - `일시 중지`

### 상단 정보에서 먼저 볼 항목
- **상태와 생성 방식**: 지금 이 인스턴스에 어떤 작업을 할 수 있는지, 그리고 직접 관리 대상인지 먼저 판단합니다.
- **설치 유형**: `OS 이미지`인지 `ISO 파일`인지에 따라 편집 가능 범위가 달라집니다.
- **vCPU/메모리/보안 유형**: 현재 스펙과 보안 옵션이 기대한 값인지 확인합니다.
- **사용자 계정/시작 템플릿 여부**: 초기 생성 방식과 접속 준비 상태를 확인할 때 유용합니다.

### 하단 구성

상세 화면 하단은 3개 영역으로 구성됩니다.

- **인스턴스 구성 요소**
  - 좌측 사이드 네비게이션으로 `네트워크 인터페이스`, `디스크`, `스냅샷`, `호스트 장치`, `AI 가속기`, `사용자 스크립트`, `가상 CPU 피닝`을 전환합니다.
- **연결된 리소스**
  - 좌측 사이드 네비게이션으로 `파일 시스템`, `플로팅 IP`, `선호도 그룹`, `SSH 키`를 전환합니다.
- **모니터링 정보**
  - CPU/메모리, 디스크, 네트워크 지표를 그래프로 확인합니다.

### 하단 영역 활용 방식
- **인스턴스 구성 요소**: 인스턴스 자체에 붙은 자원과 내부 구성을 확인할 때 사용합니다.
- **연결된 리소스**: 플로팅 IP, 파일 시스템, SSH 키처럼 외부 리소스와의 연결 상태를 확인할 때 사용합니다.
- **모니터링 정보**: 운영 중 성능과 트래픽 상태를 확인하고, 증설이나 점검 필요성을 판단할 때 사용합니다.

#### 인스턴스 구성 요소

- **네트워크 인터페이스**
  - `서브넷`, `보안그룹`, `IP 주소`, `기본 게이트웨이`, `DNS 서버`, `기본 인터페이스`, `MAC 주소` 정보를 확인합니다.
  - 게이트웨이와 DNS 서버를 보면 게스트 OS 안에서 어떤 네트워크 값을 참고해야 하는지 판단할 수 있습니다.
  - NIC 변경은 인스턴스 편집 화면에서 수행합니다.

- **디스크**
  - `부팅 순서`, `타입`, `장치 레이블`, `버스 타입`, `크기`, `소스 타입`, `소스 이름`을 확인합니다.
  - 부팅 디스크와 추가 디스크를 함께 비교하면서 어떤 장치가 실제 부팅 대상인지 확인할 수 있습니다.
  - 디스크가 2개 이상이고 인스턴스가 `종료됨` 상태이면 **부팅 순서 변경** 버튼이 활성화됩니다.

- **스냅샷**
  - `스냅샷 이름`, `스냅샷 ID`, `설명`, `생성 일시`를 확인합니다.
  - 목록은 단일 선택입니다.
  - `롤백`은 인스턴스가 `종료됨` 상태일 때만 사용할 수 있습니다.
  - 새 스냅샷은 인스턴스 목록의 `더보기 > 스냅샷 생성`에서 만듭니다.

- **호스트 장치**
  - 인스턴스에 연결된 호스트 장치 목록을 읽기 전용으로 보여줍니다.
  - `장치 분류`, `공급업체`, `제품명`을 확인할 수 있습니다.

- **AI 가속기**
  - 현재 상세 화면의 AI 가속기 관리는 **NPU 기준**으로 제공됩니다.
  - `공급업체`, `제품명`, `장치 유형`, `카드 수`를 확인합니다.
  - `추가`는 인스턴스가 `종료됨` 상태이고 `ISO 파일` 설치 유형이 아니며, 기존 항목이 없을 때 활성화됩니다.
  - `삭제`는 인스턴스가 `종료됨` 상태이고 항목을 선택했을 때 활성화됩니다.

- **사용자 스크립트**
  - 실행 순서, 파일 이름, 경로, 단계, 내용을 확인합니다.
  - 변경은 인스턴스 편집 화면에서 수행합니다.

- **가상 CPU 피닝**
  - `vCPU ID`, `논리 CPU 범위`를 확인합니다.
  - `편집`은 `종료 상태`의 인스턴스이면서 `ISO 파일` 설치 유형이 아닐 때 활성화됩니다.
  - 저장한 설정은 다음 실행 시 반영되는 것으로 이해하고 작업하는 것이 안전합니다.

#### 연결된 리소스

- **파일 시스템**
  - 연결된 파일 시스템의 이름, ID, 타입, 크기, 마운트 포인트, 생성 일시를 확인합니다.
- **플로팅 IP**
  - 연결된 플로팅 IP의 이름, ID, VPC, 외부 IP, 리소스 IP, 생성 일시를 확인합니다.
- **선호도 그룹**
  - 선호도 그룹의 이름, ID, 활성화 여부, 설명, 생성 일시를 확인합니다.
- **SSH 키**
  - 연결된 SSH 키의 이름, ID, 지문, 생성 일시를 확인합니다.

### 상세에서 보이는 정보와 실제 변경 위치
- 네트워크 인터페이스와 사용자 스크립트는 상세에서 조회하고, 실제 변경은 인스턴스 편집 화면에서 진행합니다.
- 플로팅 IP 연결 상태는 상세에서 확인할 수 있지만, 연결/해제 자체는 `플로팅 IP` 메뉴에서 함께 관리하는 것이 일반적입니다.
- SSH 키는 상세에서 현재 연결 상태를 보고, 필요 시 프로젝트 리소스와 인스턴스 편집 화면에서 함께 확인합니다.

#### 모니터링 정보

- CPU 사용률 (%)
- 메모리 사용률 (%)
- 디스크 읽기 (bytes/sec)
- 디스크 쓰기 (bytes/sec)
- 네트워크 입력 (bytes/sec)
- 네트워크 출력 (bytes/sec)
- 우측 상단 `CSV` 버튼으로 데이터 내보내기
- 날짜/시간 범위 선택기로 조회 구간 변경
- 차트 범례와 툴팁 제공

### 그래프를 볼 때 확인할 사항
- CPU나 메모리 사용률이 지속적으로 높으면 실행 중 증설이나 스펙 재조정을 검토합니다.
- 네트워크 입출력이 기대보다 낮거나 없는 경우에는 보안그룹, 플로팅 IP, 게스트 OS 내부 서비스 상태도 함께 확인합니다.
- `CSV`는 원하는 기간을 선택한 뒤 데이터 확인이나 외부 분석 용도로 내보낼 때 사용합니다.

### 외부 접속이 되지 않을 때 확인할 사항
- 인스턴스 전원 상태가 `실행 중`인지 확인합니다.
- 필요한 경우 플로팅 IP가 올바른 인터페이스에 연결되어 있는지 확인합니다.
- 보안그룹 인바운드 규칙에 접속 포트와 소스 IP 또는 CIDR이 포함되어 있는지 확인합니다.
- 게스트 OS 내부에서 실제 서비스가 실행 중인지 확인합니다.
- 다중 NIC 환경이라면 올바른 인터페이스에 IP가 설정되어 있는지, 라우팅이 기대한 방향으로 구성되어 있는지 함께 확인합니다.

## 인스턴스 삭제

인스턴스를 삭제할 때는 **버튼이 눌리는 조건**과 **실제로 삭제할 수 있는 조건**이 다릅니다.

1. 목록에서 삭제할 인스턴스를 선택합니다.
2. 상단의 **`삭제`** 버튼을 클릭합니다.
3. 삭제 확인 창에서 인스턴스 정보와 스냅샷 개수를 확인합니다.
4. 삭제 조건을 만족하면 **`삭제`** 버튼을 눌러 진행합니다.

### 삭제 버튼이 비활성화되는 상태
- `생성 중`
- `실행 중`
- `삭제 중`
- `부팅 중`
- `재부팅 중`
- `종료 중`

### 실제 삭제가 가능한 조건
- 생성 방식이 **`수동 생성`**
- 연결된 인스턴스 스냅샷 개수가 **0개**

### 삭제 전 점검 항목
- `생성 방식`이 수동 생성인지 확인합니다.
- 연결된 인스턴스 스냅샷이 남아 있지 않은지 확인합니다.
- 이 인스턴스가 실제 서비스, 배치, 테스트 환경 등 어디에 사용 중인지 영향 범위를 먼저 확인합니다.

> 참고:
> `오토 스케일링 그룹`, `쿠버네티스`, `파일 시스템` 등 다른 방식으로 생성된 인스턴스는 보통 해당 기능 화면에서 정리해야 합니다.
>
> 예를 들어 `오토 스케일링 그룹` 인스턴스는 오토 스케일링 그룹 화면에서, `쿠버네티스` 인스턴스는 쿠버네티스 화면에서 관리하는 것이 일반적입니다.

## 디스크 부팅 순서 변경

디스크가 2개 이상 연결된 인스턴스는 부팅 시 어떤 장치를 먼저 시도할지 순서를 바꿀 수 있습니다.

### 이 기능이 필요한 경우
- OS 재설치 후 다른 디스크로 먼저 부팅해야 할 때
- 여러 부팅 후보 디스크 중 특정 장치를 우선 순위로 올려야 할 때

1. 부팅 순서를 변경할 인스턴스의 상세 화면으로 이동합니다.
2. `인스턴스 구성 요소`에서 **`디스크`** 탭을 선택합니다.
3. 인스턴스가 `종료됨` 상태인지 확인합니다.
4. 상단의 **`부팅 순서 변경`** 버튼을 클릭합니다.
5. 모달에서 장치를 드래그해 순서를 바꾸고, 필요하면 **부팅 포함 여부 토글**을 조정합니다.
6. **`저장`** 버튼을 눌러 반영합니다.

> 주의:
> 최소 1개 장치는 부팅 순서에 포함되어 있어야 합니다.
> 따라서 마지막으로 남은 부팅 장치는 제외할 수 없습니다.

## 인스턴스 스냅샷 생성

인스턴스 스냅샷은 특정 시점의 인스턴스 디스크 상태를 저장해 두었다가, 이후 롤백이나 관리 작업에 활용할 수 있는 기능입니다.

### 스냅샷 활용 예시
- 패치나 업그레이드 전에 복구 지점을 확보할 때
- 애플리케이션 설정 변경 전 디스크 상태를 백업할 때
- 테스트 후 이전 상태로 되돌릴 가능성이 있을 때

> 참고:
> Ubuntu 등 기본 이미지에 게스트 에이전트가 포함되지 않은 경우, `실행 중` 상태에서 스냅샷 생성이 실패할 수 있습니다.
> 이때는 게스트 OS 내부에 QEMU Guest Agent를 설치한 뒤 다시 시도하는 것이 좋습니다.
>
> ```bash
> sudo apt update
> sudo apt install -y qemu-guest-agent
> sudo systemctl enable --now qemu-guest-agent
> ```

1. 목록 화면에서 스냅샷을 만들 인스턴스를 선택합니다.
2. **`더보기 > 스냅샷 생성`**을 클릭합니다.
3. 스냅샷 생성 모달에서 다음 정보를 입력합니다.
   - **스냅샷 이름**: 필수 입력, 최대 60자
   - **설명**: 선택 입력, 최대 200자
4. **`생성`** 버튼을 눌러 스냅샷을 만듭니다.

### 스냅샷 작업 전 주의사항
- 스냅샷 생성은 `실행 중` 또는 `종료됨` 상태에서 사용할 수 있습니다.
- 실행 중 스냅샷은 게스트 OS 내부 애플리케이션 정합성을 항상 보장하는 것은 아니므로, 중요 작업 전에는 애플리케이션 상태도 함께 확인하는 것이 좋습니다.
- `qemu-guest-agent`가 있으면 실행 중 스냅샷의 정합성을 높이는 데 도움이 됩니다.

## 인스턴스 스냅샷 편집

스냅샷 이름과 설명은 인스턴스 상세 화면에서 수정할 수 있습니다.

1. 인스턴스 상세 화면에서 `인스턴스 구성 요소 > 스냅샷`을 선택합니다.
2. 수정할 스냅샷 1개를 선택합니다.
3. **`편집`** 버튼을 클릭합니다.
4. 스냅샷 이름과 설명을 수정합니다.
5. **`편집`** 버튼을 눌러 반영합니다.

## 인스턴스 스냅샷 삭제

1. 인스턴스 상세 화면에서 `인스턴스 구성 요소 > 스냅샷`을 선택합니다.
2. 삭제할 스냅샷 1개를 선택합니다.
3. **`삭제`** 버튼을 클릭합니다.
4. 삭제 확인 메시지를 확인한 뒤 **`삭제`**를 눌러 진행합니다.

## 인스턴스 스냅샷 롤백

스냅샷 롤백은 현재 인스턴스 디스크 상태를 선택한 스냅샷 시점으로 되돌리는 작업입니다.

1. 인스턴스가 `종료됨` 상태인지 확인합니다.
2. 인스턴스 상세 화면에서 `인스턴스 구성 요소 > 스냅샷`을 선택합니다.
3. 롤백할 스냅샷 1개를 선택합니다.
4. **`롤백`** 버튼을 클릭합니다.
5. 확인 메시지를 검토한 뒤 **`롤백`**을 눌러 진행합니다.

> 주의:
> 롤백은 되돌릴 수 없는 작업이므로, 현재 디스크 상태를 계속 보존해야 하는지 먼저 확인하는 것이 좋습니다.

## 네트워크 인터페이스 수동 IP 설정 가이드

이 가이드는 다음과 같은 경우에 특히 필요합니다.

- 실행 중 편집으로 NIC를 추가한 경우
- 다중 NIC 기반으로 서비스 트래픽을 분리하는 경우
- 특정 NIC에 고정 IP를 직접 넣어 운영해야 하는 경우

`실행 중 편집(라이브 업데이트)`으로 네트워크 인터페이스를 추가하면, CloudiA에서는 NIC와 IP를 정상적으로 할당합니다. 하지만 게스트 OS 안에서는 새 인터페이스에 IP가 자동으로 설정되지 않을 수 있으므로, 사용자가 직접 구성해야 합니다.

1. 인스턴스 상세 화면에서 새 인터페이스에 할당된 IP를 확인합니다.
2. 같은 화면에서 해당 인터페이스의 **기본 게이트웨이**를 확인합니다.
3. 필요하면 같은 탭에서 `DNS 서버` 정보도 함께 확인합니다.
4. 게스트 OS 안에서 추가된 인터페이스 이름을 확인합니다.

```bash
ip link
```

5. 아래와 같이 IP를 수동 설정합니다.

```bash
IFACE=eth1
IPADDR=10.0.0.66/24
GW=10.0.0.1

nmcli con add type ethernet ifname "$IFACE" con-name "$IFACE" ip4 "$IPADDR" gw4 "$GW"
nmcli con up "$IFACE"
```

> 참고:
> CIDR과 게이트웨이는 반드시 해당 서브넷 정보에 맞춰 입력해야 합니다.

## 인스턴스 가상 CPU 피닝 편집

가상 CPU 피닝은 인스턴스의 vCPU를 특정 논리 CPU 범위에 고정해 배치하고 싶을 때 사용합니다. 특히 성능 튜닝이나 특정 워크로드의 CPU 배치를 일정하게 유지하고 싶을 때 활용할 수 있습니다.

1. 피닝을 설정할 인스턴스의 상세 화면으로 이동합니다.
2. 인스턴스가 `종료 상태`인지 확인합니다.
3. `인스턴스 구성 요소`에서 **`가상 CPU 피닝`**을 선택합니다.
4. **`편집`** 버튼을 클릭합니다.
5. 필요하면 **`추가`** 버튼으로 새 규칙을 만들고, 기존 규칙은 선택 후 **`삭제`**로 정리합니다.
6. 각 규칙에 대해 `vCPU ID`와 `논리 CPU 범위(시작, 끝)`를 입력합니다.
7. **`편집`** 버튼을 눌러 변경 사항을 저장합니다.

> 참고:
> UI에서는 `종료됨` 또는 `정지됨` 상태이면서 `ISO 파일` 설치 유형이 아닐 때 편집 버튼이 활성화됩니다.
>
> 저장한 가상 CPU 피닝 규칙은 **다음 인스턴스 실행 시 반영되는 설정**으로 이해하고 작업하는 것이 안전합니다.
