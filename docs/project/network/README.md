# 네트워크

프로젝트 `네트워크` 메뉴는 VPC와 서브넷 구성부터 외부 노출, 로드 밸런싱, 내부 DNS까지 프로젝트 단위 네트워크 자원을 관리하는 영역입니다.

## 구성 메뉴
- **VPC**: 프로젝트 최상위 가상 네트워크를 관리합니다.
- **서브넷**: VPC 내부 주소 구간과 퍼블릭/프라이빗 속성을 관리합니다.
- **외부 IP**: 프로젝트에 배정된 외부 IP 현황을 조회합니다.
- **플로팅 IP**: 외부 IP를 인스턴스 또는 내부 IP에 직접 연결합니다.
- **포트 포워딩**: 외부 포트를 내부 리소스로 전달합니다.
- **로드 밸런서**: 리스너와 타겟 그룹 기반 트래픽 분산을 구성합니다.
- **타겟 그룹**: 로드 밸런서가 전달할 대상 집합을 관리합니다.
- **내부 DNS**: VPC 내부에서만 해석되는 전용 DNS 존과 레코드를 관리합니다.

## 권장 작업 순서
1. `VPC`를 생성합니다.
2. `서브넷`을 생성해 주소 구간과 퍼블릭/프라이빗 유형을 정합니다.
3. 외부 서비스 노출 방식에 따라 `플로팅 IP`, `포트 포워딩`, `로드 밸런서` 중 적절한 리소스를 구성합니다.
4. 서비스 간 이름 기반 통신이 필요하면 `내부 DNS`를 추가합니다.

## 참고 사항
- `외부 IP` 메뉴는 조회 전용이며, 실제 사용은 `플로팅 IP`, `포트 포워딩`, `로드 밸런서` 생성 과정에서 이뤄집니다.
- 리소스 간 선후관계가 많으므로 삭제는 보통 `로드 밸런서/포트 포워딩/플로팅 IP/내부 DNS` 정리 후 `서브넷`, 마지막으로 `VPC` 순으로 진행합니다.
