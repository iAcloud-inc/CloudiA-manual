# 내부 DNS
**내부 DNS**는 특정 VPC 네트워크 내부에서만 해석되고 사용되는 전용 내부 DNS입니다.

## 내부 DNS 목록
### 주요 작업
- `생성`: 새로운 내부 DNS를 생성합니다.
- `편집`: 선택한 내부 DNS를 편집합니다.
- `삭제`: 선택한 내부 DNS를 삭제합니다.

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **도메인 이름** | 내부 DNS의 도메인 이름입니다. 클릭 시 **내부 DNS 상세**로 이동합니다. |
| **ID** | 내부 DNS의 고유 ID입니다. |
| **네트워크** | 내부 DNS가 속한 VPC 이름입니다. |
| **레코드 수** | 등록된 레코드 개수입니다. |
| **설명** | 내부 DNS 관리자 작성 설명입니다. |
| **생성 일시** | 내부 DNS가 생성된 날짜와 시간입니다. |

## 내부 DNS 상세
### 상단 요약
- **도메인 이름**: 내부 DNS의 기준 도메인(Zone)입니다. 하위 레코드는 이 도메인을 기준으로 이름이 결합되어 전체 레코드 주소(FQDN)가 됩니다. 예) `app` + `example.com` → `app.example.com`
- **ID**: 내부 DNS의 고유 ID입니다.
- **네트워크**: VPC 이름과 ID가 함께 표시됩니다.
- **레코드 수**: 등록된 레코드 개수입니다.
- **생성 일시**: 내부 DNS가 생성된 날짜와 시간입니다.
- **설명**: 내부 DNS 설명입니다.

### 하단 탭
아래 **레코드** 섹션에서는 현재 내부 DNS에 설정된 **레코드** 목록이 제공됩니다. 각 레코드는 이름(도메인)과 값(IP/별칭/텍스트 등)의 매핑을 의미합니다.  
레코드 정보 표의 우측 상단 **[편집]** 버튼을 클릭하면 레코드를 수정할 수 있는 **레코드 편집 화면**으로 이동합니다.  
(레코드 편집 가이드는 아래 **[내부 DNS 생성·편집 > 레코드 설정]** 항목 참고)

#### 하단 탭 요약
| 컬럼 | 설명 |
| --- | --- |
| **이름** | `레코드 이름.도메인 이름` 형식의 전체 레코드 주소(FQDN)로 표시됩니다. |
| **타입** | 레코드 타입입니다. |
| **값** | A 타입은 선택한 리소스의 IP가 표시되며, CNAME/TXT/SRV는 입력한 값이 표시됩니다. 값이 여러 개인 경우 콤마(`,`)로 구분되어 표시됩니다. |
| **TTL (초)** | TTL(Time To Live) 값(초)입니다. |
| **생성 일시** | 레코드가 생성된 날짜와 시간입니다. |

## 내부 DNS 생성·편집
내부 DNS 생성과 편집은 동일한 화면에서 진행합니다. 생성 화면은 **내부 DNS 생성**, 편집 화면은 **내부 DNS 편집** 제목으로 표시됩니다.
편집 모드에서는 **도메인 이름**과 **VPC**가 비활성화되며, 레코드 영역이 표시되지 않습니다.

### 1. 기본 정보
- **도메인 이름** (필수): 내부 DNS의 도메인 이름을 입력합니다. (예: `example.com`)
- **적용 범위** (필수): `프라이빗`으로 고정 표시됩니다. 선택한 VPC 네트워크 내부에서만 해석되고 사용되는 내부 DNS임을 의미합니다.
- **VPC** (필수): 내부 DNS가 적용될 VPC를 선택합니다.
- **설명** (선택): 내부 DNS 설명을 입력합니다. (최대 200자)

### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
| --- | --- | --- | --- |
| **도메인 이름** | 영문 소문자/숫자/`-`/`.` (최대 80자) | 연속 점(`..`) 불가 <br/> 점으로 시작/끝 불가 <br/> 각 레이블 1~63자<br/> 레이블 하이픈 시작/끝 불가<br/> `xn--`로 시작 불가 | 필수, 생성 후 변경 불가 |
| **적용 범위** | `프라이빗` | 고정 값, 변경 불가 | |
| **VPC** | 프로젝트 내 VPC | 선택 필수 | 필수, 생성 후 변경 불가 |
| **설명** | 최대 200자 | | 선택 |

> ⚠️ 주의  : 내부 DNS **편집 모드**에서는 설명만 편집 할 수 있습니다.

### 2. 레코드 (생성 모드)
생성 화면에서만 레코드 영역이 표시됩니다.

#### 주요 작업
- `추가`: 새 레코드를 추가합니다.
- `편집`: 선택한 1개 레코드를 편집합니다.
- `삭제`: 선택한 레코드를 삭제합니다.

#### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **이름** | `이름.도메인 이름` 형식으로 표시됩니다. |
| **타입** | 레코드 타입입니다. |
| **값** | A 타입은 선택한 리소스의 IP가 표시되며, CNAME/TXT/SRV는 입력한 값이 표시됩니다. |
| **TTL (초)** | TTL 값(초)입니다. |
| **생성 일시** | 레코드가 생성된 날짜와 시간입니다. 생성 전 항목은 `-`로 표시됩니다. |

> ⚠️ 주의  : 기본 NS, SOA 레코드는 자동 생성되며 편집/삭제할 수 없습니다.
- [추가]는 VPC 선택 및 도메인 이름 유효 시 활성화됩니다.

## 레코드 편집
기존 내부 DNS의 레코드를 추가, 편집, 삭제할 수 있는 화면입니다.

### 주요 작업
- `추가`: 새 레코드를 추가합니다.
- `편집`: 선택한 1개 레코드를 편집합니다.
- `삭제`: 선택한 레코드를 삭제합니다.
- `저장`: 변경 사항을 저장합니다.
- `취소`: 저장하지 않고 내부 DNS 상세로 돌아갑니다.

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **이름** | `이름.도메인 이름` 형식으로 표시됩니다. 이름이 없으면 도메인 이름만 표시됩니다. |
| **타입** | 레코드 타입입니다. |
| **값** | A 타입은 선택한 리소스의 IP가 표시되며, CNAME/TXT/SRV는 입력한 값이 표시됩니다. |
| **TTL (초)** | TTL 값(초)입니다. |
| **생성 일시** | 레코드가 생성된 날짜와 시간입니다. |

### 주의사항
- 기본 NS, SOA 레코드는 편집/삭제할 수 없습니다.
- [편집]은 1개 선택, [삭제]는 1개 이상 선택 시 활성화됩니다.
- [저장]은 변경 사항이 있을 때만 활성화됩니다.

---

## 레코드 추가·편집
레코드 목록에서 [추가] 또는 [편집]을 클릭하면 레코드 추가·편집 창이 열립니다.

### 입력 항목
- **이름**: 레코드 이름을 입력합니다. 입력한 이름과 도메인 이름을 합친 **전체 레코드 주소**가 표시됩니다.
- **TTL**: TTL 값을 초 단위로 설정합니다.
- **타입**: `A`, `CNAME`, `TXT`, `SRV` 중에서 선택합니다.
- **값**: 타입에 따라 입력 방식이 달라지며, [추가]로 값을 목록에 등록합니다.

> 💡 참고: "* 실제 서비스에서 접근에 사용되는 전체 DNS 이름입니다."

### 타입별 값 입력
#### A 타입
- **인스턴스**: 인스턴스를 선택한 뒤 **내부 IP 주소**를 선택합니다.
- **로드 밸런서**: `외부 로드 밸런서` 또는 `내부 로드 밸런서`를 선택한 뒤 IP를 선택합니다.
- **포트 포워딩**: 포트 포워딩 목록에서 **외부 IP**를 선택합니다.
- **플로팅 IP**: 플로팅 IP 목록에서 **외부 IP**를 선택합니다.
- **IP 주소**: **내부 IP 주소**를 직접 입력합니다.

#### CNAME 타입
- 값은 1개만 추가할 수 있습니다.

#### TXT, SRV 타입
- 여러 개의 값을 추가할 수 있습니다.

### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
| --- | --- | --- | --- |
| **이름** | 영문 소문자/숫자/`-`/`.` | 도메인 이름 규칙을 따르며, 이미 존재하는 레코드 이름은 사용할 수 없습니다. | 필수 |
| **TTL** | 60~172800 | "TTL(Time To Live)은 60초(1분)에서 172800초(2일) 사이의 값으로 설정할 수 있습니다." | 필수 |
| **타입** | `A` / `CNAME` / `TXT` / `SRV` | 타입 선택 필수 | 필수 |
| **값** | 타입별 입력 방식 | A는 리소스 선택 또는 **내부 IP 주소** 입력, CNAME은 1개만 허용, TXT/SRV는 여러 값 허용, 중복 값 추가 불가, 최대 200자 | 필수 |

### 값 목록
#### A 타입 값 목록
| 컬럼 | 설명 |
| --- | --- |
| **입력 값** | 선택한 리소스의 IP 주소가 표시됩니다. |
| **타입** | 리소스 타입이 표시됩니다. |
| **삭제** | 값을 목록에서 제거합니다. |

#### CNAME/TXT/SRV 타입 값 목록
| 컬럼 | 설명 |
| --- | --- |
| **입력 값** | 입력한 값이 표시됩니다. |
| **삭제** | 값을 목록에서 제거합니다. |

### 주의사항
- A 타입에서 **IP 주소**를 직접 입력하는 경우, VPC CIDR 범위 내의 IPv4만 허용됩니다.
- 값은 [추가]로 목록에 등록해야 [추가]/[편집] 완료가 가능합니다.
- "A, TXT, SRV 타입은 여러 개의 값을 입력할 수 있습니다."
- "CNAME 타입은 하나의 값만 입력할 수 있습니다."

---
## 내부 DNS 삭제
목록 화면에서 `삭제`를 클릭하면 선택한 내부 DNS의 정보가 표시된 삭제 확인 창에서 삭제 할 수 있습니다.

> ⚠️ **주의:** 삭제는 **복구할 수 없습니다.** 사용 여부를 반드시 확인 후 진행하세요. 
> 💡 참고: 기본 NS 및 SOA 레코드를 제외한 레코드가 존재하는 경우 내부 DNS를 삭제할 수 없습니다.
