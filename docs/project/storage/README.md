# 스토리지

프로젝트 `스토리지` 메뉴는 프로젝트 내 블록, 파일 시스템, 이미지, 스냅샷을 관리하는 영역입니다. 가상머신과 쿠버네티스에 필요한 저장 자원을 준비하고 운영할 때 사용합니다.

## 구성 메뉴
- **이미지**: 프로젝트 전용 OS 이미지를 등록하고 복제, 삭제합니다.
- **블록**: 인스턴스에 연결하는 블록 스토리지를 생성하고 관리합니다.
- **파일 시스템**: NFS 또는 VIRTIOFS 기반 파일 시스템을 생성하고 연결 상태를 관리합니다.
- **스냅샷**: 인스턴스에서 생성한 스냅샷을 조회하고 이미지 생성에 활용합니다.

## 권장 작업 
1. 커스텀 부팅 이미지가 필요하면 `이미지`를 관리합니다.
2. 인스턴스 운영 방식에 맞춰 `블록` 또는 `파일 시스템`을 먼저 준비합니다.
3. 운영 중 백업과 복원을 위해 `스냅샷`을 함께 관리합니다.

## 참고 사항
- 블록, 파일 시스템, 이미지, 스냅샷은 서로 참조 관계가 있으므로 삭제 전에 연결 대상을 먼저 확인해야 합니다.
- 저장 자원 성격이 다르므로 인스턴스 전용인지, 공유용인지 구분해 설계하는 것이 좋습니다.
