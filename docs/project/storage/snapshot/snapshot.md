# 스냅샷

`스냅샷`은 모든 인스턴스의 스냅샷을 한 화면에서 관리할 수 있는 인터페이스입니다.  
인스턴스 목록에서 사전에 생성한 스냅샷이 존재해야 합니다. (**인스턴스 사용자 가이드 - 인스턴스 목록 - 더보기** 참고)   
사용자는 스냅샷을 활용하여 인스턴스를 특정 시점으로 복원하거나, OS 이미지를 생성하여 사용할 수 있습니다.

## 화면 개요
- `목록 화면`: 스냅샷 현황을 조회하고 삭제, 이미지 생성 작업을 시작합니다.
- `이미지 생성 모달`: 선택한 스냅샷을 기반으로 새 프로젝트 이미지를 생성합니다.

## 스냅샷 목록
### 주요 작업
- `이미지 생성`: 선택한 1개 스냅샷으로부터 새 이미지를 생성합니다. 해당 이미지는 **스토리지 > 이미지** 메뉴에서 확인 가능하며 인스턴스 생성 시 사용할 수 있습니다.
- `삭제`: 선택한 1개 스냅샷을 삭제합니다.

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **이름** | 스냅샷 이름입니다. |
| **ID** | 스냅샷 고유 ID입니다. |
| **부팅가능여부** | 스냅샷을 OS 디스크로 사용할 수 있는지 여부입니다. |
| **크기** | 스냅샷 크기입니다. |
| **블록** | 원본 인스턴스의 블록 디바이스 이름입니다. |
| **타입** | 원본 디스크 포맷 형식입니다. |
| **생성 일시** | 스냅샷 생성 시각입니다. |

## 스냅샷으로 이미지 생성
스냅샷 목록 화면에서 특정 스냅샷을 선택하여 OS 이미지를 생성할 수 있습니다. **스토리지 > 이미지**에서 생성된 이미지를 확인할 수 있습니다.
> 이미지 생성 완료에는 몇 분의 시간이 소요될 수 있으며 이미지가 서버에서 생성 중인 경우에는 OS 타입 UNKNOWN, 크기 0GB로 나타납니다.

### 입력 항목
- **스냅샷 이름**: 선택한 스냅샷 이름을 읽기 전용으로 표시합니다.
- **스토리지 도메인**: 생성할 이미지를 저장할 스토리지 도메인을 선택합니다.
- **이미지 이름**: 새 이미지 이름을 입력합니다. 비워두면 스냅샷 이름을 기반으로 생성될 수 있습니다.

### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
| --- | --- | --- | --- |
| **스냅샷 이름** | 선택된 스냅샷 값 | 읽기 전용입니다. 직접 수정할 수 없습니다. | 기준 스냅샷입니다. |
| **스토리지 도메인** | 현재 프로젝트에서 선택 가능한 스토리지 도메인 | 필수입니다. 1개를 선택합니다. | 이미지 저장 위치입니다. |
| **이미지 이름** | 영문, 숫자, `-`, `_` 조합의 문자열 | 선택 입력입니다. 40자 이내로 입력합니다. | 새 이미지 식별용 이름입니다. |

### 생성 시 참고 사항
- 이미지 생성 후 결과는 `스토리지 > 이미지` 메뉴에서 확인합니다.
- 생성 직후에는 이미지가 준비 중 상태로 표시될 수 있습니다.

## 스냅샷 삭제
- 삭제는 한 번에 1개만 수행합니다.
- 삭제 전에 해당 스냅샷을 복구 또는 이미지 생성에 사용할 필요가 없는지 확인해야 합니다.
