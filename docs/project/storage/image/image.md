# 이미지
**이미지** 기능은 가상머신 생성에 필요한 운영체제(OS) 이미지를 업로드하고 관리할 수 있는 공간입니다.  
사용자는 프로젝트 내에서 이미지 목록을 확인하고, 새 이미지를 추가하거나 기존 이미지를 삭제할 수 있습니다.  
또한, 관리자가 추가한 이미지는 모든 프로젝트에서 공통적으로 사용할 수 있습니다.

## 이미지 목록

### 주요 작업
- `생성`: 이미지를 생성합니다.
- `삭제`: 선택한 이미지를 삭제합니다.
- `복제`: 선택한 이미지를 다른 스토리지 도메인으로 복제합니다.

> 💡 **참고:** 목록은 단일 선택으로 동작하며, 항목 선택 시 `삭제`, `복제` 버튼이 활성화됩니다.

### 검색
- **검색 조건**: `이름`
- **검색어 입력** 후 `검색`을 실행합니다.

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **이름** | 이미지 이름입니다.<br>생성 시 사용한 이미지 파일 이름과 동일합니다. |
| **ID** | 이미지 고유 ID입니다. |
| **분류** | 이미지 분류입니다.<br>`프로젝트 이미지`, `공용 이미지`로 구분됩니다. |
| **OS 타입** | 이미지의 OS 타입(Variant)입니다. |
| **크기** | 이미지 파일 크기입니다. |
| **스토리지 도메인** | 이미지가 저장된 스토리지 도메인입니다. |
| **생성 일시** | 이미지를 추가한 일시입니다. |

### 이미지 분류
이미지 분류는 프로젝트 이미지와 공용 이미지로 구성되어 있습니다.   
프로젝트 이미지는 사용자가 추가하여 특정 프로젝트 내에서만 사용할 수 있는 이미지이고,   
공용 이미지는 관리자가 추가하여 모든 프로젝트에서 공통적으로 사용할 수 있는 이미지입니다.

## 이미지 생성
### 기본 정보
- **스토리지 도메인**: 이미지가 사용할 스토리지 도메인을 선택합니다.
- **이미지**: `.qcow2`, `.img` 확장자의 운영체제(OS) 이미지를 업로드합니다.

### 입력 필드 유효성
| 필드 | 허용 값 | 검증 규칙 | 비고 |
| --- | --- | --- | --- |
| **스토리지 도메인** | 선택 가능한 스토리지 도메인 | 1개 선택 필수 | 필수 |
| **이미지 파일** | `.qcow2` 또는 `.img` 파일 | 파일 선택 필수 | 필수 |

### 생성 버튼 활성화 조건
- 스토리지 도메인과 이미지 파일을 모두 입력하면 `생성` 버튼이 활성화됩니다.

### 이미지 활용
- **가상머신 > 인스턴스 생성**에서 인스턴스 생성을 위한 OS 이미지로 활용합니다.
- **가상머신 > 시작 템플릿**에서 인스턴스 템플릿 생성을 위한 OS 이미지로 활용합니다.

## 이미지 복제
1. 목록에서 복제할 이미지를 선택합니다.
2. `복제`를 클릭합니다.
3. 복제 대상 스토리지 도메인을 선택합니다.
4. 복제 요청을 실행합니다.

> ⚠️ **주의:** 원본 이미지와 동일한 스토리지 도메인은 복제 대상으로 선택할 수 없습니다.

## 이미지 삭제
1. 목록에서 삭제할 이미지를 선택합니다.
2. `삭제`를 클릭합니다.
3. 삭제 확인 창에서 내용을 확인한 뒤 삭제를 진행합니다.

> ⚠️ **주의:** 프로젝트 영역에서는 `프로젝트 이미지`만 삭제할 수 있습니다. `공용 이미지`는 관리자 영역의 **공용 이미지 관리**에서 삭제할 수 있습니다.
