# 파일 시스템

`파일 시스템`은 프로젝트에서 공유 가능한 파일 기반 저장 공간을 생성하고 관리하는 화면입니다. `NFS`와 `VIRTIOFS` 타입을 지원합니다.
`NFS`는 NFS 서버를 생성한 뒤, 해당 서버의 공유 경로를 인스턴스에 마운트하는 방식입니다.
`VIRTIOFS`는 스토리지 도메인의 디렉토리를 인스턴스에 직접 마운트하는 방식입니다.


## 화면 개요
- `목록 화면`: 파일 시스템 현황을 조회하고 생성, 편집, 삭제 작업을 시작합니다.
- `상세 화면`: 기본 정보와 타입별 하단 정보를 확인합니다.
- `생성/편집 화면`: 이름, 타입, 크기, 스토리지 도메인과 네트워크 정보를 입력하거나 수정합니다.

## 파일 시스템 목록
### 주요 작업
- `생성`: 새로운 파일 시스템을 생성합니다.
- `편집`: 선택한 1개 파일 시스템을 편집합니다. (`NFS`: 이름/크기, `VIRTIOFS`: 이름)
- `삭제`: 선택한 1개 파일 시스템을 삭제합니다.

### 테이블 컬럼
| 컬럼 | 설명 |
| --- | --- |
| **이름** | 파일 시스템 이름입니다. |
| **ID** | 파일 시스템 고유 ID입니다. |
| **타입** | 파일 시스템 타입입니다. `NFS`, `VIRTIOFS` 중 하나입니다. |
| **크기** | 파일 시스템 크기입니다. |
| **연결된 인스턴스 수** | 현재 파일 시스템이 연결된 인스턴스 개수입니다. |
| **생성 일시** | 파일 시스템 생성 시각입니다. |

## 파일 시스템 상세
파일 시스템 상세 화면에서는 선택한 파일 시스템의 상세 정보를 확인할 수 있습니다.  
파일 시스템의 타입이 `NFS`인 경우에는 연결된 네트워크 인터페이스 규칙 정보를, `VIRTIOFS`인 경우에는 스토리지 도메인 정보를 조회할 수 있습니다.

### 상단 요약
- **이름**: 파일 시스템 이름입니다.
- **ID**: 파일 시스템 고유 ID입니다.
- **타입**: 파일 시스템 타입입니다. `NFS`, `VIRTIOFS` 중 하나입니다.
- **크기**: 파일 시스템 크기입니다.
- **연결된 인스턴스 수**: 현재 연결된 인스턴스 개수입니다.
- **생성 일시**: 파일 시스템 생성 시각입니다.

### 하단 탭

  | 스토리지 도메인 이름 | 해당 파일 시스템이 사용하는 스토리지 도메인의 이름을 나타냅니다. |

  | 서브넷 | 해당 파일 시스템이 사용하는 서브넷을 나타냅니다. |
  | 보안그룹 | 해당 파일 시스템이 사용하는 보안그룹을 나타냅니다. |

- **인스턴스 정보**

  | 항목 | 설명 |
  |------|------|
  | 이름 | 연결된 인스턴스의 이름을 나타냅니다. |
  | ID | 연결된 인스턴스의 ID를 나타냅니다. |
  | 상태 | 연결된 인스턴스의 상태을 나타냅니다. |
  | vCPU | 연결된 인스턴스에 할당된 vCPU를 나타냅니다. |
  | 메모리 | 연결된 인스턴스에 할당된 메모리를 나타냅니다. |
  | 마운트 포인트 | 연결된 인스턴스에서의 파일시스템 마운트 포인트를 나타냅니다. |
  | 생성 일시 | 연결된 인스턴스가 생성된 일시를 나타냅니다. |

## 파일 시스템 생성 및 편집
파일 시스템 생성 화면에서는 새로운 파일 시스템을 생성할 수 있으며, 편집 화면에서는 기존 파일 시스템의 일부 정보를 수정할 수 있습니다.  
사용자는 파일 시스템의 기본 정보를 입력해야 하고 파일 시스템의 타입을 `NFS`로 선택하는 경우 추가적으로 `네트워크 인터페이스 구성` 정보를 입력합니다.    
이 경우에는 연결 가능한 `VPC`, `서브넷`, `보안그룹`이 사전에 구성되어 있어야 합니다.  
반면, 파일 시스템의 타입을 `VIRTIOFS`로 선택하는 경우 추가적으로 `스토리지` 정보를 입력합니다.

### 기본 정보
- **이름**: 파일 시스템의 이름을 입력합니다.
- **타입**: 파일 시스템의 타입을 선택합니다. 선택 가능한 타입은 `NFS`, `VIRTIOFS`입니다.
- **크기**: `NFS` 타입의 파일 시스템 크기를 입력합니다. 크기 단위는 GB, MB입니다.
- **스토리지 도메인**: 파일 시스템이 구성될 스토리지 도메인을 선택합니다.

> ⚠️ **주의:** `VIRTIOFS`는 `CEPH` 스토리지 도메인을 지원하지 않습니다.

### `NFS` 타입 네트워크 정보
- **VPC**: 파일 시스템이 사용할 VPC를 선택합니다.
- **서브넷**: 파일 시스템이 사용할 서브넷을 선택합니다.
- **IP 주소**: `자동 할당` 또는 `수동 할당`으로 설정합니다. <br>`수동 할당`으로 선택하는 경우 서브넷의 IP/CIDR 규칙에 맞는 IP 주소를 입력합니다.<br>예시) 서브넷의 IP/CIDR가 10.0.0.0/24인 경우 IP 주소 10.0.0.10
- **보안그룹**: 파일 시스템에 적용할 보안그룹을 선택합니다.

### 입력 필드 유효성
| 필드 | 허용 값                                   | 검증 규칙 | 비고 |
| --- |----------------------------------------| --- | --- |
| **이름** | 영문, 숫자, `-`, `_` 조합의 문자열               | 필수입니다. 20자 이내로 입력합니다. | 파일 시스템 식별용 이름입니다. |
| **타입** | `NFS`, `VIRTIOFS`                      | 필수입니다. 두 값 중 하나를 선택합니다. | 생성 후 변경할 수 없습니다. |
| **크기** | 0 이상의 숫자 + 단위 (MB, GB)                 | `NFS` 타입에서 필수입니다. | 편집 시 축소는 지원하지 않습니다. |
| **스토리지 도메인** | 현재 프로젝트에서 선택 가능한 스토리지 도메인              | 필수입니다. `VIRTIOFS`에서는 `CEPH`를 선택할 수 없습니다. | 생성 후 변경할 수 없습니다. |
| **VPC** | 현재 프로젝트의 VPC                           | `NFS` 타입에서 필수입니다. | 네트워크 연결 정보입니다. |
| **서브넷** | 선택한 VPC의 서브넷                           | `NFS` 타입에서 필수입니다. | 네트워크 연결 정보입니다. |
| **IP 주소** | `자동 할당` 또는 `수동 할당`| `NFS` 타입에서 필수입니다. 수동 할당 시 서브넷 범위 내 유효한 IPv4 주소여야 합니다. | 서비스 IP입니다. |
| **보안그룹** | 현재 프로젝트의 보안그룹                          | `NFS` 타입에서 필수입니다. | 네트워크 보안 규칙입니다. |

### 편집 시 주의사항
- 편집 화면에서는 `NFS`는 이름과 크기를, `VIRTIOFS`는 이름만 수정할 수 있습니다.
- 타입, 스토리지 도메인, 네트워크 정보는 생성 후 변경할 수 없습니다.
- `NFS` 크기는 기존 값 이상으로만 수정할 수 있습니다.

## 파일 시스템 삭제
- 삭제는 한 번에 1개만 수행합니다.
- 연결된 인스턴스가 남아 있으면 삭제 전 영향도를 확인해야 합니다.
