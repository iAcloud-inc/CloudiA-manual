# 네트워크 토폴로지

`네트워크 토폴로지`는 선택한 VPC 내 리소스들의 논리적 연결 구조를 한눈에 파악할 수 있는 시각화 도구입니다.
사용자는 VPC, 서브넷, 로드 밸런서, 인스턴스 간의 연결 관계를 확인하고, 각 리소스의 핵심 네트워크 정보를 우측 상세 패널에서 즉시 조회할 수 있습니다.

> 💡 **참고:** 네트워크 토폴로지에서는 다음과 같은 정보를 확인 할 수 있습니다. 
> - 네트워크 구조 파악: VPC 내 퍼블릭 및 프라이빗 서브넷의 배치 구조를 직관적으로 이해할 수 있습니다.
> - 리소스 분포 확인: 특정 서브넷에 집중된 로드 밸런서와 인스턴스 현황을 파악하여 자원 배분을 점검할 수 있습니다.
> - 트래픽 흐름 추적: 로드 밸런서와 연결된 타겟 인스턴스들을 확인하며 서비스의 네트워크 흐름을 검토할 수 있습니다.
> - 외부 접점 식별: 인스턴스에 적용된 외부 IP 연결 방식이 플로팅 IP인지 포트 포워딩인지 명확히 구분하여 관리할 수 있습니다.


## 화면 개요
- `상단 툴바`: 조회할 VPC를 선택하고, 리소스 이름으로 검색합니다.
- `토폴로지`: 선택한 VPC 내부 구조를 시각적으로 표시합니다. 리소스를 선택하면 우측 패널에서 해당 리소스의 상세 정보를 확인할 수 있습니다.

## 상단 툴바
### VPC 선택
- 현재 프로젝트에 생성된 VPC 목록 중 하나를 선택합니다.
- VPC를 선택하면 네트워크 토폴로지도 해당 VPC 기준으로 다시 갱신됩니다.

### 검색
- 리소스 이름을 검색할 수 있습니다.
- 검색어를 입력한 뒤 `검색`버튼을 클릭하거나 `Enter`를 누르면 검색이 적용됩니다.
- VPC, 서브넷, 로드 밸런서, 인스턴스 이름에 대해 부분 일치 검색합니다.
- 검색 결과가 있으면 일치한 리소스가 화면에서 강조 표시됩니다. VPC 중심의 토폴로지 화면에서 로드 밸런서나 인스턴스가 검색되면, 해당 리소스가 연결된 서브넷도 함께 강조되며 서브넷에는 일치한 리소스 수가 표시됩니다. 이 상태에서 해당 서브넷을 클릭해 서브넷 중심 화면으로 전환해도 검색어는 유지되어 탐색을 이어갈 수 있습니다.
> 💡 **참고:** 검색된 상태에서 서브넷이나 VPC를 클릭하면 검색어가 유지되어 같은 검색 기준으로 탐색을 계속할 수 있습니다. 반면, 다른 로드 밸런서나 인스턴스를 클릭하면 검색이 자동으로 초기화되어 선택한 리소스를 중심으로 토폴로지를 바로 확인할 수 있습니다.

## 토폴로지
토폴로지는 선택한 VPC 내부의 네트워크 구조를 계층적으로 보여줍니다. 최초 화면에는 선택한 VPC와 그 안에 속한 서브넷이 표시되며, 서브넷은 `퍼블릭 서브넷`과 `프라이빗 서브넷` 유형에 따라 구분되어 배치됩니다. VPC 노드에서는 연결된 서브넷 수와 함께 VPC 기준의 로드 밸런서, 인스턴스 현황을 확인할 수 있습니다.

특정 서브넷을 선택하면 해당 서브넷을 중심으로 화면이 전환되며, 그 서브넷에 내부 IP를 할당받은 로드 밸런서와 네트워크 인터페이스를 가진 인스턴스가 함께 표시됩니다. 이를 통해 사용자는 서브넷 단위의 네트워크 연결 구조를 보다 구체적으로 확인할 수 있습니다.


### 기본 구성
- `VPC`: 현재 선택한 VPC를 나타냅니다.
- `서브넷`: VPC 아래에 연결된 서브넷을 표시합니다.
- `로드 밸런서`: 특정 서브넷을 선택하여 해당 서브넷 중심 화면으로 전환했을 때, `내부 IP 주소`가 해당 서브넷 대역에 속한 로드 밸런서가 표시됩니다.
- `인스턴스`: 특정 서브넷을 선택하여 해당 서브넷 중심 화면으로 전환했을 때, 네트워크 인터페이스 IP가 해당 서브넷 대역에 속한 인스턴스가 표시됩니다. 
> 💡 **참고:** 인스턴스는 VPC 내 여러 서브넷의 네트워크 인터페이스를 가질 수 있습니다. 따라서, 동일한 인스턴스가 여러 서브넷 중심의 토폴로지 화면에서 중복되어 표시될 수 있습니다.

### 토폴로지에서 할 수 있는 동작
- `VPC 클릭`: 우측에 VPC 정보 패널을 엽니다.
- `서브넷 클릭`: 해당 서브넷 중심으로 화면이 전환되며, 연결된 로드 밸런서와 인스턴스를 확인할 수 있습니다. 우측에는 서브넷 정보 패널이 열립니다. 서브넷 중심 화면에서 해당 서브넷을 감싸는 VPC를 클릭하면 다시 VPC 중심 화면으로 돌아갑니다.
- `로드 밸런서 클릭`: 우측에 로드 밸런서 정보 패널을 열고, 해당 로드 밸런서와 같은 서브넷 내 타겟 인스턴스를 강조 표시합니다. 관련 없는 로드 밸런서와 인스턴스는 흐리게 표시됩니다.
- `인스턴스 클릭`: 우측에 인스턴스 정보 패널을 엽니다.
- `빈 공간 클릭`: 현재 열린 정보 패널을 닫습니다.
- `줌 컨트롤`: 그래프 확대/축소와 화면 맞춤을 사용할 수 있습니다.

### 서브넷 확장 동작
- 기본 상태에서는 VPC와 서브넷 중심으로 전체 구조를 봅니다.
- 특정 서브넷을 선택하면 해당 서브넷을 중심으로 화면이 전환되며, 그 안에 연결된 로드 밸런서와 인스턴스가 함께 표시됩니다.
- 서브넷이 확장된 상태에서 다른 서브넷을 선택하면 선택 대상이 해당 서브넷으로 전환됩니다.

## 우측 정보 패널
그래프에서 리소스를 선택하면 우측에 상세 정보 패널이 열립니다.

### VPC 정보 패널
- **이름**: VPC 이름입니다.
- **IP/CIDR**: VPC 전체 주소 대역입니다.
- **서브넷 목록**: VPC에 속한 서브넷 목록이 표시되며, 서브넷의 정보(이름, IP/CIDR, 타입, 주소 사용 현황, 연결된 리소스 수)를 확인할 수 있습니다. 특정 서브넷을 클릭하면 해당 서브넷을 중심으로 토폴로지 화면이 전환됩니다.

### 서브넷 정보 패널
- **이름**: 서브넷 이름입니다.
- **타입**: 서브넷의 타입입니다. (`퍼블릭 서브넷` 또는 `프라이빗 서브넷`)
- **IP/CIDR**: 서브넷 IP 주소 대역입니다.
- **주소 사용 현황**: 사용 중인 IP 수, 전체 IP 수, 사용률(%)이 함께 표시됩니다.
- **사용률 바**: IP 사용률을 막대 그래프로 표시합니다.
- **서브넷 리소스 현황**: 연결된 로드 밸런서 수와 인스턴스 수를 표시합니다.

### 로드 밸런서 정보 패널
- **이름**: 로드 밸런서 이름입니다.
- **유형**: `외부` 또는 `내부`
- **외부 IP 주소**: 연결된 외부 IP가 있으면 표시됩니다.
- **내부 IP 주소**: 로드 밸런서 내부 IP입니다.
- **리스너 수**: 연결된 리스너 개수입니다.

리스너별로 다음 정보를 확인할 수 있습니다.
- **프로토콜 / 포트**
- **타겟 그룹 프로토콜 / 포트**
- **연결된 인스턴스 목록**
- 각 인스턴스의 **이름**, **상태**, **연결 IP**
- 로드 밸런서가 속한 서브넷 기준으로 `현재 서브넷` 또는 `다른 서브넷` 관계 정보를 확인할 수 있습니다.

### 인스턴스 정보 패널
- **이름**: 인스턴스 이름입니다.
- **상태**: `실행 중`, `종료됨`, `생성 중` 등 실행 상태가 표시됩니다.
- **네트워크 인터페이스**: 인스턴스에 연결된 네트워크 인터페이스 목록입니다.
> 💡 **참고:** 인스턴스는 VPC 내 여러 서브넷의 네트워크 인터페이스를 가질 수 있습니다. 각 네트워크 인터페이스 IP 옆에는 현재 펼쳐 본 서브넷을 기준으로 `현재 서브넷` 또는 `다른 서브넷` 관계 정보가 함께 표시되므로, 해당 IP가 현재 서브넷 대역에 속하는지 다른 서브넷 대역에 속하는지 구분할 수 있습니다.

인스턴스에 외부 IP가 연결된 경우 외부 IP 주소 목록이 표시되며, 연결 방식에 따라 상세 정보를 확인할 수 있습니다.
- **플로팅 IP**: 외부 IP 주소 정보 제공
- **포트 포워딩**: 외부 IP 주소와 함께 내·외부 포트 및 프로토콜 정보 제공
