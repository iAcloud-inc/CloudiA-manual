# 프로젝트 홈

프로젝트 홈은 선택한 프로젝트의 기본 정보, 자원 사용량, 최근 이벤트를 한 화면에서 확인하는 요약 대시보드입니다.

## 화면 구성
- 상단에 현재 프로젝트와 경로(`홈 > 프로젝트 홈`)가 표시됩니다.
- 좌측 패널에는 `프로젝트 정보`, `리소스 모니터링` 카드가 표시됩니다.
- 우측 패널에는 `최근 이벤트` 카드가 표시됩니다.

## 상단 프로젝트 선택
- 헤더의 프로젝트 드롭다운에서 접근 가능한 프로젝트를 선택할 수 있습니다.
- 프로젝트를 변경하면 `프로젝트 홈`으로 이동하면서 선택한 프로젝트 ID(`pid`) 기준으로 화면이 갱신됩니다.
- 프로젝트 영역(`/project/*`) 내 화면 이동 시에는 현재 `pid`가 자동으로 유지됩니다.

## 프로젝트 정보
- 데이터는 프로젝트 상세 API(`/projects/{projectId}`)를 기준으로 표시됩니다.
- 현재 화면에서는 다음 항목을 제공합니다.
- `ID`: 프로젝트 식별자
- `설명`: 프로젝트 설명

## 리소스 모니터링
- 4개의 원형 차트로 자원 사용량을 표시합니다.
- `vCPU`: 사용량 / 할당량
- `메모리`: 사용량 / 할당량
- `외부 IP`: 사용량 / 할당량
- `디스크`: 사용량 / 할당량
- 차트 중앙에는 `사용량 / 할당량` 수치가 표시되며, 색상으로 사용 비율을 직관적으로 확인할 수 있습니다.
- 메모리와 디스크 값은 MiB 기준 원본 값을 화면 표시 단위(GiB/TiB 포함)로 자동 변환하여 보여줍니다.

## 최근 이벤트
- 최근 이벤트 API(`/projects/{projectId}/events/dashboard`) 결과를 사용합니다.
- 화면에는 최대 7건이 표시됩니다.
- 각 항목은 이벤트명, 사용자, 메시지, 발생 시각을 표시합니다.
- 시각은 사용자 로컬 타임존 기준 `MM-DD HH:mm` 형식으로 표시됩니다.
- 이벤트 레벨(`INFO`, `WARN`, `ERROR`, `CRITICAL`)에 따라 점 색상이 달라지며, 오류 계열(`ERROR`, `CRITICAL`)은 행 배경이 강조됩니다.
- `더보기 >`를 클릭하면 `모니터링 > 이벤트` 화면으로 이동하여 전체 이벤트를 확인할 수 있습니다.
- 표시할 이벤트가 없으면 빈 상태 문구 `데이터가 존재하지 않습니다.`가 나타납니다.
