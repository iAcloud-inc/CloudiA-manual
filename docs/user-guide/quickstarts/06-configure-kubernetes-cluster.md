# 쿠버네티스 클러스터 구성하기

이 문서는 Kubernetes 클러스터 생성 전 레지스트리 준비부터 워커 노드 추가까지의 절차를 안내합니다.

## 사전 준비
- 프로젝트 리소스 여유
  - 예: vCPU 10 이상, RAM 20GB 이상
- VPC, 서브넷 준비
  - 클러스터와 레지스트리에 사용할 서브넷 1개
- `ca.crt`, `ca.key` 파일 준비 ([참고](https://teams.microsoft.com/l/message/19:0decee3a-5a1e-4989-9593-9d851528f665_a0e71533-8bb1-4adb-a2ad-046796346d90@unq.gbl.spaces/1776147400302?context=%7B%22contextType%22%3A%22chat%22%7D))

## 개요
1. [레지스트리 준비](#step-1)
2. [클러스터(컨트롤 플레인) 생성](#step-2)
3. [클러스터 실행 상태 확인](#step-3)
4. [워커 노드 추가](#step-4)
5. [워커 노드 READY 상태 확인](#step-5)

<a id="step-1"></a>
## 1단계: 레지스트리 준비
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `쿠버네티스 > 클러스터`로 이동한 뒤 `생성` 버튼을 클릭합니다.
2. VPC를 선택합니다.
3. 레지스트리 정보 영역의 상태를 확인합니다.
4. `해당 VPC에 레지스트리가 없으니 생성하세요.` 메시지가 표시되면 `레지스트리 생성` 버튼을 클릭합니다.
5. 서브넷 선택 모달에서 레지스트리 생성을 위한 서브넷을 선택하고 `생성`을 클릭합니다.
6. 레지스트리 상태가 생성 중에서 생성 완료로 바뀔 때까지 대기합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| VPC | `example-vpc` |
| 서브넷 | `example-subnet` |

### 확인
- 레지스트리 생성 중 메시지가 표시됩니다.
- 완료 후 `해당 VPC에 레지스트리가 존재합니다.` 메시지가 표시됩니다.
- 레지스트리 생성 버튼이 비활성화됩니다.

<a id="step-2"></a>
## 2단계: 클러스터(컨트롤 플레인) 생성
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. 클러스터 생성 화면에서 클러스터 기본 정보를 입력합니다.
2. 네트워크(서브넷), 인스턴스 구성, 버전, 컨트롤 플레인 개수를 설정합니다.
3. 사용자 계정(이름/비밀번호)과 CA 파일을 등록합니다.
4. `생성` 버튼을 클릭합니다.

### 입력 예시
| 항목 | 값                     |
|---|-----------------------|
| 클러스터 이름 | `example-cluster-001` |
| VPC | `example-vpc`         |
| 서브넷 | `example-subnet`      |
| 인스턴스 타입 | `t2.medium`           |
| Kubernetes 버전 | `1.35`                |
| 컨트롤 플레인 개수 | `3`                   |
| 사용자 이름 | `cloudia`             |
| 비밀번호 | `cloudia`             |
| CA 인증서 | `ca.crt`              |
| 개인 키 | `ca.key`              |

### 확인
- 클러스터가 목록에 `생성 중` 상태로 표시됩니다.

<a id="step-3"></a>
## 3단계: 클러스터 실행 상태 확인
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. 클러스터 목록에서 생성한 클러스터 상태를 확인합니다.
2. 상태 전환이 완료될 때까지 대기 후 재조회합니다.

### 확인
- 클러스터 상태가 `실행 중`으로 전환됩니다.
- 컨트롤 플레인 수가 `3/3`으로 표시됩니다.

<a id="step-4"></a>
## 4단계: 워커 노드 추가
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. 대상 클러스터 상세 화면으로 이동합니다.
2. `컴퓨트` 탭을 선택합니다.
3. `워커 노드 추가` 버튼을 클릭합니다.
4. 워커 노드 구성과 계정 정보를 입력합니다.
5. `생성` 버튼을 클릭합니다.

### 입력 예시
| 항목 | 값                     |
|---|-----------------------|
| 대상 클러스터 | `example-cluster-001` |
| 워커 노드 개수 | `2`                   |
| 인스턴스 타입 | `t2.medium`           |
| 사용자 이름 | `cloudia`             |
| 비밀번호 | `cloudia`             |

### 확인
- 컴퓨트 탭 노드 목록에 신규 워커 노드가 추가됩니다.
- 초기 상태가 `생성 중`으로 표시될 수 있습니다.

<a id="step-5"></a>
## 5단계: 워커 노드 READY 상태 확인
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. 컴퓨트 탭 노드 목록을 재조회합니다.
2. 신규 워커 노드의 상태와 역할을 확인합니다.

### 확인
- 역할이 `Worker`로 표시됩니다.
- 상태가 `실행 중(READY)`으로 전환됩니다.
