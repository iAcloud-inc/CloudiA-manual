# 가상 네트워크 생성과 Linux 인스턴스 생성하기

이 문서는 프로젝트 사용자 관점에서 VPC/서브넷 생성 후 Linux 인스턴스를 생성하고 실행하는 절차를 안내합니다.

## 사전 준비
- [프로젝트 생성하기](./02-create-project.md) 완료
- `example-member` 계정이 `example-project`의 `project-owner`
- 외부 IP 쿼터 1 이상
- Linux 이미지 준비
  - 예: `Rocky-9-GenericCloud-Base.latest.x86_64.qcow2`

## 개요
1. [VPC 생성](#step-1)
2. [퍼블릭 서브넷 생성](#step-2)
3. [Linux 인스턴스 생성](#step-3)
4. [인스턴스 실행](#step-4)

<a id="step-1"></a>
## 1단계: VPC 생성
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `네트워크 > VPC`로 이동합니다.
2. `생성` 버튼을 클릭합니다.
3. 이름, IP/CIDR, 설명을 입력하고 생성합니다.

### 입력 예시
| 항목      | 값 |
|---------|---|
| 이름      | `example-vpc` |
| IP/CIDR | `10.0.0.0/16` |
| 설명      | `vpc create test` |

### 확인
- `example-vpc`가 목록에 표시됩니다.
- 기본 보안그룹이 `example-vpc-default-sg`자동 생성됩니다.
- 외부 IP(VPC 용도)가 자동 할당됩니다.

<a id="step-2"></a>
## 2단계: 퍼블릭 서브넷 생성
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `네트워크 > 서브넷`으로 이동합니다.
2. `생성` 버튼을 클릭합니다.
3. 서브넷 정보를 입력하고 생성합니다.

### 입력 예시
| 항목      | 값                        |
|---------|--------------------------|
| 이름      | `example-subnet`         |
| VPC     | `example-vpc`            |
| IP/CIDR | `10.0.1.0/24`            |
| 타입      | `퍼블릭 서브넷`                |
| DNS 서버  | `10.0.0.2`                |
| 설명      | `public subnet for example` |

### 확인
- `example-subnet`이 목록에 표시됩니다.
- IP/CIDR, 타입, DNS 값이 입력값과 일치합니다.

<a id="step-3"></a>
## 3단계: Linux 인스턴스 생성
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 인스턴스`로 이동합니다.
2. `생성` 버튼을 클릭합니다.
3. 인스턴스 타입, 이미지, 디스크, 네트워크, 사용자 계정을 입력합니다.
4. `생성` 버튼을 클릭합니다.

### 입력 예시
| 항목 | 값                                         |
|---|-------------------------------------------|
| 이름 | `example-instance-001`                    |
| 인스턴스 타입 | `t2.medium`                               |
| 이미지 | `Rocky-9-GenericCloud-Base.latest.x86_64` |
| 이미지 디스크 크기 | `30 GB`                                   |
| VPC | `example-vpc`                             |
| 서브넷 | `example-subnet`                          |
| IP 주소 | `자동 할당`                                   |
| 보안그룹 | `example-vpc-default-sg`                  |
| 사용자 이름 | `cloudia`                                 |
| 비밀번호 | `cloudia`                               |

### 확인
- 생성 직후 인스턴스 상태가 `생성 중`으로 표시됩니다.
- 생성 완료 후 상태가 `종료됨`으로 전환됩니다.

<a id="step-4"></a>
## 4단계: 인스턴스 실행
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. 인스턴스 목록에서 `example-instance-001`을 선택합니다.
2. 상태가 `종료됨`인지 확인합니다.
3. `실행` 버튼을 클릭하고 확인 모달에서 실행을 확정합니다.

### 확인
- 상태가 `부팅 중`을 거쳐 `실행 중`으로 전환됩니다.
