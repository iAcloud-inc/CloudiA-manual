# 블록 생성과 인스턴스 연결하기

이 문서는 블록 스토리지를 먼저 생성한 뒤, **인스턴스 생성 과정에서 추가 블록 1개를 연결**하고 인스턴스 내부에서 마운트 및 `df -h`까지 확인하는 절차를 안내합니다.

## 사전 준비
- [가상 네트워크 생성과 Linux 인스턴스 생성하기](./03-create-network-and-linux-instance.md) 기준의 네트워크 준비 완료
  - VPC, 서브넷, 보안그룹
- 블록 생성 가능한 스토리지 도메인 준비
  - 예: `NFS-SD`
- Linux 이미지 준비
  - 예: `Rocky-9-GenericCloud-Base.latest.x86_64.qcow2`

## 개요
1. [블록 스토리지 생성](#step-1)
2. [인스턴스 생성 시 추가 블록 연결](#step-2)
3. [인스턴스 내부에서 디스크 확인](#step-3)
4. [마운트 및 `df -h` 확인](#step-4)

<a id="step-1"></a>
## 1단계: 블록 스토리지 생성
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `스토리지 > 블록`으로 이동합니다.
2. `생성` 버튼을 클릭합니다.
3. 이름, 스토리지 도메인, 크기를 입력하고 생성합니다.

### 입력 예시
| 항목 | 값                          |
|---|----------------------------|
| 이름 | `example-block-device-001` |
| 스토리지 도메인 | `NFS-SD`                   |
| 크기 | `50 GB`                    |

### 확인
- 블록 목록에 `example-block-device-001`이 표시됩니다.
- `부팅가능여부`가 `불가능`으로 표시됩니다.
- `연결된 인스턴스` 값이 `-`로 표시됩니다.

<a id="step-2"></a>
## 2단계: 인스턴스 생성 시 추가 블록 연결
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 인스턴스`로 이동합니다.
2. `생성` 버튼을 클릭합니다.
3. 인스턴스 기본 정보(이름, 타입, 이미지, 네트워크, 계정)를 입력합니다.
4. `고급 설정 > 스토리지 추가 > 블록`에서 `추가` 버튼을 클릭합니다.
5. `example-block-device-001`을 선택해 블록 목록에 추가합니다.
6. `생성` 버튼을 클릭합니다.
7. 생성 완료 후 인스턴스를 실행합니다.

### 입력 예시
| 항목 | 값                                         |
|---|-------------------------------------------|
| 인스턴스 이름 | `example-instance-storage-001`            |
| 인스턴스 타입 | `t2.medium`                               |
| 이미지 | `Rocky-9-GenericCloud-Base.latest.x86_64` |
| 이미지 디스크 크기 | `30 GB`                                   |
| VPC | `example-vpc`                             |
| 서브넷 | `example-subnet`                          |
| IP 주소 | `자동 할당`                                   |
| 보안그룹 | `example-vpc-default-sg`                  |
| 사용자 이름 | `cloudia`                                 |
| 비밀번호 | `cloudia`                                 |
| 추가 블록 | `example-block-device-001` (1개)           |

### 확인
- 인스턴스 생성 후 상태가 `생성 중 → 종료됨`으로 전환됩니다.
- 인스턴스 실행 후 상태가 `실행 중`으로 전환됩니다.
- 인스턴스 상세 `디스크` 섹션에 부트 디스크 + 추가 블록 1개가 표시됩니다.

<a id="step-3"></a>
## 3단계: 인스턴스 내부에서 디스크 확인
**수행 계정/화면:** `project-owner 계정` / `인스턴스 콘솔`

### 절차
1. 인스턴스 콘솔에 접속합니다.
2. `lsblk`를 실행해 디스크 목록을 확인합니다.

### 입력 예시
```bash
lsblk
```

### 확인
- 부트 디스크 외에 추가 블록 디스크가 1개 표시됩니다.

<a id="step-4"></a>
## 4단계: 마운트 및 `df -h` 확인
**수행 계정/화면:** `project-owner 계정` / `인스턴스 콘솔`

### 절차
1. 추가 블록 디스크에 파일시스템을 생성합니다.
2. 마운트 포인트를 생성하고 디스크를 마운트합니다.
3. `df -h`로 마운트 결과를 확인합니다.

### 입력 예시
```bash
# 예시 디바이스: /dev/sdb
sudo mkfs.ext4 /dev/sdb
sudo mkdir -p /mnt/data
sudo mount /dev/sdb /mnt/data
df -h
```

### 확인
- `/mnt/data`가 `df -h` 결과에 표시됩니다.
- 마운트된 경로에서 읽기/쓰기가 가능합니다.
