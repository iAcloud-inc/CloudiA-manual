# 플로팅 IP로 인스턴스 접속하기

이 문서는 플로팅 IP 생성/연결 후 보안그룹 인바운드 규칙을 적용해 SSH 접속을 확인하는 절차를 안내합니다.

## 사전 준비
- [가상 네트워크 생성과 Linux 인스턴스 생성하기](./03-create-network-and-linux-instance.md) 완료
- 대상 인스턴스 `실행 중`
- 플로팅 IP 자동 할당 가능

## 개요
1. [플로팅 IP 생성 및 인스턴스 연결](#step-1)
2. [보안그룹 인바운드 SSH 규칙 추가](#step-2)
3. [SSH 접속 확인](#step-3)

<a id="step-1"></a>
## 1단계: 플로팅 IP 생성 및 인스턴스 연결
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `네트워크 > 플로팅 IP`로 이동합니다.
2. `생성` 버튼을 클릭합니다.
3. VPC/외부 IP 설정 후 `리소스 연결`을 `적용`으로 변경합니다.
4. 대상 유형을 `인스턴스`로 선택하고 인스턴스/내부 IP를 지정합니다.
5. `생성` 버튼을 클릭합니다.

### 입력 예시
| 항목       | 값                      |
|----------|------------------------|
| 이름       | `example-fip-001`      |
| VPC      | `example-vpc`          |
| 외부 IP    | `자동 할당`                |
| 리소스 연결   | `적용`                   |
| 대상 유형    | `인스턴스`                 |
| 인스턴스     | `example-instance-001` |
| 내부 IP 주소 | `10.0.1.x`             |

### 확인
- 플로팅 IP가 생성되고 인스턴스에 연결된 상태로 표시됩니다.
- 외부 IP 목록의 사용 현황이 `플로팅 IP`로 표시됩니다.

<a id="step-2"></a>
## 2단계: 보안그룹 인바운드 SSH 규칙 추가
**수행 계정/화면:** `project-owner 계정` / `프로젝트 페이지`

### 절차
1. `보안 > 보안그룹`으로 이동합니다.
2. 대상 보안그룹 상세 화면으로 진입합니다.
3. `인바운드 규칙` 탭에서 `편집`을 클릭합니다.
4. `규칙 추가`를 클릭하고 SSH 허용 규칙을 입력합니다.
5. `규칙 저장`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 보안그룹 | `example-vpc-default-sg` |
| 프로토콜 | `TCP` |
| IP/CIDR | `0.0.0.0/0` |
| 포트 | `22` |
| 설명 | `inbound-ssh` |

### 확인
- 인바운드 규칙 목록에 신규 규칙이 표시됩니다.
- 인바운드 규칙 수가 즉시 반영됩니다.

<a id="step-3"></a>
## 3단계: SSH 접속 확인
**수행 계정/화면:** `project-owner 계정` / `로컬 터미널`

### 절차
1. 터미널에서 플로팅 IP로 SSH 접속을 시도합니다.
2. 접속 성공을 확인합니다.

### 입력 예시
```bash
ssh -o ConnectTimeout=10 cloudia@<floating-ip>
```

### 확인
- SSH 접속이 성공합니다.
- 로그인 프롬프트 또는 쉘 프롬프트가 표시됩니다.
