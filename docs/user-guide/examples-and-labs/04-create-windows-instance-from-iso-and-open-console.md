# 윈도우 가상머신 ISO 생성 및 콘솔 확인

이 문서는 `ISO 파일` 설치 유형으로 Windows 인스턴스를 생성하고, 콘솔에서 설치 화면을 확인하고, 필요하면 Windows 11의 네트워크 요구 사항을 우회해 로컬 계정으로 먼저 진입하는 실습입니다.

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- 인스턴스에서 사용할 VPC와 서브넷이 사전에 생성되어 있어야 함
- Windows ISO 이미지 업로드 완료
- 필요하면 `virtio-win.iso`를 준비하거나 연결 가능한 환경이어야 함

## 개요
1. [ISO 설치 유형으로 인스턴스 생성](#step-1)
2. [인스턴스 실행 및 콘솔 접속](#step-2)
3. [Windows 11 네트워크 연결 요구 사항 우회](#step-3)
4. [VirtIO 드라이버 설치](#step-4)

<a id="step-1"></a>
## 1단계: ISO 설치 유형으로 인스턴스 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 인스턴스`에서 `생성`을 클릭합니다.
2. 이름, 인스턴스 구성(vCPU/메모리)을 입력합니다.
3. 설치 유형에서 `ISO 파일`을 선택합니다.
4. ISO 파일을 선택하고 부팅 디스크 크기를 입력합니다.
5. 네트워크 인터페이스(VPC/서브넷/보안그룹)를 설정합니다.
6. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 인스턴스 이름 | `example-instance-win-iso-001` |
| vCPU | `4` |
| 메모리 | `8 GB` |
| 설치 유형 | `ISO 파일` |
| ISO 파일 | `Win11_25H2_Korean_x64_v2.iso` |
| 부팅 디스크 크기 | `64 GB` |
| VPC | `example-vpc` |
| 서브넷 | `example-public-subnet-001` |
| 보안그룹 | `example-vpc-default-sg` |

### 확인
- 인스턴스 생성 후 상태가 `종료됨`으로 표시됩니다.
- 상세 페이지의 설치 유형이 `ISO 파일`로 표시됩니다.

<a id="step-2"></a>
## 2단계: 인스턴스 실행 및 콘솔 접속
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 인스턴스를 선택하고 `실행`을 클릭합니다.
2. 상태가 `실행 중`이 되면 `콘솔`을 클릭합니다.
3. 부팅 장치 선택 화면이 보이면 첫 번째 `QEMU DVD-ROM` 항목을 선택합니다.
4. 콘솔 화면에서 Windows 설치 초기 화면을 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 인스턴스 | `example-instance-win-iso-001` |
| 목표 상태 | `실행 중` |
| 부팅 장치 | 첫 번째 `QEMU DVD-ROM` |

### 확인
- 콘솔 창이 정상 열립니다.
- Windows 설치 화면이 표시됩니다.

<a id="step-3"></a>
## 3단계: Windows 11 네트워크 연결 요구 사항 우회
**수행 계정/화면:** `인스턴스 콘솔` / `Windows 설치 화면`

이 단계는 Windows 11 설치 중 네트워크 드라이버가 아직 잡히지 않았거나, 일단 Windows 바탕화면에 먼저 진입한 뒤 드라이버를 설치하고 싶을 때 사용합니다.

### 절차
1. 네트워크 연결 화면에서 `Shift + F10`을 눌러 명령 프롬프트를 엽니다.
2. 다음 명령을 입력하고 Enter를 누릅니다.

```cmd
OOBE\BYPASSNRO
```

3. VM이 자동으로 재부팅되면 설치 과정을 다시 진행합니다.
4. 네트워크 연결 화면에서 `인터넷에 연결되어 있지 않음`을 선택합니다.
5. `제한된 설치로 계속`을 클릭해 로컬 계정으로 Windows에 먼저 진입합니다.
### 확인
- 네트워크 요구 사항 화면을 통과해 로컬 계정으로 Windows에 진입할 수 있습니다.

<a id="step-4"></a>
## 4단계: VirtIO 드라이버 설치
**수행 계정/화면:** `인스턴스 콘솔` / `Windows`

Windows에서는 `virtio-win.iso`를 이용해 네트워크, 스토리지, `QEMU Guest Agent`를 설치합니다. 가장 쉬운 방법은 통합 설치 파일을 실행하는 것이고, 필요한 경우에는 `장치 관리자`에서 개별 드라이버 경로를 지정합니다.

### 방법 A. 통합 설치 파일 사용
1. Windows에 로그인합니다.
2. `File Explorer > This PC`로 이동합니다.
3. CD-ROM에 연결된 `virtio-win.iso`를 엽니다.
4. ISO 루트의 `virtio-win-guest-tools`를 실행합니다.
5. 설치 마법사에 따라 설치를 완료합니다.
6. 필요하면 Windows를 재부팅합니다.

### 방법 B. 장치 관리자에서 개별 설치
1. `장치 관리자(Device Manager)`를 엽니다.
2. 경고 표시가 있는 네트워크 또는 스토리지 장치를 선택합니다.
3. `드라이버 업데이트`를 클릭합니다.
4. `드라이버를 검색하여 설치`를 선택합니다.
5. `virtio-win.iso` 안의 해당 드라이버 경로를 지정합니다.

### 드라이버 경로 예시
| 용도 | 경로 예시 |
|---|---|
| Windows 11 네트워크 드라이버 | `NetKVM\w11\amd64` |
| Windows Server 2022 네트워크 드라이버 | `NetKVM\2k22\amd64` |
| Windows 11 VirtIO-SCSI 드라이버 | `vioscsi\w11\amd64` |
| Windows Server 2022 VirtIO-SCSI 드라이버 | `vioscsi\2k22\amd64` |

### 확인
- `장치 관리자`에서 네트워크 또는 스토리지 장치의 경고 표시가 줄어들거나 해제됩니다.
- 필요하면 `Computer Management > Services`에서 `QEMU Guest Agent` 서비스 상태를 확인할 수 있습니다.

> 참고
>
> `virtio-win-guest-tools`를 실행하면 주요 `VirtIO` 드라이버와 `QEMU Guest Agent`를 함께 설치할 수 있습니다. 특정 장치만 직접 설치해야 하면 `장치 관리자`에서 위 경로를 지정합니다.
