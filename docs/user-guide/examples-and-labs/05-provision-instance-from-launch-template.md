# 시작 템플릿을 통한 빠른 인스턴스 구성

이 문서는 시작 템플릿을 생성하고 템플릿 기반으로 인스턴스를 빠르게 생성하는 실습입니다.

## 사전 준비
- CloudiA 웹 콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- 시작 템플릿 생성에 사용할 VPC/서브넷/보안그룹/이미지가 사전에 준비되어 있어야 함

## 개요
1. [시작 템플릿 생성](#step-1)
2. [템플릿으로 인스턴스 생성](#step-2)

<a id="step-1"></a>
## 1단계: 시작 템플릿 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 시작 템플릿`에서 `생성`을 클릭합니다.
2. 이름/설명, 인스턴스 타입, OS 이미지를 설정합니다.
3. VPC/서브넷/보안그룹을 설정합니다.
4. 사용자 계정과 인증(비밀번호 또는 SSH 키)을 설정합니다.
5. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 템플릿 이름 | `example-launch-template-001` |
| 설명 | `example launch template for quick provisioning` |
| 인스턴스 타입 | `t2.medium` |
| 이미지 | `Rocky-9-GenericCloud-Base.latest.x86_64.qcow2` |
| VPC | `example-vpc` |
| 서브넷 | `example-subnet-001` |
| IP 주소 | `자동 할당`                                   |
| 보안그룹 | `example-vpcDefaultSG` |
| 사용자 이름 | `cloudia` |
| 비밀번호 | `cloudia` |

### 확인
- 템플릿 목록에 `example-launch-template-001`이 표시됩니다.
- 상세 정보가 입력값과 일치합니다.

<a id="step-2"></a>
## 2단계: 템플릿으로 인스턴스 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 템플릿 목록에서 생성한 템플릿을 선택합니다.
2. `인스턴스 생성`을 클릭합니다.
3. 인스턴스 이름을 입력하고 `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 시작 템플릿 | `example-launch-template-001` |
| 인스턴스 이름 | `example-instance-from-template-001` |

### 확인
- 인스턴스 목록에 새 인스턴스가 생성됩니다.
- 인스턴스 사양/네트워크/인증 설정이 템플릿과 일치합니다.
