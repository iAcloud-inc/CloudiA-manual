# 인스턴스 마이그레이션하기

이 문서는 컴퓨트 노드 화면에서 인스턴스를 다른 호스트로 마이그레이션하는 실습입니다.
라이브/콜드 방식의 차이와 제약을 함께 확인합니다.

## 시작하기 전 
### 라이브 마이그레이션과 콜드 마이그레이션 비교
| 구분 | 라이브 마이그레이션 | 콜드 마이그레이션 |
|---|---|---|
| 동작 방식 | 실행 중인 인스턴스를 중단 최소화 상태로 이동 | 마이그레이션 시 인스턴스 재기동(중단 발생 가능) 기반으로 이동 |
| 대상 인스턴스 상태 | 현재 `실행 중` 인스턴스 대상 | 인스턴스 `실행/종료` 여부와 무관하게 대상 지정 가능 |
| 서비스 영향 | 상대적으로 짧은 끊김 또는 무중단에 가까움 | 라이브 마이그레이션보다 중단 시간이 길 수 있음 |
| 대상 노드 선택 | 제약 조건을 만족하는 노드만 선택 가능 | 라이브 마이그레이션 대비 선택 가능한 노드 범위가 넓음 |
| 주요 제약 | 보안 타입, CPU 피닝, NPU 패스스루, Windows OS 등 조건에 따라 제한될 수 있음 | 라이브 마이그레이션 대비 제약이 적음 |
| 권장 상황 | 운영 중 서비스 영향 최소화가 필요한 경우 | 유지보수/점검처럼 짧은 중단 허용이 가능한 경우 |

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- `admin` 계정으로 로그인 가능
- 실행 중 인스턴스 1개 이상 존재
- 컴퓨트 노드 2대 이상 존재

## 개요
1. [라이브 마이그레이션 수행](#step-1)
2. [콜드 마이그레이션 수행](#step-2)


<a id="step-1"></a>
## 1단계: 라이브 마이그레이션 수행
**수행 계정/화면:** `admin 계정` / `어드민 페이지`

### 절차
1. 마이그레이션 모달에서 `라이브 마이그레이션` 토글을 켭니다.
2. 대상 노드를 선택하고 `실행`을 클릭합니다.
3. 완료 후 소스 노드/대상 노드 인스턴스 목록을 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 라이브 마이그레이션 | `켜짐` |
| 대상 노드 | `compute-node-002` |

### 확인
- 마이그레이션 요청/완료 토스트가 표시됩니다.
- 인스턴스가 대상 노드로 이동합니다.
- 상태가 `실행 중`으로 유지됩니다.

<a id="step-2"></a>
## 2단계: 콜드 마이그레이션 수행
**수행 계정/화면:** `admin 계정` / `어드민 페이지`

### 절차
1. 동일 인스턴스 또는 다른 인스턴스로 마이그레이션 모달을 다시 엽니다.
2. `라이브 마이그레이션` 토글을 끕니다.
3. 대상 노드를 선택하고 `실행`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 라이브 마이그레이션 | `꺼짐` |
| 대상 노드 | `compute-node-002` |

### 확인
- 콜드 모드에서도 마이그레이션이 정상 완료됩니다.
- 대상 노드에서 인스턴스가 확인됩니다.
