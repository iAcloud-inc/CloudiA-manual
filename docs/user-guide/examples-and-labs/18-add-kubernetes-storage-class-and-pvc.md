# 쿠버네티스 스토리지 클래스 추가 및 PVC 생성

이 문서는 쿠버네티스 클러스터의 스토리지 탭에서 스토리지 클래스를 만들고 PVC를 생성하는 실습입니다.

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- [쿠버네티스 클러스터 구성하기](../quickstarts/06-configure-kubernetes-cluster.md) 완료
- 스토리지 클래스로 사용할 NFS 파일 시스템이 사전에 준비되어 있어야 함

## 개요
1. [스토리지 클래스 생성](#step-1)
2. [PVC 생성](#step-2)
3. [바인딩 상태 확인](#step-3)

<a id="step-1"></a>
## 1단계: 스토리지 클래스 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `쿠버네티스 > 클러스터`에서 대상 클러스터 상세로 이동합니다.
2. `스토리지` 탭에서 스토리지 타입을 `스토리지 클래스`로 선택합니다.
3. `스토리지 클래스 생성`을 클릭합니다.
4. 이름, 파일 시스템, 반환 정책을 입력합니다.
5. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 클러스터 | `example-cluster-001` |
| 이름 | `example-sc-001` |
| 파일 시스템 | `example-fs-nfs-001` |
| 반환 정책 | `Delete` |

### 확인
- 스토리지 클래스 목록에 `example-sc-001`이 표시됩니다.

<a id="step-2"></a>
## 2단계: PVC 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 스토리지 탭에서 스토리지 타입을 `PVC`로 선택합니다.
2. `PVC 생성`을 클릭합니다.
3. 이름, 용량, 스토리지 클래스, 네임스페이스, 접근 모드를 입력합니다.
4. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 이름 | `example-pvc-001` |
| 용량 | `1 GiB` |
| 스토리지 클래스 | `example-sc-001` |
| 네임스페이스 | `default` |
| 접근 모드 | `ReadWriteMany` |

### 확인
- PVC 목록에 `example-pvc-001`이 표시됩니다.
- 상태가 `Pending` 또는 `Bound`로 표시됩니다.

<a id="step-3"></a>
## 3단계: 바인딩 상태 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. PVC 목록에서 생성한 PVC의 상태와 볼륨 컬럼을 확인합니다.
2. 필요 시 잠시 대기 후 새로고침합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 PVC | `example-pvc-001` |
| 확인 항목 | `상태, 볼륨, 설정 용량, 접근 모드` |

### 확인
- PVC가 정상 바인딩되면 `Bound` 상태로 표시됩니다.
- 용량/접근 모드가 입력값과 일치합니다.
