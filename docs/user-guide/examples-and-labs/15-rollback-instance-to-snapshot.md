# 인스턴스 스냅샷 롤백하기

이 문서는 실행 중 인스턴스에서 롤백 제한을 확인하고, 종료 상태에서 스냅샷 롤백을 수행하는 실습입니다.

## 사전 준비
- CloudiA 웹 콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- [스케줄러를 통한 주기적인 인스턴스 스냅샷 생성](./14-schedule-recurring-instance-snapshots.md) 또는 수동 스냅샷 생성 완료

## 개요
1. [롤백 전 파일 변경](#step-1)
2. [실행 중 롤백 제한 확인](#step-2)
3. [종료 후 롤백 수행](#step-3)

<a id="step-1"></a>
## 1단계: 롤백 전 파일 변경
**수행 계정/화면:** `프로젝트 계정 + 인스턴스 콘솔` / `프로젝트 페이지`

### 절차
1. 인스턴스 콘솔에 접속합니다.
2. 테스트 파일을 생성하고 내용(`before snapshot`)을 기록합니다.
3. 스냅샷을 생성합니다.
4. 파일 내용을 `after snapshot`으로 변경합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 테스트 파일 | `/tmp/example-rollback-test.txt` |
| 변경 전 내용 | `before snapshot` |
| 변경 후 내용 | `after snapshot` |
| 스냅샷 이름 | `example-snapshot-rollback-001` |

### 확인
- 스냅샷 생성 후 파일 내용이 변경된 상태임을 확인합니다.

<a id="step-2"></a>
## 2단계: 실행 중 롤백 제한 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 인스턴스가 `실행 중` 상태인지 확인합니다.
2. 상세의 `스냅샷` 탭에서 스냅샷을 선택합니다.
3. `롤백` 버튼 상태를 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 인스턴스 상태 | `실행 중` |

### 확인
- 실행 중 상태에서는 `롤백` 버튼이 비활성화됩니다.

<a id="step-3"></a>
## 3단계: 종료 후 롤백 수행
**수행 계정/화면:** `프로젝트 계정 + 인스턴스 콘솔` / `프로젝트 페이지`

### 절차
1. 인스턴스를 `종료됨` 상태로 전환합니다.
2. 상세 `스냅샷` 탭에서 대상 스냅샷을 선택합니다.
3. `롤백`을 실행합니다.
4. 인스턴스를 다시 실행하고 콘솔에 접속합니다.
5. 테스트 파일 내용을 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 롤백 대상 스냅샷 | `example-snapshot-rollback-001` |
| 확인 명령 | `cat /tmp/example-rollback-test.txt` |
| 기대 내용 | `before snapshot` |

### 확인
- 롤백 완료 후 파일 내용이 스냅샷 시점으로 복원됩니다.
- 롤백 완료 토스트가 표시됩니다.
