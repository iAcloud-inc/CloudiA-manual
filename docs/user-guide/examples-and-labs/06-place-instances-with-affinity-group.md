# 선호도 그룹을 통한 가상머신 배치

이 문서는 VM 2개를 반선호도(서로 다른 호스트 배치)로 설정하고,
종료된 VM을 실행하면서 실제로 다른 물리 노드에 배치되는지 확인하는 실습입니다.

## 사전 준비
- CloudiA 웹 콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- 선호도 그룹을 적용할 VM 인스턴스 2개가 `종료됨` 상태로 준비되어 있어야 함
- VM이 배치될 수 있는 컴퓨트 호스트가 최소 2대 이상 `정상` 상태여야 함

## 개요
1. [반선호도 선호도 그룹 생성](#step-1)
2. [대상 가상머신 지정](#step-2)
3. [종료된 VM 실행 후 다른 노드 배치 확인](#step-3)

<a id="step-1"></a>
## 1단계: 반선호도 선호도 그룹 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 선호도 그룹`에서 `생성`을 클릭합니다.
2. 이름/설명/활성화 여부를 입력합니다.
3. 가상머신 선호도를 `반선호도`, 강제성을 `강제`로 설정합니다.
4. 물리 머신 선호도는 `사용 안함`으로 둡니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 이름 | `example-affinity-group-001` |
| 설명 | `vm anti-affinity policy` |
| 활성화 여부 | `활성화` |
| 가상머신 선호도 | `반선호도` |
| 가상머신 강제성 | `강제` |
| 물리머신 선호도 | `사용 안함` |

### 확인
- 목록에 선호도 그룹이 생성됩니다.
- 활성화 상태가 `활성화`로 표시됩니다.
- 상세에서 `가상머신 반선호도 + 강제`가 표시됩니다.

<a id="step-2"></a>
## 2단계: 대상 가상머신 지정
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 생성한 그룹 상세로 이동합니다.
2. `가상머신` 탭에서 `편집`을 클릭해 대상 VM을 추가합니다.
3. 저장 후 대상 VM 2개가 등록되었는지 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 가상머신 | `example-vm-01`, `example-vm-02` |

### 확인
- 가상머신 탭에 VM 2개가 표시됩니다.
- 저장 후 새로고침해도 대상이 유지됩니다.

<a id="step-3"></a>
## 3단계: 종료된 VM 실행 후 다른 노드 배치 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 인스턴스`에서 대상 VM 2개의 상태가 `종료됨`인지 확인합니다.
2. `example-vm-01`을 먼저 `실행`합니다.
3. `example-vm-02`를 `실행`합니다.
4. 각 VM 상세 화면에서 `물리 머신` 항목을 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 VM 1 | `example-vm-01 (종료됨 -> 실행 중)` |
| 대상 VM 2 | `example-vm-02 (종료됨 -> 실행 중)` |
| 확인 항목 | `인스턴스 상세 > 물리 머신` 값 비교 |

### 확인
- 두 VM의 `물리 머신` 값이 서로 다르게 표시됩니다.
- 호스트 자원이 부족하거나 규칙을 만족하지 못하면 두 번째 VM 실행이 제한될 수 있습니다.
