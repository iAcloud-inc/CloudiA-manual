# 스케줄러를 통한 주기적인 인스턴스 스냅샷 생성

이 문서는 스케줄러에서 Cron 기반 일정으로 인스턴스 스냅샷을 주기적으로 생성하는 실습입니다.

## 사전 준비
- CloudiA 웹 콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- 스냅샷 대상 인스턴스 1개 이상 존재

## 개요
1. [스냅샷 일정 생성](#step-1)
2. [일정 목록 확인](#step-2)
3. [스냅샷 생성 결과 확인](#step-3)

<a id="step-1"></a>
## 1단계: 스냅샷 일정 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `스케줄러 > 일정`에서 `생성`을 클릭합니다.
2. 이름/설명 입력 후 일정 유형을 `Cron`으로 선택합니다.
3. Cron 표현식(분/시/일/월/요일)을 입력합니다.
4. 서비스로 `스냅샷 생성`을 선택합니다.
5. 대상 인스턴스를 추가합니다.
6. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 일정 이름 | `example-sched-snapshot-001` |
| 설명 | `scheduled snapshot every 5 minutes` |
| Cron 분 | `*/5` |
| Cron 시 | `*` |
| Cron 일 | `*` |
| Cron 월 | `*` |
| Cron 요일 | `*` |
| 서비스 | `스냅샷 생성` |
| 대상 인스턴스 | `example-instance-snapshot-001` |

### 확인
- 생성 요청이 성공하고 목록으로 이동합니다.
- 일정이 목록에 표시됩니다.

<a id="step-2"></a>
## 2단계: 일정 목록 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 일정 목록에서 생성한 스케줄을 검색합니다.
2. 표현식/서비스/활성화 상태를 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 검색어 | `example-sched-snapshot-001` |

### 확인
- 유형이 `CRON`, 서비스가 `스냅샷 생성`으로 표시됩니다.
- 표현식이 입력값과 일치합니다.

<a id="step-3"></a>
## 3단계: 스냅샷 생성 결과 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 스케줄 실행 시점 이후 `가상머신 > 인스턴스`로 이동합니다.
2. 대상 인스턴스 상세의 `스냅샷` 탭을 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 인스턴스 | `example-instance-snapshot-001` |
| 확인 시간 | `스케줄 등록 후 5~10분` |

### 확인
- 스냅샷 목록에 신규 항목이 생성됩니다.
- 스냅샷 생성 시각이 스케줄 주기와 맞게 증가합니다.
