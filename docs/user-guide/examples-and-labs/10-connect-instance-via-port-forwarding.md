# 포트포워딩으로 인스턴스 연결하기

이 문서는 외부 IP 자동 할당 기반 포트포워딩을 생성하고 SSH 연결까지 확인하는 실습입니다.

## 사전 준비
- CloudiA 웹 콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- 접속 대상 Linux 인스턴스 실행 중
- 인스턴스 보안그룹에서 `22/tcp` 인바운드 허용

## 개요
1. [포트포워딩 생성](#step-1)
2. [SSH 포워딩 규칙 추가](#step-2)
3. [외부 SSH 접속 확인](#step-3)

<a id="step-1"></a>
## 1단계: 포트포워딩 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `네트워크 > 포트 포워딩`에서 `생성`을 클릭합니다.
2. 이름, 설명, VPC를 입력합니다.
3. 외부 IP를 `자동 할당`으로 둡니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 이름 | `example-port-forwarding-ssh-001` |
| 설명 | `ssh forwarding test` |
| VPC | `example-vpc` |
| 외부 IP | `자동 할당` |

### 확인
- 생성 화면에서 외부 IP 자동 할당이 설정됩니다.

<a id="step-2"></a>
## 2단계: SSH 포워딩 규칙 추가
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 규칙 `추가`를 클릭합니다.
2. 프로토콜 `TCP`, 외부 포트 `10022`를 입력합니다.
3. 내부 리소스를 `인스턴스`로 선택하고 대상 VM/내부 IP를 선택합니다.
4. 내부 포트 `22`를 입력하고 저장합니다.
5. 포트포워딩을 생성합니다.

### 입력 예시
| 항목 | 값                          |
|---|----------------------------|
| 프로토콜 | `TCP`                      |
| 외부 포트 | `10022`                    |
| 내부 리소스 | `인스턴스`                     |
| 대상 인스턴스 | `example-instance-ssh-001` |
| 내부 IP | `10.0.1.21`                |
| 내부 포트 | `22`                       |

### 확인
- 규칙 목록에 `TCP :10022 -> 10.0.1.21:22`가 표시됩니다.
- 포트포워딩 목록에 외부 IP가 할당됩니다.

<a id="step-3"></a>
## 3단계: 외부 SSH 접속 확인
**수행 계정/화면:** `로컬 터미널` / `SSH 클라이언트`

### 절차
1. 포트포워딩 외부 IP를 확인합니다.
2. 로컬에서 포워딩 포트로 SSH 접속을 시도합니다.

### 입력 예시
| 항목 | 값                                                                      |
|---|------------------------------------------------------------------------|
| 접속 명령 | `ssh -p 10022 example-user@192.168.11.13`                              |
| 키 기반 접속 명령 | `ssh -i ~/.ssh/example-key-ed25519 -p 10022 example-user@192.168.11.13` |

### 확인
- SSH 연결이 성공합니다.
- 접속 대상이 지정한 인스턴스와 일치합니다.
