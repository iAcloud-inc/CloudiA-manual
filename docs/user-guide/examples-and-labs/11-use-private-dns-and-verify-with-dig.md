# 내부 DNS 활용하기

이 문서는 내부 DNS를 생성하고 A 레코드를 등록한 뒤 `dig`로 해석 결과를 확인하는 실습입니다.

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- DNS 질의용 내부 인스턴스 1대 준비(`dig` 명령어 사용 가능)
- [포트 포워딩으로 인스턴스 연결하기](./10-connect-instance-via-port-forwarding.md) 또는 인스턴스/로드 밸런서/플로팅 IP 리소스 준비

## 개요
1. [내부 DNS 생성](#step-1)
2. [A 레코드 등록](#step-2)
3. [dig로 해석 확인](#step-3)

<a id="step-1"></a>
## 1단계: 내부 DNS 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `네트워크 > 내부 DNS`에서 `생성`을 클릭합니다.
2. 도메인 이름, VPC, 설명을 입력합니다.
3. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 도메인 이름 | `example-svc.internal` |
| VPC | `example-vpc` |
| 설명 | `private dns for internal services` |

### 확인
- 내부 DNS 상세에 기본 `NS`, `SOA` 레코드 2개가 표시됩니다.

<a id="step-2"></a>
## 2단계: A 레코드 등록
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 내부 DNS 상세에서 `편집`을 클릭합니다.
2. `추가`를 눌러 A 레코드를 생성합니다.
3. 리소스 타입을 선택하고 값(IP)을 추가합니다.
4. `저장`을 클릭합니다.

### 입력 예시
| 항목 | 값                            |
|---|------------------------------|
| 레코드 이름 | `app-1`                      |
| TTL | `300`                        |
| 타입 | `A`                          |
| 리소스 타입 | `인스턴스`                       |
| 값 | `10.0.1.31`                  |

### 확인
- 레코드 목록에 `app-1.example-svc.internal`가 표시됩니다.
- 레코드 수가 `3개`로 표시됩니다.

<a id="step-3"></a>
## 3단계: dig로 해석 확인
**수행 계정/화면:** `내부 인스턴스 터미널` / `CLI`

### 절차
1. 내부 인스턴스에서 `dig` 명령으로 FQDN을 조회합니다.
2. 출력 IP가 등록값과 일치하는지 확인합니다.
3. 필요 시 로컬 PC에서도 동일 질의를 수행해 내부 DNS 범위를 비교합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| dig 명령 | `dig +short app-1.example-svc.internal` |
| 기대 결과 | `10.0.1.31` |

### 확인
- 내부 인스턴스에서는 A 레코드가 정상 해석됩니다.
- 외부 환경에서는 해석 실패(`NXDOMAIN`)가 발생합니다.
