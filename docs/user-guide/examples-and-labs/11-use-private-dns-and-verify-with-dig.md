# 내부 DNS 활용하기

이 문서는 VPC 내 리소스(인스턴스)에 고유한 도메인 이름을 부여하는 내부 DNS 구축과 레코드 해석 과정을 다루는 실습입니다. 복잡한 IP 주소 대신 직관적인 도메인을 사용하여 내부 서비스 간 연결 편의성을 높일 수 있습니다.

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- DNS 질의를 수행할 Rocky Linux 계열(Rocky, CentOS, RHEL) 내부 인스턴스 1대 필요
    - 참고: 원활한 실습을 위해 Rocky Linux 기반 인스턴스 사용을 권장합니다.
    - 질의용 인스턴스에 dig 명령어가 포함된 bind-utils 패키지가 설치되어 있어야 합니다. (설치 명령: `sudo dnf install -y bind-utils`)

## 개요
1. [내부 DNS 생성](#step-1)
2. [A 타입 레코드 등록](#step-2)
3. [dig로 해석 확인](#step-3)

<a id="step-1"></a>
## 1단계: 내부 DNS 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `네트워크 > 내부 DNS`에서 `생성`을 클릭합니다.
2. 도메인 이름, VPC, 설명을 입력합니다.
3. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 도메인 이름 | `example-svc.internal` |
| VPC | `example-vpc` |
| 설명 | `private dns for internal services` |

### 확인
- 생성 후 상세 화면에서 `NS`, `SOA` 타입의 기본 레코드 2개가 자동으로 생성되었는지 확인하세요.

<a id="step-2"></a>
## 2단계: A 타입 레코드 등록
**수행 계정/화면:** `프로젝트 계정` / `내부 DNS 상세 페이지`

### 절차
1. 1단계에서 생성한 내부 DNS 상세 페이지 하단에서 `편집`을 클릭합니다.
2. 레코드 편집 화면에서 `추가`를 눌러 레코드 추가창을 엽니다.
3. 레코드 이름(`app-1`), TTL(`300`), 타입(`A`)을 설정합니다.
4. 리소스 타입을 `인스턴스`로 선택한 뒤, 대상 인스턴스와 IP를 지정하고 `추가`를 눌러 값을 확정합니다.
5. 레코드 추가창 하단의 `추가`를 눌러 레코드 구성을 완료합니다.
5. 마지막으로 페이지 하단의 `저장`을 클릭하여 반영합니다.

### 입력 예시
| 항목 | 값                            |
|---|------------------------------|
| 레코드 이름 | `app-1`                      |
| TTL | `300`                        |
| 타입 | `A`                          |
| 리소스 타입 | `인스턴스`                       |
| 인스턴스 | `example-instance-dns-001`                       |
| 내부 IP 주소 | `10.0.1.31`                  |

### 확인
- 레코드 목록에 `app-1.example-svc.internal`가 표시됩니다.
- 레코드 수가 `3개`로 표시됩니다.

<a id="step-3"></a>
## 3단계: dig로 해석 확인
**수행 계정/화면:** `내부 인스턴스 터미널 (Rocky Linux)`

### 절차
1. 내부 인스턴스 상세 화면에서 `콘솔`을 클릭하여 터미널에 접속합니다.
2. 터미널 창에서 dig 명령을 사용하여 생성한 FQDN(전체 도메인 경로)을 조회합니다.
3. 출력된 IP 주소가 등록한 값과 일치하는지 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| dig 명령어 | `dig +short app-1.example-svc.internal` |
| 기대 결과 |  `10.0.1.31` (등록된 IP 주소 출력) |

### 확인
- 내부 인스턴스에서는 정상적으로 주소가 해석되지만, 외부 로컬 PC 등에서는 보안 격리 정책에 따라 해석 실패(`NXDOMAIN`)가 발생하는 것이 정상입니다.
