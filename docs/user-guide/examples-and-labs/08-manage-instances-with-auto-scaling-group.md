# 오토 스케일링 그룹을 통한 인스턴스 관리

이 문서는 시작 템플릿 기반으로 오토 스케일링 그룹을 생성하고 타겟 그룹을 함께 연결해 인스턴스를 관리하는 실습입니다.

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- 프로젝트 계정(`project-owner`)으로 로그인 가능
- [시작 템플릿을 통한 빠른 인스턴스 구성](./05-provision-instance-from-launch-template.md) 완료
- 타겟 그룹 연결 가능한 네트워크 준비

## 개요
1. [오토 스케일링 그룹 생성](#step-1)
2. [타겟 그룹 연결 확인](#step-2)
3. [용량 조정 동작 확인](#step-3)

<a id="step-1"></a>
## 1단계: 오토 스케일링 그룹 생성
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. `가상머신 > 오토 스케일링 그룹`에서 `생성`을 클릭합니다.
2. 이름, 시작 템플릿, 설명을 입력합니다.
3. 타겟 그룹을 `생성`으로 선택하고 프로토콜/포트를 입력합니다.
4. 최소/최대/원하는 용량, 스케일 간격, 정책 값을 입력합니다.
5. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값                             |
|---|-------------------------------|
| 이름 | `example-asg-001`             |
| 시작 템플릿 | `example-launch-template-001` |
| 타겟 그룹 | `생성`                          |
| 프로토콜 | `TCP`                         |
| 포트 번호 | `8080`                        |
| 최소 용량 | `2`                           |
| 최대 용량 | `4`                           |
| 원하는 용량 | `2`                           |
| 스케일 간격 | `180`                         |
| 지표 유형 | `CPU 평균 사용량`                  |
| 스케일 인 대상 | `20`                          |
| 스케일 아웃 대상 | `80`                          |

### 확인
- 오토 스케일링 그룹이 생성되고 상태가 `실행 중`으로 전환됩니다.
- 인스턴스 상태가 `2/2`로 표시됩니다.

<a id="step-2"></a>
## 2단계: 타겟 그룹 연결 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 생성된 오토 스케일링 그룹 상세로 이동합니다.
2. 상단의 타겟 그룹 정보를 확인합니다.
3. `네트워크 > 타겟 그룹` 목록에서도 연동 상태를 확인합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 오토 스케일링 그룹 | `example-asg-001` |
| 생성된 타겟 그룹 | `example-asg-001` |

### 확인
- 오토 스케일링 그룹 상세에 타겟 그룹 이름/ID가 표시됩니다.
- 타겟 그룹에 오토 스케일링 그룹 인스턴스가 등록됩니다.

<a id="step-3"></a>
## 3단계: 용량 조정 동작 확인
**수행 계정/화면:** `프로젝트 계정` / `프로젝트 페이지`

### 절차
1. 오토 스케일링 그룹 상세에서 `원하는 용량`을 변경해 수동 스케일 동작을 확인합니다.
2. 상태 변화를 모니터링합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 변경 전 원하는 용량 | `2` |
| 변경 후 원하는 용량 | `3` |

### 확인
- 인스턴스 수가 변경된 원하는 용량에 맞춰 반영됩니다.
- 생성된 인스턴스가 시작 템플릿 사양을 따릅니다.
