# 알림 설정 및 이벤트 트리거

이 문서는 SMTP 메일 알림 채널을 구성하고, 이벤트 레벨 기준 이벤트 트리거를 등록하는 실습입니다.

## 사전 준비
- CloudiA 웹 콘솔 접근 가능
- `admin` 계정으로 로그인 가능
- SMTP 서버 정보 및 테스트 수신 메일 주소 보유
- 알림을 받을 멤버 계정 존재

## 개요
1. [SMTP 메일 알림 설정](#step-1)
2. [이벤트 트리거 생성 및 알림 채널 등록](#step-2)

<a id="step-1"></a>
## 1단계: SMTP 메일 알림 설정
**수행 계정/화면:** `admin 계정` / `어드민 페이지`

### 절차
1. `환경설정 > 알람 설정`으로 이동합니다.
2. **메일 전송 설정(SMTP)** 탭에서 `편집` 버튼을 클릭합니다.
3. SMTP 서버/포트/발신자 정보/보안 연결을 입력합니다.
4. `편집`을 클릭해 저장합니다.
5. `연결 테스트`를 클릭하고 수신 메일을 입력해 발송합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| SMTP 서버명 | `smtp.example.local` |
| SMTP 포트 | `587` |
| 발신자 이메일 주소 | `example-alert@example.com` |
| 발신자 이메일 비밀번호 | `ExampleSmtpPassword!23` |
| 보안 연결 | `StartTLS` |
| 수신 이메일 주소 | `example-receiver@example.com` |

### 확인
- SMTP 설정 저장 성공 토스트가 표시됩니다.
- 연결 테스트 메일이 정상 수신됩니다.

<a id="step-2"></a>
## 2단계: 이벤트 트리거 생성 및 알림 채널 등록
**수행 계정/화면:** `admin 계정` / `어드민 페이지`

### 절차
1. `모니터링 > 이벤트 트리거`에서 `생성`을 클릭합니다.
2. 이름/설명을 입력합니다.
3. **이벤트 트리거 조건**의 `추가`를 클릭합니다.
4. 조건 유형 `이벤트 레벨`, 비교 연산자 `>= (이상)`, 값 `ERROR`, 알림 주기 `60초`를 설정하고 `추가`를 클릭합니다.
5. **알림 채널**의 `추가`를 클릭합니다.
6. 채널 타입 `Email`과 수신 사용자를 선택하고 `추가`를 클릭합니다.
6. `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 이름 | `example-event-trigger` |
| 설명 | `notify when event level is error or higher` |
| 조건 유형 | `이벤트 레벨` |
| 비교 연산자 | `>= (이상)` |
| 값 | `ERROR` |
| 알림 주기(초) | `60` |
| 채널 유형 | `이메일` |
| 수신 사용자 | `example-member` |

### 확인
- 이벤트 트리거가 목록에 생성됩니다.
- 상세에서 조건 `이벤트 레벨 >= ERROR`와 이메일 채널이 표시됩니다.
