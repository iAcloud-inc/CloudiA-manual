# 역할 만들기 및 권한 부여

이 문서는 Cloud:iA에서 역할을 생성하고 멤버에 연결하는 실습입니다.
조회 전용(Read-Only) 역할 구성과 계정 연결 방법을 다룹니다.

## 사전 준비
- Cloud:iA 웹콘솔 접근 가능
- `admin` 계정으로 로그인 가능
- [Cloud:iA 계정 생성하기](../quickstarts/01-create-account.md) 완료
- [프로젝트 생성하기](../quickstarts/02-create-project.md) 완료

## 개요
1. [Read-Only 역할 생성](#step-1)
2. [멤버 계정에 역할 연결](#step-2)

<a id="step-1"></a>
## 1단계: Read-Only 역할 생성
**수행 계정/화면:** `admin 계정` / `어드민 페이지`

### 절차
1. `접근 관리 > 역할`에서 `생성`을 클릭합니다.
2. 이름/설명을 입력합니다.
3. **권한 추가**의 `추가`를 클릭해 권한 선택 모달을 엽니다.
4. 검색 조건을 `구분`으로 바꾸고 `view`를 검색합니다.
5. 검색 결과를 `전체 선택`하고 `추가`를 클릭합니다.
6. 역할 생성 화면에서 `생성`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 역할 이름 | `example-role-readonly` |
| 설명 | `read-only role for project members` |
| 검색 필드 | `구분` |
| 검색어 | `view` |

### 확인
- 목록에 `example-role-readonly`가 생성됩니다.
- 상세 권한 탭에서 `*-view` 권한이 일괄 등록된 것을 확인합니다.

<a id="step-2"></a>
## 2단계: 멤버 계정에 역할 연결
**수행 계정/화면:** `admin 계정` / `어드민 페이지`

### 절차
1. `접근 관리 > 멤버`에서 대상 멤버를 선택합니다.
2. 하단 `역할` 영역에서 `편집`을 클릭합니다.
3. 프로젝트와 `example-role-readonly`를 선택합니다.
4. `저장`을 클릭합니다.

### 입력 예시
| 항목 | 값 |
|---|---|
| 대상 멤버 | `example-member` |
| 대상 프로젝트 | `example-project` |
| 부여 역할 | `example-role-readonly` |

### 확인
- 멤버 상세 `역할` 영역에 설정한 역할이 반영됩니다.
- 저장 후 새로고침해도 역할 연결이 유지됩니다.
